The rapid expansion of LLM ecosystems into vast multi-agent networks has shifted the bottleneck from capability generation to **task routing**. The core challenge addressed is "Tool-to-Agent Retrieval": accurately mapping a user's query to the correct managing agent when functionality is distributed across hundreds of granular tools. Traditional architectures isolate tools and agents, creating a semantic disconnect where tools are found, but their owning agents remain hidden. This isolation prevents dynamic resource discovery and limits scalability in complex, multi-step environments.

1.  **Broad Retrieval:** One-pass search of top $N$ candidates across both corpora.
2.  **Candidate Aggregation:** Mapping matched tools back to parent agents and merging with directly matched agents.
3.  **Deduplication & Ranking:** Selecting top $K$ unique agents.

## Technical Details

### ğŸ§± Core Concept & Problem Formulation
*   **Unified Vector Space:** Embeds both tools and parent agents into the same semantic space to bridge the disconnect between granular functionality and high-level management.
*   **Graph Modeling:** The system is modeled as a bipartite graph $G = (A, T, E)$, where:
    *   $A$ represents the set of Agents.
    *   $T$ represents the set of Tools.
    *   $E$ represents the edges (relationships) between them.
*   **Goal:** One-pass retrieval of either tools or agent bundles.

### ğŸ—„ï¸ Indexing Architecture (Catalog Structure)
The architecture relies on a unified index $C$ composed of two distinct but linked parts:
1.  **Tool Corpus ($C_T$):** Contains tool metadata linked to their owners.
2.  **Agent Corpus ($C_A$):** Contains agent descriptions and profiles.
*   **Metadata Traversal:** The system allows for navigation from specific tools in $C_T$ up to their parent agents in $C_A$.

### âš™ï¸ Retrieval Algorithm (Combined Toolâ€“Agent Top-K)
The retrieval process operates in three distinct phases:
1.  **Broad Retrieval:** Identifies the top $N$ entities from the combined corpus ($C_T + C_A$) using semantic similarity.
2.  **Candidate Aggregation:** Maps retrieved tools back to their parent agents and adds directly retrieved agents to the candidate pool.
3.  **Deduplication & Ranking:** Filters duplicates and ranks the results to select the final top $K$ unique agents.

**Query Paradigms Supported:**
*   **Direct Querying:** Standard search against the index.
*   **Step-wise Querying:** Iterative refinement.
