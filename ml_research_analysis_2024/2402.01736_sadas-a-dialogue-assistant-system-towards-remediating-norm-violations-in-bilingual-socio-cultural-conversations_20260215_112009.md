# SADAS: A Dialogue Assistant System Towards Remediating Norm Violations in Bilingual Socio-Cultural Conversations

## Quick Facts
- arXiv ID: 2402.01736
- Source URL: https://arxiv.org/abs/2402.01736
- Authors: Yuncheng Hua; Zhuang Li; Linhao Luo; Kadek Ananta Satriadi; Tao Feng; Haolan Zhan; Lizhen Qu; Suraj Sharma; Ingrid Zukerman; Zhaleh Semnani-Azad; Gholamreza Haffari
- Reference count: 12
- Primary result: First real-time system for detecting and remediating norm violations in bilingual socio-cultural conversations, achieving 48% success rate in mitigating high-impact violations

## Executive Summary
SADAS is the first real-time system designed to detect and remediate norm violations in bilingual socio-cultural conversations. The system addresses the challenge of ensuring culturally appropriate communication between individuals with different linguistic and cultural backgrounds. SADAS employs a pipeline architecture with modules for norm category classification, violation detection, impact assessment, remediation generation, and justification. The system uses state-of-the-art NLP techniques, including ChatGPT and ChatYuan models, and was evaluated on the SocialDial dataset. In a user study with 40 dialogues, SADAS successfully mitigated high-impact norm violations in approximately 48% of cases.

## Method Summary
SADAS implements a pipeline architecture consisting of norm category classification, violation detection, impact assessment, remediation generation, and justification generation modules. The system uses ChatGPT for norm-related tasks, ChatYuan with Pfeiffer adapter for remediation generation, and RoBERTa-zh for impact classification. The architecture includes input/output modules (mobile and mixed-reality applications), message middleware, ASR, MT, and a main engine implemented as a finite-state machine. The system was trained and evaluated on the SocialDial dataset with additional human-written dialogues, using BLEU, ROUGE-L, MAUVE, BERT-Score, and human preference ratings as evaluation metrics.

## Key Results
- Successfully mitigated high-impact norm violations in approximately 48% of cases
- Average response time was 1.5 seconds without remediation and 6.7 seconds with low-impact remediation
- User study conducted with 40 dialogues demonstrating system effectiveness

## Architecture Onboarding

- Component map: Input/Output modules (mobile application, mixed-reality application) -> Message middleware -> ASR -> MT -> Core modules (norm category classifier, norm violation classifier, norm violation impact classifier, remediation generator, justification generator) -> Main engine (finite-state machine)
- Critical path: The critical path for the SADAS system involves the following steps: speech recognition, machine translation, norm violation detection, impact assessment, remediation generation, and message delivery to the receiver.
- Design tradeoffs:
  - Accuracy vs. speed: The system must balance the need for accurate norm violation detection and remediation with the requirement for real-time processing and low response times.
  - Automation vs. user control: The system must decide when to automatically handle low-impact norm violations and when to involve the sender in the decision-making process for high-impact violations.
  - Complexity vs. usability: The system must strike a balance between providing advanced features and maintaining a user-friendly interface for non-technical users.
- Failure signatures:
  - High false positive rate in norm violation detection
  - Inaccurate impact assessment leading to inappropriate remediation actions
  - Slow response times or system crashes due to high computational demands
  - User confusion or dissatisfaction with the system's remediation messages and justifications
- First 3 experiments:
  1. Evaluate the accuracy of the norm category classification and violation detection modules on a test set of annotated conversations.
  2. Assess the performance of the norm violation impact classifier in distinguishing between low-impact and high-impact violations.
  3. Conduct a user study to evaluate the effectiveness of the system in mitigating norm violations and the user satisfaction with the remediation messages and justifications.

## Limitations
- The 48% success rate for high-impact violation mitigation indicates the system fails to address norm violations in over half of cases, suggesting significant gaps in either detection accuracy or remediation effectiveness
- Limited evaluation scope - the system was tested only on Chinese-English conversations in one cultural context, raising questions about generalizability to other language pairs and cultural norms
- User study sample size of 40 dialogues provides limited statistical power for drawing robust conclusions about real-world performance

## Confidence
- High confidence in the system architecture and technical implementation details, as these are clearly specified with concrete model choices and evaluation metrics
- Medium confidence in the reported performance metrics (48% mitigation rate, 1.5s/6.7s response times) due to limited evaluation scope and lack of comparison with baseline approaches
- Low confidence in the system's generalizability and real-world effectiveness, as the evaluation did not include diverse language pairs, cultural contexts, or long-term user studies

## Next Checks
1. Cross-cultural validation: Test the system on diverse language pairs (e.g., English-Spanish, Japanese-Korean) and cultural contexts to assess generalizability of norm detection and remediation capabilities
2. A/B testing with baselines: Compare SADAS performance against simpler approaches (rule-based systems, human mediators) to establish whether the complex pipeline architecture provides meaningful advantages
3. Long-term user study: Conduct a 3-month deployment study with regular users to evaluate system adoption, user satisfaction, and whether the system actually reduces cross-cultural communication failures in real-world settings
