# MGCBS: An Optimal and Efficient Algorithm for Solving Multi-Goal Multi-Agent Path Finding Problem

## Quick Facts
- arXiv ID: 2404.19518
- Source URL: https://arxiv.org/abs/2404.19518
- Reference count: 28
- Primary result: Proposes MGCBS algorithm achieving up to 7x speedup with 65-99% success rates on MG-MAPF

## Executive Summary
This paper addresses the multi-goal multi-agent pathfinding (MG-MAPF) problem where each agent must visit multiple pre-assigned goals without conflicts. The authors identify limitations in existing decoupled methods that cannot guarantee optimal solutions. They propose MGCBS, a novel algorithm that uses goal safe interval visiting order search instead of traditional decoupling approaches, achieving both optimality and significant computational efficiency improvements.

The key innovation is the Time-Interval-Space Forest (TIS Forest) data structure, which efficiently maintains shortest paths to goal safe intervals from any start vertex/time, eliminating redundant calculations. Theoretical analysis proves MGCBS is both optimal and complete, while experiments demonstrate consistent optimal solution finding with up to 7x speedup over state-of-the-art methods across various grid maps.

## Method Summary
MGCBS solves the MG-MAPF problem through a fundamentally different approach than existing methods. Rather than decoupling goal vertex visiting order from single-agent pathfinding (DVS), MGCBS uses goal safe interval visiting order search (DSS). The algorithm introduces a Time-Interval-Space Forest (TIS Forest) data structure that maintains shortest paths to goal safe intervals from any start vertex/time, significantly reducing redundant calculations. This approach enables the algorithm to find optimal solutions while maintaining computational efficiency. The method is proven both optimal and complete through theoretical analysis, and is validated through experiments on various grid maps with different agent and goal counts.

## Key Results
- Achieves up to 7x speedup compared to state-of-the-art methods
- Success rates of 65-99% compared to 0-100% for baseline methods across different agent/goal counts
- Consistently finds optimal solutions on tested grid maps
- Maintains completeness while improving computational efficiency

## Why This Works (Mechanism)
The MGCBS algorithm works by fundamentally changing the search strategy from goal vertex visiting order to goal safe interval visiting order. This shift allows the algorithm to consider temporal constraints more naturally during pathfinding. The TIS Forest data structure enables efficient maintenance of shortest paths to goal safe intervals from any start position and time, eliminating redundant path calculations that plague other methods. By integrating temporal safety considerations directly into the search process rather than as a post-processing step, MGCBS can find optimal solutions more efficiently while guaranteeing completeness.

## Architecture Onboarding

Component Map:
Start State -> Goal Safe Interval Search -> TIS Forest Maintenance -> Path Construction -> Conflict Resolution

Critical Path:
The critical path begins with identifying start states for each agent, followed by goal safe interval visiting order search using the DSS approach. The TIS Forest is maintained throughout to provide efficient access to shortest paths. Path construction occurs incrementally while checking for conflicts, with resolution mechanisms ensuring all agents can reach their goals safely.

Design Tradeoffs:
The primary tradeoff is between memory usage and computational efficiency. The TIS Forest requires additional memory to store path information, but this investment pays off through reduced redundant calculations. The algorithm sacrifices some memory efficiency to achieve significant speedups and guaranteed optimality. Additionally, the focus on grid-based environments limits generalizability but enables the specific optimizations that make MGCBS effective.

Failure Signatures:
Common failure modes include memory exhaustion when handling very large instances with many agents and goals, and potential performance degradation on non-grid domains where the TIS Forest structure is less effective. The algorithm may also struggle with highly dynamic environments where goals or obstacles change during execution, as it assumes static goal configurations.
