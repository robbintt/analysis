# ERASOR++: Height Coding Plus Egocentric Ratio Based Dynamic Object Removal for Static Point Cloud Mapping

## Quick Facts
- arXiv ID: 2403.05019
- Source URL: https://arxiv.org/abs/2403.05019
- Reference count: 23
- Primary result: ERASOR++ improves dynamic object removal in 3D point clouds using Height Coding Descriptor and enhanced tests, achieving higher PR and F1 score than ERASOR.

## Executive Summary
This paper presents ERASOR++, an enhanced method for removing dynamic objects from 3D point cloud maps. Building on the previous ERASOR approach, ERASOR++ introduces a Height Coding Descriptor that combines height difference and sparse middle layer encoding, along with additional tests including Height Stack Test, Ground Layer Test, and Surrounding Points Test. The method aims to improve the accuracy of static map generation for autonomous systems by more precisely identifying and removing dynamic objects while preserving static ones.

## Method Summary
ERASOR++ processes 3D LiDAR point clouds by dividing them into bins, computing a Height Coding Descriptor (HCD) for each bin, and applying a series of tests to identify dynamic objects. The method includes Ground Layer Test (GLT) for ground detection, Height Stack Test (HST) for overlap analysis, Surrounding Points Test (SPT) for neighbor checking, and Region-wise Ground Plane Fitting (R-GPF) refinement. The approach is evaluated on the SemanticKITTI dataset, showing improved performance over the original ERASOR method in terms of Preservation Rate, Rejection Rate, and F1 score.

## Key Results
- ERASOR++ achieves higher Preservation Rate (PR) and F1 score compared to ERASOR on SemanticKITTI dataset.
- In sequence 05, ERASOR++ achieves PR of 96.5% and F1 score of 0.971, compared to ERASOR's PR of 87.0% and F1 score of 0.921.
- The method demonstrates faster execution time in some cases due to reduced iterations of the computationally expensive R-GPF step.

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Height Coding Descriptor (HCD) combines height difference and sparse middle layer encoding to capture both extreme and intermediate vertical structure information, reducing information loss compared to simple height-difference-only descriptors.
- Mechanism: The descriptor stores max-min height difference (extreme values) and uses bit encoding to represent occupancy in intermediate layers. This preserves vertical profile shape while keeping storage and computation efficient.
- Core assumption: The vertical distribution of points in a bin is discriminative enough to separate dynamic from static regions when combined with other tests.
- Evidence anchors:
  - [abstract] Introduces HCD combining height difference and height layer information.
  - [section] Defines DD and DE parts, showing encoding of sparse middle values.
  - [corpus] Related works focus on height-based filtering but none combine extreme and sparse middle values explicitly.
- Break condition: If point density is too low or uniform, intermediate layer encoding becomes uninformative, reducing discriminative power.

### Mechanism 2
- Claim: Height Stack Test (HST) uses overlap layer positions to avoid bad removal in areas with blocked ground or vegetation.
- Mechanism: HST computes bitwise AND of current and map descriptors, then masks ground layers, yielding overlap count above ground. High overlap in upper layers indicates static, low overlap suggests dynamic.
- Core assumption: Static objects leave consistent height traces across scans; dynamic objects do not.
- Evidence anchors:

  - [abstract] HST "leverages the description of points in the middle height area in HCD".
  - [corpus] No direct corpus evidence; assumption based on ERASOR baseline behavior.
- Break condition: If objects move vertically within the scan (e.g., tall vegetation swaying), overlap patterns may be misleading.

### Mechanism 3
- Claim: Ground Layer Test (GLT) provides an absolute height reference to improve HST reliability.
- Mechanism: GLT scans nearest bins, identifies layer with most ground points, encodes all layers below it as ground, which HST uses to mask irrelevant overlaps.
- Core assumption: Ground points are reliably detectable near the sensor and stable across scans.
- Evidence anchors:
  - [section] GLT iteration from near bins, selects layer with most ground points, defines Layer(Ground).
  - [abstract] GLT "serves to complement the algorithm and address specific challenges".
  - [corpus] Weak evidence; GLT is novel in this paper.
- Break condition: If ground is occluded or uneven (e.g., stairs, ramps), ground layer detection fails and HST masking becomes incorrect.

## Architecture Onboarding

- Component map: Input Point Cloud -> Voxelization -> HCD Computation -> GLT -> HST -> SPT -> R-GPF -> Static Point Cloud Map
- Critical path: Voxelization -> HCD -> GLT -> HST -> SPT -> R-GPF
- Design tradeoffs:
  - HCD vs. full point cloud: Faster but coarser; balances accuracy vs. speed.
  - HST vs. simple ratio test: More robust to partial occlusions but requires GLT.
  - SPT vs. none: Reduces isolated false positives at cost of extra neighborhood search.
- Failure signatures:
  - Low PR with high RR: Over-aggressive removal, likely HST or SPT over-triggering.
  - High PR but low RR: Conservative, missing dynamic objects; check GLT ground detection.
  - Slow runtime: Excessive R-GPF calls, possibly from missing SPT filtering.
- First 3 experiments:
  1. Run ERASOR++ on sequence 05 with default parameters; record PR, RR, time.
  2. Disable GLT only; compare PR drop to quantify ground layer contribution.
  3. Disable SPT only; measure isolated dynamic bin count increase.

## Limitations
- Exact threshold values for the Height Stack Test and other algorithm constants are not specified, potentially affecting reproducibility and performance consistency.
- Implementation details of the R-GPF method used in the baseline comparison are incomplete, limiting accurate benchmarking.
- The method's performance in highly unstructured environments with complex vertical features (e.g., urban canyons, dense vegetation) remains unverified.

## Confidence
- **High confidence** in the core mechanism of Height Coding Descriptor and its role in improving dynamic object detection.
- **Medium confidence** in the overall performance improvements, given the strong experimental results on SemanticKITTI but limited environmental diversity.
- **Low confidence** in the generalizability to other LiDAR sensors and point cloud datasets due to unspecified parameter values and lack of cross-dataset validation.
