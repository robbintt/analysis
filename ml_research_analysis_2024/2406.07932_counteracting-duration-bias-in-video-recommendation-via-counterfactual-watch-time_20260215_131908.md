# Counteracting Duration Bias in Video Recommendation via Counterfactual Watch Time

## Quick Facts
- arXiv ID: 2406.07932
- Source URL: https://arxiv.org/abs/2406.07932
- Reference count: 40
- Addresses duration bias in video recommendation by modeling counterfactual watch time

## Executive Summary
This paper tackles a fundamental challenge in video recommendation systems: duration bias, where observed watch time is truncated by video length, leading to inaccurate user interest estimation. The authors introduce the concept of counterfactual watch time (CWT) - the potential watch time a user would spend if video duration were unlimited - and develop a Counterfactual Watch Model (CWM) that models user behavior from an economic perspective. By treating CWT as the time users get maximum benefit from videos, the model uses a cost-based transform function to convert CWT into user interest estimates. The approach is validated across three real-world datasets and shows significant improvements over existing methods, with online A/B testing demonstrating a 2.9% increase in mean watch time in production systems.

## Method Summary
The Counterfactual Watch Model (CWM) addresses duration bias by modeling user watching behavior through an economic lens, where counterfactual watch time (CWT) represents the time users would spend to achieve maximum benefit from videos. The model employs a cost-based transform function that converts CWT into user interest estimates, trained using a counterfactual likelihood function that specifically accounts for truncation effects. This framework effectively separates the inherent user interest from the artificial constraint of video duration, allowing for more accurate recommendations that aren't biased toward longer videos simply because they provide more opportunities for watch time accumulation.

## Key Results
- CWM outperforms existing methods with 6-8% XAUC improvement in watch time prediction
- Significant relevance ranking improvements with 2-3% AUC enhancement
- Online A/B testing shows 2.9% increase in mean watch time in production systems
- Particularly effective for shorter videos where duration bias is most severe

## Why This Works (Mechanism)
The model works by addressing the fundamental issue that observed watch time is a truncated version of true user interest. By introducing counterfactual watch time as a latent variable representing potential engagement, CWM can estimate what users would watch if unconstrained by video length. The economic framing treats watching as an investment where users allocate time until marginal benefit equals marginal cost. The cost-based transform function bridges this economic model to actual user interest estimation, while the counterfactual likelihood function properly handles the truncation during training, ensuring the model learns from the underlying patterns rather than the artificial constraints.

- Economic modeling of user behavior: Treats time allocation as investment decisions; Quick check: Are the marginal benefit and cost assumptions empirically validated?
- Truncation-aware training: Accounts for censored observations in likelihood function; Quick check: How does the model perform when truncation is severe?
- Cost-based transform functions: Converts latent economic variables to observable interest signals; Quick check: Is the transform function robust to different video genres?

## Architecture Onboarding
Component map: User embedding -> CWT estimation -> Cost-based transform -> Interest prediction -> Counterfactual likelihood loss

Critical path: The core inference flow moves from user and video embeddings through CWT estimation to final interest prediction, with the cost-based transform serving as the key bridge between economic theory and practical recommendation.

Design tradeoffs: The economic framing provides theoretical grounding but adds complexity compared to simpler heuristic approaches. The cost-based transform introduces additional parameters that must be learned but enables more nuanced modeling of user behavior.

Failure signatures: Poor performance on very short videos might indicate issues with the CWT estimation; degradation in diversity metrics could suggest over-reliance on economic similarity; failure to converge during training might indicate problems with the counterfactual likelihood function.

## Limitations
- Generalizability across different platform types and user demographics remains uncertain
- The economic framing may oversimplify complex viewing motivations that vary contextually
- Additional modeling complexity from the cost-based transform function without clear justification for its superiority
