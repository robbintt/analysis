# Transient Multi-Agent Path Finding for Lifelong Navigation in Dense Environments

## Quick Facts
- arXiv ID: 2412.04256
- Source URL: https://arxiv.org/abs/2412.04256
- Reference count: 40
- Primary result: TMAPF algorithms achieve up to 25% higher throughput than MAPF algorithms when targets are limited to a small number of locations

## Executive Summary
This paper addresses a fundamental mismatch between Multi-Agent Path Finding (MAPF) and Lifelong Multi-Agent Path Finding (LMAPF), where MAPF requires all agents to reach their targets simultaneously but LMAPF does not. The authors propose a new problem variant called Transient Multi-Agent Path Finding (TMAPF) where agents must visit their targets but need not all be there at once. They adapt several MAPF algorithms (CBS, PrP, LNS) to solve TMAPF by extending the A* search state to track whether an agent has visited its target. Experiments show that TMAPF algorithms achieve significantly higher throughput in scenarios where targets are limited to a small number of locations, with improvements of up to 25% compared to traditional MAPF approaches.

## Method Summary
The authors propose Transient MAPF (TMAPF) as a new problem variant where agents must visit their targets but not necessarily simultaneously. They adapt existing MAPF algorithms (PrP, CBS, LNS) by extending the A* search state to include a Boolean flag indicating whether the agent has visited its target. These TMAPF algorithms are then evaluated within the RHCR framework for LMAPF problems, with a planning horizon of 10 and 5-second planning budget. The key innovation is modifying the goal condition from requiring all agents to be at their targets simultaneously to requiring each agent to visit its target at some point along its path.

## Key Results
- TMAPF algorithms achieve up to 25% higher throughput than MAPF algorithms when targets are limited to a small number of locations
- In scenarios with uniform target distribution, MAPF and TMAPF algorithms perform similarly
- TMAPF algorithms successfully solve pathological cases that cause MAPF algorithms to fail due to position swapping impossibilities

## Why This Works (Mechanism)

### Mechanism 1
- Claim: TMAPF allows agents to reach targets independently without requiring simultaneous arrival, which is more flexible than MAPF for LMAPF scenarios.
- Mechanism: By modifying the goal condition to require only that each agent visits its target at some point (rather than all agents arriving simultaneously), TMAPF removes the restrictive requirement that all agents must be at their targets at the same time. This enables agents to continue moving after visiting their target, avoiding deadlocks that occur in MAPF.
- Core assumption: The key assumption is that the order of target completion does not matter for the overall system performance as long as each agent eventually visits its target.
- Evidence anchors:
  - [abstract]: "TMAPF, where agents must visit their targets but need not all be there at once."
  - [section]: "In TMAPF, the objective is to find conflict-free paths for the agents such that each agent visits its target at some point along its path, but there is no need to reach a configuration in which all agents are at their targets at the same time."
- Break condition: This mechanism breaks if the system requires all agents to coordinate their target arrival times for some reason, such as synchronized task completion.

### Mechanism 2
- Claim: Adapting A* search to track whether an agent has visited its target enables efficient TMAPF pathfinding.
- Mechanism: By extending the A* state representation to include a Boolean flag indicating whether the agent has visited its target, the algorithm can efficiently search for paths that satisfy the TMAPF condition. This allows the search to distinguish between states that have and haven't visited the target, preventing incorrect pruning of valid solutions.
- Core assumption: The assumption is that tracking this flag does not significantly increase the state space complexity beyond what is manageable.
- Evidence anchors:
  - [section]: "We propose to extend the state definition in A* to include a Boolean flag indicating whether or not any of the state's ancestors visited the agent's target (⟨v, x, b⟩)."
- Break condition: This mechanism breaks if the additional state dimension causes the search space to explode in size, making the algorithm intractable for larger problems.

### Mechanism 3
- Claim: Limited planning horizon in RHCR framework can mask the benefits of TMAPF algorithms.
- Mechanism: When using a limited planning horizon, agents only consider conflicts within a fixed time window. This can allow agents to find paths to their targets without needing to coordinate with all other agents simultaneously, partially mitigating the MAPF-LMAPF mismatch even without TMAPF.
- Core assumption: The assumption is that the limited horizon is sufficient to find reasonable paths without requiring global coordination.
- Evidence anchors:
  - [section]: "This may be due to the limited planning horizon, which, as mentioned above, can mitigate the effect of the LMAPF-MAPF mismatch."
- Break condition: This mechanism breaks when the planning horizon is too short to find feasible paths, or when global coordination becomes necessary for optimal performance.

## Limitations
- Performance gains of TMAPF are only demonstrated in scenarios with limited target locations (up to 25% improvement)
- Computational overhead of tracking target visitation states is not thoroughly analyzed
- Limited evaluation of TMAPF performance in general LMAPF scenarios with uniformly distributed targets

## Confidence

- High confidence in the mechanism by which TMAPF removes the simultaneous arrival requirement and enables more flexible routing
- Medium confidence in the experimental results showing throughput improvements, as the evaluation focuses on specific scenarios with concentrated targets
- Low confidence in the generalizability of results to all LMAPF scenarios, given the limited scope of experimental conditions
