# Entertainment chatbot for the digital inclusion of elderly people without abstraction capabilities

## Quick Facts
- arXiv ID: 2404.01327
- Source URL: https://arxiv.org/abs/2404.01327
- Reference count: 40
- Primary result: High user satisfaction (average score of 4 out of 5) with improved abstraction capabilities for elderly users

## Executive Summary
The study presents EBER, a chatbot system designed to reduce the digital divide for elderly users by functioning as an "intelligent radio" that reads news interspersed with light dialogues adapted to the user's mood. The system combines Artificial Intelligence Modelling Language (AIML) for chatbot personality, Natural Language Generation (NLG) for varied dialogue, and Sentiment Analysis (SA) to adapt responses to user emotions. EBER uses facial recognition for activation and voice commands for interaction, making it accessible for elderly users. The chatbot reads news items and engages users in short dialogues, extracting keywords from user responses to improve content recommendations. Experimental results with 31 elderly users showed high satisfaction (average score of 4 out of 5) and demonstrated that the system improved users' abstraction capabilities.

## Method Summary
EBER combines AIML for chatbot personality with NLG for varied dialogue and SA to adapt responses to user emotions. The system uses facial recognition for activation and voice commands for interaction, making it accessible for elderly users. News items are sourced from the RTVE API, and the NLG module uses aLexiS lexicon for syntax structure inference, lexicalisation, and morphological inflections to generate coherent Spanish sentences. The SA module employs a Decision Tree classifier trained on a dataset of 676 negative, 1,334 neutral, and 918 positive clauses, using character and word n-grams, and word dictionaries as features. The system extracts keywords from user responses to improve content recommendations and integrates semantic knowledge for personalized news searches.

## Key Results
- High user satisfaction with an average score of 4 out of 5
- Demonstrated improvement in users' abstraction capabilities
- Effective news personalization and search functionality

## Why This Works (Mechanism)
The system leverages familiar media formats (radio-style news reading) combined with adaptive AI technologies to create an accessible interface for elderly users. By using voice commands and facial recognition, it removes technological barriers while the SA module ensures emotionally appropriate responses. The NLG component provides varied dialogue to maintain engagement, and the semantic knowledge integration enables personalized content recommendations even for users with limited technological skills.

- **Natural Language Generation (NLG)**: Why needed - to produce varied, natural-sounding dialogue; Quick check - ensure generated sentences maintain grammatical coherence in Spanish
- **Sentiment Analysis (SA)**: Why needed - to adapt responses to user emotions; Quick check - validate classification accuracy across positive, neutral, and negative sentiment categories
- **Facial Recognition Activation**: Why needed - to provide accessible system activation for elderly users; Quick check - confirm recognition works under various lighting conditions and user positions
- **Semantic Knowledge Integration**: Why needed - to enable personalized news searches based on user interests; Quick check - verify NGD values correctly identify relevant news items
- **Voice Command Interface**: Why needed - to eliminate complex navigation requirements; Quick check - test recognition accuracy across different user accents and speech patterns

## Architecture Onboarding

**Component Map**: User -> Facial Recognition -> Voice Commands -> SA -> AIML -> NLG -> News API -> User Feedback

**Critical Path**: Facial recognition activation → voice command processing → sentiment analysis → personalized response generation → news content delivery → user feedback collection

**Design Tradeoffs**: The system prioritizes accessibility over advanced features, using familiar radio formats rather than complex interfaces. This choice sacrifices some customization options but significantly reduces the learning curve for elderly users. The decision to use AIML templates rather than more sophisticated conversational AI models ensures predictable, controlled interactions that are easier to debug and maintain.

**Failure Signatures**: Poor user engagement indicates issues with SA module accuracy or inappropriate AIML responses. Inefficient news personalization suggests problems with semantic knowledge integration or keyword extraction. System crashes during facial recognition point to camera or processing issues.

**3 First Experiments**:
1. Test facial recognition activation with 10 elderly users under different lighting conditions
2. Validate SA module accuracy by having users read pre-classified emotional statements
3. Measure NLG output coherence by having language experts rate generated Spanish sentences

## Limitations
- Small sample size of 31 elderly users limits generalizability
- Lack of objective measures for abstraction capability improvements
- Missing technical specifications for critical components like AIML configuration

## Confidence

**High Confidence**: The basic system architecture combining AIML, NLG, and SA modules is clearly specified and technically feasible

**Medium Confidence**: User satisfaction results are reliable but may not generalize beyond the specific Spanish elderly population studied

**Low Confidence**: Claims about improved abstraction capabilities lack objective validation measures and detailed assessment methodologies

## Next Checks

1. Conduct a larger-scale study (n > 100) across diverse elderly populations to validate generalization of satisfaction scores and abstraction improvements
2. Implement objective pre/post testing protocols to measure actual improvements in abstraction capabilities using standardized cognitive assessment tools
3. Perform a technical audit of the AIML knowledge base and SA module configurations to ensure reproducibility and identify optimal parameter settings for elderly user engagement
