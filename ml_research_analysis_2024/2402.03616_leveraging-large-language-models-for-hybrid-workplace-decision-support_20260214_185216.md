# Leveraging Large Language Models for Hybrid Workplace Decision Support
## Quick Facts
- arXiv ID: 2402.03616
- Source URL: https://arxiv.org/abs/2402.03616
- Reference count: 40
- LLM-based system for hybrid workplace desk selection that influences workspace choices

## Executive Summary
This study develops an LLM-based decision support system for hybrid workplace desk selection using GPT-4. The system suggests workspaces and provides explanations considering factors like collaborator locations, meeting rooms, and amenities. A user study with 42 participants showed the system effectively influenced workspace choices and was perceived as convenient, though explanations did not significantly impact satisfaction levels. The LLM demonstrated reasoning abilities beyond prompt guidelines, managing trade-offs among workspace resources while following an explore-exploit strategy.

## Method Summary
The system was developed using GPT-4 to generate workspace suggestions and explanations based on multiple factors including collaborator locations, meeting room availability, and amenities. The user study involved 42 participants who interacted with the system over an unspecified duration. Participants were divided into groups that received either explanations or no explanations for the workspace suggestions. The study evaluated system influence on workspace choices, user satisfaction, and perception of convenience.

## Key Results
- System influenced workspace selection choices among participants
- Users perceived the system as convenient and easy to use
- No significant difference in satisfaction between explanation and no-explanation conditions
- LLM demonstrated reasoning abilities beyond prompt guidelines
- Suggestions followed an explore-exploit strategy balancing familiar and new options

## Why This Works (Mechanism)
The system leverages GPT-4's natural language processing capabilities to analyze multiple workspace factors simultaneously and generate contextually appropriate suggestions. The LLM's reasoning abilities allow it to understand complex workplace dynamics and trade-offs between different workspace resources. By incorporating both familiar workspace suggestions and novel options, the system maintains user engagement while promoting workplace flexibility.

## Foundational Learning

   - Why needed: Essential for designing effective workspace recommendation systems

2. LLM reasoning capabilities - Leveraging GPT-4's ability to process multiple factors and generate explanations
   - Why needed: Enables sophisticated decision-making beyond simple rule-based systems
   - Quick check: Does the system consistently produce logical suggestions across different scenarios?

3. Explore-exploit strategy - Balancing between familiar and new workspace options
   - Why needed: Prevents user stagnation while maintaining comfort with known options
- Quick check: Are users actually exploring new workspace options as intended?

## Architecture Onboarding
Component map: User Input -> GPT-4 Processing -> Workspace Suggestion -> Explanation Generation -> User Output

Critical path: User input regarding work needs and preferences flows through the LLM for analysis, which then generates workspace suggestions and optional explanations before presenting recommendations to the user.

Design tradeoffs: The system prioritizes flexibility and user convenience over strict optimization, allowing for human judgment in workspace selection while providing AI assistance.

Failure signatures: System may produce irrelevant suggestions if input data is incomplete or if the LLM misinterprets user needs. Explanation quality may vary based on prompt effectiveness.

## Limitations
- Small sample size of 42 participants limits generalizability
- Study duration and observation period were not specified
- No assessment of long-term impacts on productivity or collaboration outcomes

## Confidence

- Medium confidence: The convenience and ease of use perception
- Low confidence: The reasoning capabilities claim and explore-exploit strategy effectiveness
