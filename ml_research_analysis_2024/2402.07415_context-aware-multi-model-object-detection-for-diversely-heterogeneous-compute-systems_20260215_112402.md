# Context-aware Multi-Model Object Detection for Diversely Heterogeneous Compute Systems

## Quick Facts
- arXiv ID: 2402.07415
- Source URL: https://arxiv.org/abs/2402.07415
- Reference count: 19
- Key outcome: SHIFT achieves up to 7.5x energy improvement and 2.8x latency reduction compared to GPU-based single model approaches

## Executive Summary
This paper presents SHIFT, a context-aware multi-model object detection system that dynamically selects from various DNN-based models based on input stream characteristics and computational constraints. The system uses a graph-based confidence mechanism to predict model accuracy in real-time without running all models on every frame. Tested on NVIDIA Xavier NX SoC and Luxonis OAK-D Lite platforms, SHIFT demonstrates significant improvements in energy efficiency and latency while maintaining competitive accuracy. The approach is particularly suited for autonomous systems where energy and latency are critical constraints.

## Method Summary
SHIFT implements a dynamic model selection framework that analyzes input streams and computational constraints to choose optimal detection models. The system employs a scheduler that considers multi-accelerator execution to optimize energy efficiency while meeting latency requirements. The core innovation is a graph-based confidence mechanism that predicts model accuracy without executing all models on every frame. This allows for real-time adaptation to varying object positions and backgrounds while maintaining computational efficiency.

## Key Results
- Up to 7.5x improvement in energy usage compared to GPU-based single model approaches
- 2.8x improvement in latency while maintaining competitive accuracy
- Effective handling of real-world scenarios with varying object positions and backgrounds

## Why This Works (Mechanism)
SHIFT's effectiveness stems from its ability to match input characteristics with the most appropriate model from a diverse model pool. By analyzing frame features and using predictive confidence scores, the system avoids running computationally expensive models when simpler ones suffice. The multi-accelerator scheduling ensures optimal resource utilization across heterogeneous compute systems, while the dynamic selection adapts to changing scene conditions without sacrificing performance.

## Architecture Onboarding

Component map: Input Stream -> Feature Extractor -> Confidence Predictor -> Model Selector -> Multi-Accelerator Scheduler -> Output

Critical path: The system's critical path involves real-time feature extraction from input frames, confidence prediction for available models, and model selection based on computational constraints and predicted accuracy.

Design tradeoffs: The primary tradeoff is between computational overhead of the selection mechanism versus the benefits of dynamic model selection. The system must balance accuracy requirements with energy and latency constraints while maintaining real-time performance.

Failure signatures: System failures typically manifest as either degraded accuracy when the confidence predictor misestimates model performance, or increased latency when the scheduler cannot effectively distribute work across available accelerators.

## Limitations

- Performance improvements appear to be platform-specific and may not generalize across different heterogeneous compute systems

- Limited discussion of how the system scales when handling multiple object classes simultaneously
