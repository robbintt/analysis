# Artificial Intelligence Based Navigation in Quasi Structured Environment

## Quick Facts
- arXiv ID: 2407.17508
- Source URL: https://arxiv.org/abs/2407.17508
- Reference count: 40
- Primary result: Modified Floyd-Warshall algorithm with ACO integration reduces time complexity to 0.001 seconds compared to 0.02 seconds for general Floyd-Warshall on quasi-structured points

## Executive Summary
This paper addresses challenges in public transportation route planning by proposing an AI-based navigation model for quasi-structured environments. The authors identify key issues with traditional route planning methods including high implementation costs, infrastructure requirements, and resistance to change. Through comparative analysis of path generation and optimization algorithms, they develop a hybrid approach combining Floyd-Warshall with Ant Colony Optimization (ACO) that demonstrates superior performance in terms of time complexity when applied to randomly structured points within boundaries.

## Method Summary
The research involves a comparative analysis of various path generation algorithms (Floyd-Warshall, Bellman-Ford, Johnson) and nature-inspired optimization algorithms (ACO, PSO, GWO) to identify the most suitable candidates for quasi-structured environment navigation. Based on this analysis, the authors select Floyd-Warshall and ACO as optimal choices. The core contribution is a modified Floyd-Warshall algorithm integrated with ACO, designed to optimize route planning by considering physical, social, economic, technical, and time constraints. The algorithm is tested on randomly structured points within boundaries, demonstrating improved performance with reduced time complexity compared to traditional approaches.

## Key Results
- Proposed hybrid algorithm achieves time complexity of 0.001 seconds versus 0.02 seconds for general Floyd-Warshall
- Floyd-Warshall provides exact all-pairs shortest paths while ACO optimizes path selection in quasi-structured domains
- The integration enables consideration of multiple constraints (physical, social, economic, technical, time) beyond pure distance optimization

## Architecture Onboarding

- Component map: Point generation module -> Floyd-Warshall core -> ACO optimization layer -> Constraint integration module -> Complexity measurement module

- Critical path: 1. Generate quasi-structured points 2. Apply Floyd-Warshall to establish baseline shortest paths 3. Initialize ACO with pheromone trails based on Floyd-Warshall results 4. Iteratively update pheromones incorporating constraint weights 5. Output optimized routes with complexity metrics

- Design tradeoffs:
  - Pre-computation vs. real-time adaptation: Floyd-Warshall provides exact results but may be computationally expensive for dynamic environments
  - Pheromone sensitivity vs. convergence speed: Higher pheromone sensitivity allows better constraint adaptation but may slow convergence
  - Constraint weight calibration: Balancing multiple constraint types requires careful parameter tuning

- Failure signatures:
  - Slow convergence: Indicates pheromone evaporation rate too high or constraint weights too conflicting
  - Suboptimal paths: Suggests insufficient ACO iterations or poor constraint weight calibration
  - Memory overflow: Occurs when graph size exceeds Floyd-Warshall's O(|V|²) storage requirements

- First 3 experiments:

  2. Test ACO convergence with different constraint weight combinations on the same point set
