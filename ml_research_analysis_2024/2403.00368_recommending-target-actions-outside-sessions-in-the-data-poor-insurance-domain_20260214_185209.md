# Recommending Target Actions Outside Sessions in the Data-poor Insurance Domain

## Quick Facts
- arXiv ID: 2403.00368
- Source URL: https://arxiv.org/abs/2403.00368

## Executive Summary
This paper addresses the challenge of providing personalized recommendations in the insurance domain, where limited user feedback and small item catalogs create data scarcity. The authors propose using recurrent neural networks (RNNs) with various loss functions and architectures to learn from multiple user sessions and different types of user actions. The models predict target actions that occur outside current sessions, learning patterns across time to improve recommendation quality. The approach demonstrates significant improvements over state-of-the-art baselines on a real-world insurance dataset, with the added benefit of maintaining fairness across demographic attributes.

## Method Summary
The authors develop RNN-based models that learn from sequences of user actions across multiple sessions. The models incorporate various loss functions and architectural designs to handle data scarcity in the insurance domain. They learn patterns from different action types and predict future target actions that occur outside the current session. The approach leverages session-based learning to capture temporal dependencies in user behavior. Demographic information can be incorporated to further enhance recommendations. The models are trained on real-world insurance data and evaluated using standard recommendation metrics.

## Key Results
- Proposed RNN models achieve hit rate of 0.8385 at cutoff 3
- Outperforms best baseline (0.8132) on real-world insurance dataset
- Combining models with demographic data further improves performance
- Models demonstrate fairness with respect to age, gender, and income

## Why This Works (Mechanism)
The RNN architecture effectively captures temporal dependencies in user behavior across multiple sessions, which is crucial in insurance where purchase decisions often span extended periods. By learning from various action types and incorporating demographic information, the models build richer user representations despite limited data. The session-based approach allows the model to understand context and predict actions that typically occur outside immediate sessions, addressing the unique challenge of insurance recommendations where conversion often happens after multiple touchpoints.
