# A Preference-oriented Diversity Model Based on Mutual-information in Re-ranking for E-commerce Search

## Quick Facts
- arXiv ID: 2405.15521
- Source URL: https://arxiv.org/abs/2405.15521
- Reference count: 21
- Key result: PODM-MI achieved 0.66% increase in user conversion rate and 0.26% increase in diversity when deployed on JD's e-commerce search platform

## Executive Summary
This paper introduces a Preference-oriented Diversity Model Based on Mutual-information (PODM-MI) for e-commerce re-ranking that addresses the challenge of balancing accuracy and diversity in search results. The model represents user preferences as multi-dimensional Gaussian distributions and maximizes mutual information between these preferences and candidate items to enhance their correlation. By incorporating both a preference-oriented network (PON) and a self-adaptive model (SAM), PODM-MI effectively captures user diversity preferences while maintaining relevance. The approach has been successfully deployed on JD's platform, demonstrating significant improvements over state-of-the-art models in both offline metrics (AUC and entropy) and online performance metrics.

## Method Summary
PODM-MI employs a two-component architecture consisting of a preference-oriented network (PON) and a self-adaptive model (SAM). The PON models user preferences using multi-dimensional Gaussian distributions, capturing the diversity in user interests through mean vectors and covariance matrices. The SAM dynamically adjusts the balance between relevance and diversity during re-ranking. The core mechanism maximizes mutual information between the preference distributions and candidate items, ensuring that selected items are both relevant and diverse. The model uses a two-stage optimization process: first learning preference distributions from user interaction data, then optimizing the re-ranking process to maximize the mutual information objective.

## Key Results
- Achieved 0.66% increase in user conversion rate when deployed on JD's e-commerce search platform
- Demonstrated 0.26% increase in diversity metrics compared to baseline models
- Showed significant improvements in offline AUC and entropy metrics over state-of-the-art re-ranking models

## Why This Works (Mechanism)
PODM-MI works by explicitly modeling the uncertainty and diversity in user preferences through Gaussian distributions, which allows the model to capture both the central tendency and the spread of user interests. By maximizing mutual information between these preference distributions and candidate items, the model ensures that selected items are not only relevant but also cover the breadth of user interests. The self-adaptive component allows the model to dynamically adjust the trade-off between relevance and diversity based on contextual factors and user behavior patterns. This approach addresses the cold-start problem for diverse preferences and prevents the model from collapsing into a single dominant preference.

## Architecture Onboarding

**Component map**: User interaction data -> Preference-oriented Network (PON) -> Self-adaptive Model (SAM) -> Re-ranked results

**Critical path**: User interaction data → PON (Gaussian preference distribution learning) → SAM (dynamic balancing) → Re-ranking optimization → Output ranked list

**Design tradeoffs**: The model trades computational complexity for improved diversity through the Gaussian distribution representation and mutual information maximization. This approach requires more sophisticated training but provides better handling of diverse user preferences compared to simpler diversity-aware re-ranking methods.

**Failure signatures**: The model may fail when user preference distributions are multi-modal or highly skewed, as Gaussian assumptions may not capture such patterns well. Additionally, sparse interaction data could lead to poorly estimated preference distributions, resulting in suboptimal re-ranking. The mutual information maximization might also be sensitive to noise in preference estimation.

## Limitations
- Gaussian distribution assumptions may not capture complex, multi-modal user preference patterns
- Model performance could be sensitive to data quality and quantity, particularly for users with sparse interaction histories
- The practical significance of improvements (0.66% conversion, 0.26% diversity) may vary across different e-commerce contexts and user segments
