# Sparse Color-Code Net: Real-Time RGB-Based 6D Object Pose Estimation on Edge Devices

## Quick Facts
- arXiv ID: 2406.02977
- Source URL: https://arxiv.org/abs/2406.02977
- Authors: Xingjian Yang; Zhitao Yu; Ashis G. Banerjee
- Reference count: 40
- Achieves 19 FPS on LINEMOD and 6 FPS on Occlusion LINEMOD on NVIDIA Jetson AGX Xavier

## Executive Summary
Sparse Color-Code Net (SCCN) introduces a 3-stage pipeline for real-time 6D object pose estimation on edge devices. The method leverages sparse object geometry features to accelerate PnP computation and introduces a novel pixel-level symmetry representation to handle symmetric objects. By focusing computation on informative regions rather than dense pixel predictions, SCCN achieves significant speed improvements while maintaining high accuracy, making it suitable for responsive pose estimation on resource-constrained hardware.

## Method Summary
SCCN employs a 3-stage pipeline: object segmentation using a modified UNet with depthwise separable convolutions, color-code estimation with anisotropic mapping, and symmetry mask prediction. The pipeline extracts object contours using Sobel filters to identify surface features, then uses only these informative pixels for PnP estimation rather than full dense predictions. This reduces computational complexity while maintaining accuracy. The method also introduces anisotropic color-code mapping to improve feature distinguishability for objects with imbalanced geometry, and a symmetry mask to resolve ambiguities in symmetric objects.

## Key Results
- Achieves 19 FPS on LINEMOD dataset and 6 FPS on Occlusion LINEMOD dataset on NVIDIA Jetson AGX Xavier
- Maintains high accuracy while significantly improving speed over existing methods
- Successfully handles symmetric objects through pixel-level symmetry representation
- Balances speed and precision by focusing computation on informative regions

## Why This Works (Mechanism)

### Mechanism 1
Sparse color-code predictions accelerate PnP computation by reducing input point set size while maintaining accuracy. The pipeline extracts object contours using Sobel filters to identify surface features. Only these informative pixels are used for PnP estimation instead of the full dense color-code prediction. This reduces the number of 2D-3D correspondences fed to RANSAC, lowering computational complexity from O(n²) to O(k²) where k << n. Core assumption: Contour pixels provide sufficient geometric information for accurate pose estimation. Break condition: If object lacks sufficient surface features (smooth/reflective surfaces), contour extraction fails and sparse selection provides inadequate geometric constraints.

### Mechanism 2
Anisotropic color-code representation improves feature distinguishability for objects with imbalanced geometry. Standard color-codes normalize objects to 1×1×1 cubes, compressing dimensions with smaller ranges. Anisotropic mapping redistributes RGB gamut to span full XYZ ranges of the object, intensifying inter-region color differences and helping networks resolve correspondences between distinct surface features. Core assumption: Networks struggle with compressed color variations in standard mapping. Break condition: For objects with nearly equal dimensions, anisotropic mapping provides minimal benefit while potentially introducing unnecessary complexity.

### Mechanism 3
Symmetry mask prediction resolves ambiguities in symmetric objects by separating symmetric regions with distinct encodings. The symmetry mask labels reflective symmetric areas with 1 and -1 values. During decoding, symmetric parts have their coordinates negated along the symmetry axis. This ensures consistent pose estimation regardless of which symmetric region is observed. Core assumption: Symmetric regions need separate encodings for unambiguous pose estimation. Break condition: For asymmetric objects, symmetry mask prediction wastes network capacity without providing benefits.

## Architecture Onboarding

- Component map: RGB image -> Sobel filter contour extraction -> Object segmentation UNet -> Bounding box generation -> Color-code estimation UNet -> Symmetry mask prediction -> Sparse point selection -> PnP estimation
- Critical path: RGB image -> Sobel filtering -> Object segmentation -> Color-code estimation -> Sparse point selection -> PnP estimation
- Design tradeoffs:
  - Accuracy vs. speed: Using sparse contours sacrifices some information but gains significant speed
  - Model complexity vs. efficiency: Modified UNets with depthwise separable convolutions reduce parameters while maintaining capacity
  - Symmetry handling: Additional symmetry mask prediction increases model size but enables handling symmetric objects
- Failure signatures:
  - Low FPS: PnP computation becoming bottleneck (check contour sparsity level)
  - Poor accuracy: Insufficient contour features or color-code prediction errors (check segmentation quality)
  - Symmetric object failures: Symmetry mask not properly separating regions (check symmetry mask loss training)
- First 3 experiments:
  1. Baseline test: Run SCCN on LINEMOD with standard color-codes, measure FPS and ADD accuracy to establish performance baseline
  2. Contour sparsity test: Vary the subsampling rate of contour pixels and measure impact on FPS and accuracy to find optimal tradeoff
  3. Symmetry test: Evaluate on symmetric objects (eggbox, glue) with and without symmetry mask prediction to quantify benefit of symmetry handling

## Limitations
- Lacks quantitative ablation studies showing exact speedup from contour-based sparsity versus dense predictions
- No empirical comparison demonstrating anisotropic color-code advantages over standard color-codes
- Assumes all symmetric objects have clear reflective symmetry, may not generalize to rotational or complex symmetries

## Confidence
- High confidence: Real-time performance claims on NVIDIA Jetson AGX Xavier (19 FPS on LINEMOD)
- Medium confidence: Accuracy metrics (ADD, AUC) on standard datasets
- Low confidence: Claims about anisotropic color-code benefits and symmetry mask effectiveness without comparative ablation studies
