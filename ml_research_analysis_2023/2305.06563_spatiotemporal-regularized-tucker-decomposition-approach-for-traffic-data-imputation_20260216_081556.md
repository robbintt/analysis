# Spatiotemporal Regularized Tucker Decomposition Approach for Traffic Data Imputation

## Quick Facts
- arXiv ID: 2305.06563
- Source URL: https://arxiv.org/abs/2305.06563

## Executive Summary
This paper introduces a novel tensor-based method for spatiotemporal traffic data imputation that combines low-rank Tucker decomposition with manifold regularization and temporal constraints. The approach reshapes traffic data into higher-order tensors using multiway delay embedding, then applies alternating proximal gradient updates to solve the optimization problem. Extensive experiments on real-world traffic datasets demonstrate superior performance compared to baseline methods across various missing scenarios, particularly when dealing with high missing data ratios.

## Method Summary
The Manifold Regularized Tucker Decomposition (ManiRTD) method addresses spatiotemporal traffic data imputation by first reshaping multivariate traffic time series into 3rd or 4th order Hankel tensors using the Multiway Delay Embedding Transform. The model then factorizes these tensors using a sparse core Tucker decomposition, where the tensor is approximated by multiplying non-negative factor matrices with a sparse core tensor. Manifold regularization is applied to capture spatial similarities between nearby sensors through a similarity graph, while Toeplitz regularization enforces temporal continuity. The optimization is solved using alternating proximal gradient updates with soft-thresholding for core tensor sparsity.

## Key Results
- ManiRTD achieves lower RMSE and NMAE scores compared to baseline methods (LATC, LR-SETD, TAS-LR) across random, non-random, and blackout missing scenarios
- The approach demonstrates particular effectiveness when missing data ratios exceed 20%, outperforming traditional low-rank decomposition methods
- Performance varies across datasets, with the method showing consistent improvement over baselines on Guangzhou urban traffic speed, Abilene internet traffic flow, and Portland highway traffic volume datasets

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Low-rank Tucker decomposition with sparse core captures global traffic patterns efficiently.
- Mechanism: The Tucker decomposition approximates the traffic tensor as a product of low-rank factor matrices and a sparse core tensor, allowing compact representation of long-term trends while avoiding the computational cost of full SVD.
- Core assumption: Traffic data inherently has low-rank structure in the Tucker sense, meaning correlations can be captured by a small set of basis vectors across modes.
- Evidence anchors:
  - [abstract]: "the tensor is approximated by multiplying non-negative factor matrices with a sparse core tensor"
  - [section]: "the Tucker decomposition is a suitable model for STDI" and "low-rank approximation (referred to decomposition-based model in this paper) is the most helpful tool for STDI"
  - [corpus]: Weak. Nearest neighbors include similar tensor-based traffic imputation papers, but none explicitly validate the low-rank assumption for traffic data.
- Break condition: If traffic data lacks consistent spatiotemporal correlations (e.g., highly localized anomalies), the low-rank assumption fails and imputation accuracy degrades sharply.

### Mechanism 2
- Claim: Manifold regularization enforces spatial similarity between nearby sensors.
- Mechanism: By constructing a similarity graph of sensors and applying Laplacian regularization to factor matrices, the model ensures that sensors with similar traffic patterns have similar latent representations, improving imputation in missing regions.
- Core assumption: Sensors that are geographically close or topologically connected exhibit similar traffic patterns over time.
- Evidence anchors:
  - [abstract]: "manifold regularization and temporal constraint are employed to capture spatiotemporal correlations"
  - [section]: "data collected by nearby sensors tend to have similar characteristics, which reflects spatial similarity" and the detailed formulation of manifold regularization with KNN and similarity matrix W
  - [corpus]: Weak. No direct corpus evidence; related papers focus on low-rank or temporal regularization, not manifold-based spatial similarity.
- Break condition: If the sensor network topology does not reflect actual traffic similarity (e.g., highways vs local roads), manifold regularization may introduce noise instead of improving accuracy.

### Mechanism 3
- Claim: Toeplitz regularization enforces temporal continuity and daily periodicity.
- Mechanism: A Toeplitz matrix is applied to the temporal factor matrix to penalize large differences between adjacent time points, capturing the smooth evolution of traffic and daily cycles.
- Core assumption: Traffic measurements at adjacent time points are highly correlated, and daily patterns repeat, making the temporal factor matrix approximately Toeplitz.
- Evidence anchors:
  - [abstract]: "temporal constraint are employed to capture spatiotemporal correlations"
  - [section]: "we use the Toeplitz constraint matrix T = Toeplitz(0, 1, -1), i.e., ∥TU∥2F to characterize the time continuity feature and the daily variation of Hankel traffic data"
  - [corpus]: Weak. No corpus neighbor explicitly discusses Toeplitz regularization for traffic imputation.
- Break condition: If traffic exhibits abrupt changes (e.g., incidents, weather), the Toeplitz assumption breaks, and the model oversmooths important variations.

## Architecture Onboarding

- Component map: Traffic matrix -> Hankel Operator -> Tucker Decomposition -> Manifold + Toeplitz Regularization -> Alternating Proximal Gradient Optimization -> Tensor Reconstruction -> Imputed Matrix
- Critical path: Hankel transformation → Tucker decomposition → Manifold + Toeplitz regularization → Alternating proximal gradient optimization → Tensor reconstruction → Matrix inversion
- Design tradeoffs:
  - Low-rank vs. sparse core: Balances global pattern capture with noise robustness
  - Manifold vs. Toeplitz regularization: Spatial vs. temporal emphasis
  - Tensor order (3rd vs. 4th): Captures more temporal lags but increases computational cost
- Failure signatures:
  - High imputation error in structurally missing data (NM/BM scenarios)
  - Slow convergence or divergence in optimization
  - Overfitting when hyperparameters α, β are too large
