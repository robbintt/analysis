# Detecting Evidence of Organization in groups by Trajectories

## Quick Facts
- **arXiv ID:** 2309.00172
- **Source URL:** https://arxiv.org/abs/2309.00172
- **Reference count:** 29
- **Primary result:** New detection approaches (graph entropy and clustering quality indices) more clearly identify organizational structure in multi-agent trajectories compared to previous methods

## Executive Summary
This paper addresses the challenge of detecting organizational structure inferences in multi-agent systems based on agent trajectories. The authors propose two new approaches: one based on evaluating graph entropy and the other on assessing clustering quality indices. They conduct experiments using four NetLogo simulations of animal kingdom scenarios (Ants, Wolf Sheep Predation, Flocking, and Ant Adaptation) to evaluate their approaches. The results demonstrate that their new detection approaches can more clearly identify the inferences of organizations or networks in the simulated scenarios compared to a previously proposed method in the literature.

## Method Summary
The proposed approaches analyze agent trajectories to detect organizational structure by first generating an adjacency matrix based on trajectory similarity and distance between agents. The graph entropy approach calculates normalized network entropy from this matrix, with lower values indicating more organized behavior. The clustering quality index approach applies DBSCAN clustering to the similarity matrix and evaluates cluster quality using the Silhouette Coefficient. The authors test these methods on four NetLogo simulations of animal kingdom scenarios, comparing results to a previously proposed method in the literature.

## Key Results
- The cluster quality index approach allows for better identification of organized vs. disorganized scenarios
- The graph entropy approach yields values closer to 0 than the previous method, indicating more organized behavior
- Both new approaches demonstrate superior detection of organizational structure compared to the previously proposed method

### Mechanism 2
- Claim: The graph entropy approach detects organizational structure by measuring the determinism of connections between nodes representing agents.
- Mechanism: After forming an adjacency matrix based on trajectory similarity and distance, the method calculates the normalized network entropy. Lower entropy values indicate more deterministic (organized) connections between agents.
- Core assumption: Entropy values close to zero reliably indicate organized behavior, while higher entropy indicates disorganized or random movement.
- Evidence anchors:
  - [abstract] "The other using graph entropy calculation to detect behavior anomalies"
  - [section] "A graph with entropy close to 0 suggests that the connections between nodes are highly deterministic, that is, there is a clear trend or pattern in the relationships"

- Break condition: If the adjacency matrix does not accurately capture meaningful relationships between agents, entropy values may not reliably indicate organization.

- Claim: The combined use of trajectory similarity and distance metrics in the adjacency matrix improves the detection of co-motion patterns compared to methods using only one metric.

## Architecture Onboarding

- Component map: Trajectory matrix generation -> Similarity and distance calculation -> Adjacency matrix formation -> DBSCAN clustering (for cluster quality index approach) -> Silhouette Coefficient calculation (for cluster quality index approach) -> Graph entropy calculation (for graph entropy approach) -> Result visualization and comparison

- Critical path:
  1. Generate trajectory matrix from agent positions over time
  2. Calculate similarity and distance metrics between all agent pairs
  3. Form adjacency matrix using combined metric
  4. Apply chosen approach (DBSCAN + Silhouette or Graph Entropy)
  5. Analyze results to identify organized vs. disorganized scenarios

- Design tradeoffs:
  - DBSCAN vs. other clustering algorithms: DBSCAN can find arbitrarily shaped clusters and handle noise, but requires careful parameter tuning
  - Window size selection: Larger windows may capture longer-term organization but miss short-term patterns; smaller windows may be too sensitive to noise
  - Combined similarity and distance metric: Captures both pattern similarity and spatial proximity, but may be complex to interpret

- Failure signatures:
  - All agents labeled as noise by DBSCAN (indicates poor parameter selection or lack of meaningful clusters)
  - Silhouette Coefficient values consistently near zero or negative (indicates poor clustering or lack of organization)
  - Graph entropy values that do not differentiate between organized and disorganized scenarios (indicates poor metric selection or implementation)

- First 3 experiments:
  1. Run the proposed approaches on a simple scenario with clearly organized agents (e.g., flocking birds) and verify that cluster quality indices are high and entropy values are low
  2. Test the approaches on a disorganized scenario (e.g., random walker agents) and verify that cluster quality indices are low/zero and entropy values are higher
  3. Vary the DBSCAN parameters (eps and min_samples) and observe their effect on cluster formation and quality index values

## Limitations

- Critical implementation details such as DBSCAN parameter selection and trajectory similarity calculations are not fully specified
