# Big Data$\unicode{x2013}$Supply Chain Management Framework for Forecasting: Data Preprocessing and Machine Learning Techniques

## Quick Facts
- arXiv ID: 2307.12971
- Source URL: https://arxiv.org/abs/2307.12971

## Executive Summary
This paper presents a comprehensive framework integrating Big Data Analytics (BDA) into Supply Chain Management (SCM) for enhanced forecasting. The framework proposes a cyclic process covering problem identification, data preprocessing, feature engineering, machine learning model training, hyperparameter optimization, performance evaluation, and optimization. It emphasizes the critical role of accurate data preprocessing and feature engineering in improving forecasting accuracy, and highlights the impact of forecasting on workforce efficiency, inventory management, and overall SC performance. The study evaluates various ML models for demand forecasting and recommends best-performing models like XGBoost-LSTM and FB-Prophet.

## Method Summary
The framework implements a cyclic connection between pre-process, control-process, and post-process phases, enabling continuous optimization of forecasting models. It involves comprehensive data preprocessing (handling duplicates, missing values, outliers), feature engineering, and training of various ML models (XGBoost-LSTM, FB-Prophet). The cyclic mechanism feeds performance metrics back into preprocessing for iterative improvements. The method was evaluated on supply chain forecasting tasks using metrics like MAPE, RMSE, and MAE, though specific datasets and quantitative results were not provided.

## Key Results
- Framework integrates 42+ data sources across supply chain levers for comprehensive analytics
- XGBoost-LSTM and FB-Prophet identified as best-performing models for demand forecasting
- Cyclic feedback mechanism enables continuous optimization of forecasting accuracy
- Proper data preprocessing directly improves ML model performance in supply chain contexts

- Mechanism: Performance metrics from the post-process phase (e.g., MAPE, RMSE) are fed back into the pre-process phase to refine data preprocessing and feature engineering, creating a closed-loop system that iteratively improves forecasting accuracy.
- Core assumption: Performance measurement data is available and reliable enough to inform preprocessing adjustments.

## Architecture Onboarding

- Component map: Data Collection → Preprocessing → Model Training → Performance Monitoring → Feedback Loop
- Critical path: Data Collection → Preprocessing → Model Training → Performance Evaluation → Feedback → Preprocessing
- Design tradeoffs:
  - Real-time vs. batch processing: Real-time enables immediate adjustments but requires more infrastructure
  - Model complexity vs. interpretability: Complex models may perform better but are harder to explain to stakeholders
  - Data volume vs. processing speed: More data can improve accuracy but increases computational requirements

- Failure signatures:
  - Performance degradation over time: Indicates feedback loop not working properly
  - High variance in error metrics: Suggests preprocessing issues or unstable data sources
  - Model training failures: May indicate data quality issues or feature engineering problems
