# Exploring Self-supervised Skeleton-based Action Recognition in Occluded Environments

## Quick Facts
- arXiv ID: 2309.12029
- Source URL: https://arxiv.org/abs/2309.12029
- Reference count: 40
- Primary result: Proposed OPSTL framework achieves up to 2.28% accuracy improvement on NTU-120 dataset for self-supervised skeleton-based action recognition under occlusions

## Executive Summary
This paper addresses the challenge of self-supervised skeleton-based action recognition in occluded environments, which is critical for robotic applications but often overlooked. The authors propose a two-stage approach that first pre-trains a model using a novel Adaptive Spatial Masking (ASM) data augmentation mechanism, then imputes missing skeleton data using a KMeans + KNN approach. The method, called Occluded Partial Spatio-Temporal Learning (OPSTL), outperforms state-of-the-art methods on the occluded versions of NTU-60 and NTU-120 datasets. The key innovation is ASM, which masks joints based on their occlusion frequency in the dataset rather than using fixed centrality-based strategies, improving robustness to realistic occlusions.

## Method Summary
The method consists of two stages: (1) Pre-training using occluded skeleton sequences with Adaptive Spatial Masking (ASM) to learn robust feature representations, and (2) Imputation using KMeans clustering on sequence embeddings followed by KNN-based completion of missing skeleton data. The first stage replaces the Central Spatial Masking (CSM) of PSTL with ASM, which adaptively switches between partial and random occlusion based on joint occlusion frequency. The second stage clusters embeddings into semantically similar groups using KMeans, then performs KNN imputation within clusters to complete missing joint coordinates. The completed sequences are then used for downstream action recognition tasks.

## Key Results
- OPSTL achieves up to 2.28% accuracy improvement on NTU-120 dataset compared to state-of-the-art methods
- The framework demonstrates effectiveness in handling occlusions through two-stage training approach
- Outperforms existing methods on both linear evaluation, semi-supervised, and finetune protocols

## Why This Works (Mechanism)

### Mechanism 1
- Claim: KMeans clustering in embedding space reduces computational burden of KNN imputation by grouping semantically similar samples
- Mechanism: Pre-trained encoder generates embeddings for each skeleton sequence, KMeans clusters these into fewer groups, KNN searches within clusters instead of entire dataset
- Core assumption: Self-supervised pre-training captures semantic similarity such that same action types are grouped together
- Evidence anchors: Abstract mentions k-means clustering on sequence embeddings; section III-D describes grouping through KMeans; weak corpus support
- Break condition: If pre-training fails to learn meaningful embeddings, KMeans won't group similar samples, causing poor imputation

### Mechanism 2
- Claim: ASM improves robustness by masking joints based on occlusion frequency rather than fixed centrality strategy
- Mechanism: ASM computes frequency of each joint being occluded, masks frequently occluded joints during training, falls back to centrality-based masking when no joints are occluded
- Core assumption: Frequently occluded joints require more robust representations to learn
- Evidence anchors: Abstract mentions ASM for better use of intact skeletons; section III-C describes dataset-driven ASM; weak corpus support
- Break condition: If occlusion frequency distribution is uniform or dataset has too few occluded samples, ASM provides no benefit over centrality-based masking

### Mechanism 3
- Claim: Two-stage training (pre-training with ASM, then imputation) improves downstream performance by learning robust representations first
- Mechanism: Stage 1 - pre-train with occluded sequences and ASM; Stage 2 - generate embeddings, cluster with KMeans, impute with KNN within clusters
- Core assumption: Learning robust representations in presence of occlusions improves embedding quality for clustering and imputation
- Evidence anchors: Abstract describes two-stage process; section III explains replacing CSM with ASM in PSTL; weak corpus support
- Break condition: If pre-training fails to learn meaningful representations, embeddings will be poor quality, leading to bad clustering and imputation

## Architecture Onboarding

- Component map: ST-GCN backbone -> Projector network -> Adaptive Spatial Masking (ASM) module -> KMeans clustering -> KNN imputer -> Linear classifier
- Critical path: 1) Pre-training: Occluded skeleton sequences → ST-GCN + ASM → embeddings; 2) Clustering: Embeddings → KMeans → pseudo-labels; 3) Imputation: Occluded sequences + pseudo-labels → KNN → completed sequences; 4) Downstream: Completed sequences → ST-GCN → linear classifier
- Design tradeoffs: KMeans + KNN trades computational efficiency for potential accuracy vs searching entire dataset; ASM focuses on realistic occlusion patterns but may not generalize to random occlusions; two-stage approach adds complexity but allows specialized occlusion handling
- Failure signatures: Poor downstream performance may indicate pre-training issues (ASM not learning robust features), clustering quality problems (embeddings not capturing semantic similarity), or imputation accuracy issues (KNN finding poor neighbors); high variance in imputation suggests imbalanced clusters or insufficient training data diversity
- First 3 experiments: 1) Ablation study comparing downstream performance with and without ASM in pre-training stage; 2) Imputation quality measurement against ground truth for subset of samples; 3) Clustering analysis with embedding visualization to assess semantic grouping quality

## Limitations

- Limited empirical validation of ASM mechanism - no direct ablation study comparing against alternative masking strategies under controlled occlusion rates
- Clustering quality assumptions - effectiveness depends on embedding quality from self-supervised pre-training, but no quantitative clustering quality analysis provided
- Dataset-specific performance claims - improvements based on synthetically occluded NTU datasets, unclear if generalizes to naturally occurring occlusions or different skeleton representations

## Confidence

- High confidence: Two-stage framework architecture (pre-training → clustering → imputation) is well-defined and technically sound
- Medium confidence: Computational efficiency gain from KMeans clustering is mathematically valid, but practical impact depends on dataset size and cluster quality
- Low confidence: Claim that ASM specifically improves robustness to realistic occlusions needs more rigorous validation across different occlusion patterns
