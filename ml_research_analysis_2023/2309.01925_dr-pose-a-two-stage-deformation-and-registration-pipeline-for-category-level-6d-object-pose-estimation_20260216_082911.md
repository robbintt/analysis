# DR-Pose: A Two-stage Deformation-and-Registration Pipeline for Category-level 6D Object Pose Estimation

## Quick Facts
- arXiv ID: 2309.01925
- Source URL: https://arxiv.org/abs/2309.01925
- Reference count: 30
- This paper addresses category-level 6D object pose estimation, which involves estimating the 6D pose and 3D metric size of objects from predetermined categories.

## Executive Summary
This paper introduces DR-Pose, a two-stage pipeline for category-level 6D object pose estimation that separates deformation and registration tasks to optimize performance for each. The method first uses point cloud completion to generate unseen object parts, which guides the deformation of shape priors to reconstruct scale-invariant CAD models. In the second stage, a registration network extracts pose-sensitive features and predicts NOCS representations to estimate the final 6D pose and size. DR-Pose achieves state-of-the-art results on both CAMERA25 and REAL275 benchmarks, particularly excelling in stricter metrics like 3D75 and 5◦2cm.

## Method Summary
DR-Pose is a two-stage pipeline that addresses category-level 6D object pose estimation. The first stage uses point cloud completion (PoinTr) to generate unseen parts of the target object, which guides the deformation of shape priors to reconstruct scale-invariant CAD models. The second stage employs a registration network that extracts pose-sensitive features and predicts the NOCS representation of observed points in canonical space. The method uses scaling factors to adjust registration results and compensate for imperfect deformation. The pipeline is trained separately for each stage, with the deformation network trained for 100 epochs and the registration network for 150 epochs on NVIDIA RTX3090 GPUs.

## Key Results
- DR-Pose achieves superior results to state-of-the-art shape prior-based methods on both CAMERA25 and REAL275 benchmarks
- On the REAL275 dataset, DR-Pose improves the state-of-the-art results by 6.3% and 5.8% respectively in terms of the strictest metrics 3D75 and 5◦2cm
- The method shows consistent improvement across all evaluation metrics, demonstrating robustness to occlusion and clutter in real-world scenarios

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Separating deformation and registration into two stages allows each task to be optimized independently, avoiding suboptimal trade-offs from combined loss functions.
- Mechanism: In single-stage methods, losses for deformation and registration are summed together, forcing the network to compromise between optimizing both tasks. The two-stage approach first trains a deformation network to reconstruct scale-invariant CAD models optimally, then trains a registration network using the well-optimized deformation results.
- Core assumption: Deformation and registration tasks have distinct optimization requirements that conflict when combined in a single network.
- Evidence anchors:
  - [abstract] "Since the registration task takes the deformed shape prior as reference, there is inherent cascaded relation between two tasks. To this end, our DR-Pose is proposed in a two-stage design and training manner."
  - [section] "In the training process, losses for the two tasks are summed together to optimize the network, which may lead to tradeoff of network optimization and sub-optimal performance for each task."
  - [section] "To this end, we propose a two-stage deformation-and-registration pipeline to optimize the network design and performance for each task."
- Break condition: If the deformation stage produces poor results, the registration stage cannot recover, making the two-stage separation ineffective.

### Mechanism 2
- Claim: Point cloud completion provides unseen object parts that guide more accurate deformation of shape priors.
- Mechanism: The completion network (PoinTr) generates missing parts of the observed partial point cloud, which serves as reference to deform the categorical shape prior. This provides more complete geometric features for accurate reconstruction of the scale-invariant CAD model.
- Core assumption: The completion network can reliably generate unseen object parts that are geometrically consistent with the observed partial point cloud.
- Evidence anchors:
  - [section] "The first stage uses a point cloud completion method to generate unseen parts of target object, guiding subsequent deformation on the shape prior."
  - [section] "Point cloud completion method to generate unseen parts of target object, guiding subsequent deformation on the shape prior."
- Break condition: If the completion network introduces artifacts or generates inconsistent geometry, the deformation results will be corrupted.

### Mechanism 3
- Claim: Scaling factors allow adjustment of registration results to compensate for imperfect deformation.
- Mechanism: The registration network predicts per-point scaling factors that adjust the correspondence matrix between observed points and deformed shape prior. This breaks the limitation where imperfect deformation would otherwise make perfect registration impossible.
- Core assumption: The registration network can learn appropriate scaling factors to correct translation, rotation, or scale errors between deformed prior and ground truth.
- Evidence anchors:
  - [section] "To alleviate this limitation, we add a branch at the head of registration network to regress a vector of scaling factors γ ∈ RNˆo×1."
  - [section] "These learnable scaling factors enable our DR-Pose to adjust the predicted Onocs to break the limit of imperfect Pdef."
- Break condition: If scaling factors overcompensate or introduce instability, pose estimation accuracy may degrade.

## Architecture Onboarding

- Component map: Segmentation -> Point cloud completion -> Deformation -> Registration -> Pose fitting
- Critical path: Segmentation → Point cloud completion → Deformation → Registration → Pose fitting
- Design tradeoffs:
  - Two-stage vs single-stage: Two-stage provides better optimization but adds complexity and potential error accumulation
  - Completion network choice: PoinTr provides good results but increases computational overhead
  - Scaling factors: Improve robustness to deformation errors but add parameters to learn
- Failure signatures:
  - Poor completion results: Visible artifacts in reconstructed geometry, inconsistent with observed parts
  - Inaccurate deformation: Large Chamfer distance between deformed prior and ground truth CAD model
  - Registration failures: Incorrect correspondences, especially on symmetric or repeated features
  - Scaling factor issues: Overcompensation leading to distorted NOCS predictions
- First 3 experiments:
  1. Verify completion network produces plausible geometry by visualizing completed point clouds against observed partial clouds
  2. Test deformation stage accuracy by measuring Chamfer distance between deformed shape prior and ground truth CAD models
  3. Validate registration stage by checking if predicted NOCS coordinates align with ground truth when scaling factors are disabled

## Limitations

- Cascaded Error Propagation: The two-stage design creates sequential dependency where errors in deformation stage directly impact registration performance
- Completion Network Dependency: Method's performance heavily relies on quality of point cloud completion network (PoinTr)
- Generalization to Novel Categories: Does not investigate performance when categories are outside training distribution

## Confidence

- **High Confidence**: Core methodology of separating deformation and registration into two stages is well-justified and supported by clear experimental validation
- **Medium Confidence**: Claim that scaling factors significantly improve registration robustness is supported by ablation study but mechanism could be more thoroughly explained
- **Low Confidence**: Assumption that point cloud completion consistently provides geometrically valid unseen parts is critical but not rigorously validated

## Next Checks
