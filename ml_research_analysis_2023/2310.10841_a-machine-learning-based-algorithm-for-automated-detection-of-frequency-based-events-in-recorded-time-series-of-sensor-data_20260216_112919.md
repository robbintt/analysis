# A Machine Learning-based Algorithm for Automated Detection of Frequency-based Events in Recorded Time Series of Sensor Data

## Quick Facts
- **arXiv ID**: 2310.10841
- **Source URL**: https://arxiv.org/abs/2310.10841
- **Reference count**: 0
- **Primary result**: Achieves 0.97 precision rate in detecting Lane Departure Warning events in vehicle testing data

## Executive Summary
This paper presents a novel machine learning-based method for automated detection of frequency-based events in recorded time series data from vehicle sensors. The approach transforms time series into time-frequency domain representations (scalograms) using continuous wavelet transform, filters them to highlight target events, and applies YOLO object detection to identify event objects in the scalograms. The detected objects are mapped back to time series data to mark event time intervals. The method is evaluated on vehicle testing data and achieves high precision in detecting Lane Departure Warning events, providing sharp time interval boundaries that are challenging to determine by visual inspection.

## Method Summary
The algorithm preprocesses time series data by computing Continuous Wavelet Transform (CWT) scalograms using Gabor wavelets, then filters scalograms to highlight LDW events using C1/C2 thresholds. YOLOv5 object detection is applied to identify event objects in filtered scalograms. The method uses manual annotation with LabelImg, splits data into 55% train, 15% validation, 30% inference sets, and trains YOLOv5 models for 150 epochs. Detection results are mapped back to original time series to mark event intervals with precise temporal boundaries.

## Key Results
- Achieves precision rate of 0.97 in detecting Lane Departure Warning events
- Provides sharp time interval boundaries that are difficult to determine by visual inspection
- YOLOv5m model outperforms smaller versions with accuracy of 0.997 and F1-score of 0.993 at confidence threshold 0.60

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Scalogram transformation enables precise localization of frequency-based events in time series
- Mechanism: CWT decomposes time series into time-frequency representations where event-related energy concentrates in distinct patterns that YOLO can detect
- Core assumption: Event frequency signatures remain consistent across instances to create recognizable patterns
- Break condition: If frequency characteristics vary significantly, scalogram patterns become inconsistent and detection fails

### Mechanism 2
- Claim: YOLO object detection provides precise temporal boundary localization for events
- Mechanism: YOLO predicts bounding boxes around event objects in scalograms, mapping directly to time intervals in original time series
- Core assumption: Event objects in scalograms have well-defined spatial extent corresponding to temporal duration
- Break condition: If event objects overlap significantly, NMS may suppress valid detections or create ambiguous boundaries

### Mechanism 3
- Claim: Scalogram filtering enhances detection accuracy by suppressing noise
- Mechanism: Filter removes CWT magnitudes outside expected event range, making event objects more prominent for YOLO detection
- Core assumption: Event-related CWT magnitudes fall within predictable range that can be isolated
- Break condition: If filter range is too narrow, event content may be eliminated; if too broad, noise won't be effectively suppressed

## Architecture Onboarding

- Component map: Raw time series → CWT → Scalogram → Filter → YOLO object detection → Bounding boxes → Time interval mapping → Event detection output
- Critical path: Scalogram generation and filtering must complete before YOLO can process the data
- Design tradeoffs: Larger YOLO models provide better accuracy but slower inference; filter range selection balances noise suppression vs. event preservation
- Failure signatures: Low precision/recall indicates filtering range issues or YOLO model inadequacy; poor time interval alignment suggests mapping errors
- First 3 experiments:
  1. Generate scalograms from sample data and verify they show expected patterns for known events
  2. Apply filtering algorithm and visually inspect whether event objects become more prominent
  3. Train simple YOLO model on synthetic scalogram data with known event locations and measure detection accuracy

## Limitations
- Scalogram filtering parameters (C1/C2 thresholds) determined through domain inspection rather than automated methods
- Manual annotation requirement using LabelImg creates labor-intensive preprocessing step
- Evaluation limited to single vehicle model and sensor configuration, raising generalizability questions
- YOLO model size selection (n/s/m) not specified, affecting accuracy and computational requirements

## Confidence

- **High Confidence**: CWT scalogram generation and mapping bounding box detections back to time series intervals
- **Medium Confidence**: Reported 0.97 precision for LDW detection based on limited evaluation data
- **Low Confidence**: Generalizability to other frequency-based events or sensor configurations without parameter tuning
