# A Hybrid ANN-SNN Architecture for Low-Power and Low-Latency Visual Perception

## Executive Summary
This paper addresses the challenge of high latency and energy consumption in Spiking Neural Networks (SNNs) by proposing a hybrid ANN-SNN architecture. The key innovation is using an auxiliary Artificial Neural Network (ANN) to periodically initialize the states of an SNN, enabling the SNN to generate high-frequency, low-latency predictions without requiring long convergence periods. The hybrid model effectively combines the accuracy of ANNs with the energy efficiency of SNNs. Experimental results on event-based 2D and 3D human pose estimation tasks demonstrate significant improvements: 88% reduction in power consumption compared to standard ANNs (with only 4% accuracy decrease) and 74% lower error rate compared to pure SNNs.

## Method Summary
The hybrid architecture uses an auxiliary ANN to initialize SNN states at low rates, enabling the SNN to generate high-frequency, low-latency predictions. The ANN processes dense event representations (stacked 2D histograms from event data) to predict joint locations and initialize SNN states. The SNN then processes incoming event streams to update predictions with high temporal resolution. The model is trained using backpropagation through time with surrogate gradients, implemented using the Spiking Jelly framework. The approach is evaluated on event-based human pose estimation datasets (DHP19 and Event-Human3.6M), with the ANN running at 30Hz and the SNN at 1000Hz to achieve the power-efficiency benefits.

## Key Results
- Hybrid model reduces power consumption by 88% compared to standard ANNs
- Only 4% decrease in accuracy relative to pure ANN baseline
- 74% lower error rate compared to pure SNN approach
- Effective on both 2D and 3D event-based human pose estimation tasks

## Architecture Onboarding

- Component map: Event-based input -> Dense event representation -> ANN -> State initialization module -> SNN -> Predictions

- Critical path:
  1. Event data is received and converted into dense event representation
  2. ANN processes dense event representation and generates initial prediction and state initialization for SNN
  3. SNN uses initialized states and generates high-frequency, low-latency predictions using spikes
  4. Predictions are output and can be sampled at any point in time

- Design tradeoffs:
  - ANN vs. SNN: ANN provides accurate initial predictions and state initialization but runs at lower rate and consumes more power; SNN generates high-frequency, low-latency predictions with low power consumption but may have lower accuracy and longer convergence times
  - Event-based vs. frame-based input: Event-based input provides sparse and asynchronous data that can be processed efficiently by SNN but may require additional preprocessing to convert into dense representation for ANN

- Failure signatures:
  - Low accuracy: If ANN cannot provide accurate initial predictions and state initialization, or if SNN cannot generate accurate predictions with low power consumption
  - High latency: If SNN cannot generate predictions quickly enough, or if ANN cannot provide initial predictions and state initialization fast enough
  - High power consumption: If ANN runs at high rate, or if SNN cannot generate predictions with low power consumption
