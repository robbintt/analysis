# The online learning architecture with edge computing for high-level control for assisting patients

## Quick Facts
- arXiv ID: 2309.05130

## Executive Summary
This paper introduces a novel online adversarial learning architecture integrated with edge computing for high-level lower-limb exoskeleton control. The proposed system uses sensor data from users processed in real-time through edge computing nodes, which interact with an online adversarial learning model to adapt to user-specific needs with minimal latency. Experimental evaluations show significant improvements in control accuracy and adaptability, as well as enhanced quality-of-service (QoS) metrics. The integration of online adversarial learning with edge computing offers a robust and efficient approach for the next generation of lower-limb exoskeleton control systems.

## Method Summary
The system processes sensor data (EMG signals, angular motion, positioning information) from users in real-time through edge computing nodes. These nodes handle signal preprocessing, feature extraction using Walsh Hadamard Transform, and classification to produce actuation signals. An online adversarial learning model continuously adapts to user-specific patterns, optimizing control parameters for individual gait and movement preferences. The architecture integrates SDN-powered USRP hardware for flexible RF communication and network optimization, enabling dynamic routing and resource allocation for EMG data streams.

## Key Results
- Significant improvements in control accuracy compared to traditional control methods
- Enhanced adaptability to individual user patterns through online adversarial learning
- Improved quality-of-service (QoS) metrics through SDN-powered communication optimization

## Architecture Onboarding

- Component map:
  Human body sensor module (EMG, angular motion sensors) -> Node MCU with edge computing (signal preprocessing, classification) -> USRP hardware (RF transmission and reception) -> SDN controller (network optimization) -> Humanoid actuation module (motor drivers, feedback sensors)

- Critical path: Sensor → Edge node preprocessing → Classification → USRP transmission → SDN routing → Actuator control

- Design tradeoffs:
  - Edge vs. cloud processing: latency vs. computational power
  - Model complexity vs. adaptation speed: richer models may adapt better but require more training data
  - Communication bandwidth vs. control precision: higher data rates enable finer control but consume more resources

- Failure signatures:
  - High latency in EMG-to-actuation loop
  - Unstable control behavior indicating model adaptation issues
  - Communication errors or dropped packets affecting QoS
