# Synthesizing Event-centric Knowledge Graphs of Daily Activities Using Virtual Space

## Quick Facts
- **arXiv ID**: 2307.16206
- **Source URL**: https://arxiv.org/abs/2307.16206
- **Reference count**: 40
- **Primary result**: Framework generates synthetic KGs with over 1M entities and 8M triples; achieves 0.75 F1-score for fall risk detection

## Executive Summary
This paper presents VirtualHome2KG, a framework that generates synthetic knowledge graphs (KGs) representing daily activities in home environments using virtual space simulation. By simulating activities in the VirtualHome platform and converting the outputs to RDF triples using an event-centric schema, the framework bypasses the need for real-world data collection while maintaining rich spatiotemporal context. The generated KGs enable context-aware analysis through querying, clustering, and embedding, and demonstrate practical utility through fall risk detection for older adults, achieving 0.75 F1-score.

## Method Summary
The VirtualHome2KG framework simulates daily activities in a virtual 3D environment (VirtualHome) and converts the simulation outputs into RDF triples using a custom event-centric schema. The process involves loading crowdsourced activity programs into the simulator, executing activity steps while recording object states, positions, and temporal changes, then transforming these simulation results into structured KGs stored in RDF format. The framework specifically focuses on representing object states, affordances, time-series changes, and spatial relationships to enable context-aware analysis of daily activities.

## Key Results
- Generated KGs contain over 1 million entities and 8 million triples from 451 simulated activities
- Event-centric schema successfully represents object states, affordances, time-series changes, and spatial relationships
- Fall risk detection application achieves 0.75 F1-score using synthetic KGs combined with expert rules
- KGs enable various analyses including querying, clustering, and embedding using RDF2Vec

## Why This Works (Mechanism)

### Mechanism 1
- **Claim**: VirtualHome2KG generates synthetic KGs with over 1 million entities and 8 million triples without real-world data collection
- **Mechanism**: Simulates daily activities in VirtualHome 3D environment and converts outputs to RDF triples using event-centric schema
- **Core assumption**: Simulation accurately represents real-world object states, affordances, and spatial relationships
- **Evidence anchors**: Experiments show KG with 1,290,252 entities, 52 properties, and 8,477,485 triples; Corpus shows related work on synthetic multimodal datasets

### Mechanism 2
- **Claim**: Event-centric KG schema enables context-aware analysis by representing object states, affordances, time-series changes, and spatial relationships
- **Mechanism**: Schema links events to agents, locations, times, while capturing object states and affordances
- **Core assumption**: Event-centric model is more expressive than frame-by-frame scene graphs for sequential activities
- **Evidence anchors**: Schema designed to represent human activities and situational changes; includes atomic actions from VirtualHome

### Mechanism 3
- **Claim**: Framework supports domain-specific applications by combining synthetic KGs with expert-defined rules
- **Mechanism**: Generated KGs enriched with domain knowledge through inference rules for automated detection
- **Core assumption**: Expert rules can be accurately mapped to KG structures and synthetic data captures real-world variability
- **Evidence anchors**: Fall risk detection achieves 0.75 F1-score; defined risk classes and subclasses for inferring high-risk behaviors

## Architecture Onboarding

- **Component map**: VirtualHome simulator (Unity + Graph Evolve) -> Activity dataset (crowdsourced programs) -> KG schema designer (event-centric, based on HomeOntology) -> KG constructor (converts simulation to RDF triples) -> GraphDB triplestore (stores and queries KGs) -> Application layer (fall risk detection, querying, clustering)

- **Critical path**: 1) Load activity program into VirtualHome simulator, 2) Execute activity steps, record object states/positions/time, 3) Convert simulation results to RDF triples using event-centric schema, 4) Store triples in GraphDB, 5) Query or analyze KGs for target application

- **Design tradeoffs**: Rendering vs. non-rendering simulation (3D coordinates vs. speed), Event-centric vs. simpler schemas (context vs. complexity), Expert rules vs. learned models (interpretability vs. generalization)

- **Failure signatures**: Missing/incorrect object states in KGs, Inability to execute certain activity programs, Low accuracy in domain-specific applications, Poor clustering/embedding results due to schema limitations

- **First 3 experiments**: 1) Generate KGs for 5 activities and verify entity/triple counts and schema structure, 2) Query KGs to retrieve specific activity patterns and validate results, 3) Apply fall risk detection rules to generated KGs and measure precision/recall

## Open Questions the Paper Calls Out
The paper does not explicitly call out open questions but implies several areas for future research through its discussion of potential extensions to other domains and the limitations of synthetic data.

## Limitations
- Limited validation of synthetic KGs' real-world applicability and generalization to actual home environments
- Fall risk detection F1-score of 0.75 based on synthetic scenarios may not translate to real-world conditions
- Heavy reliance on virtual simulation may miss complex real-world dynamics and variability
