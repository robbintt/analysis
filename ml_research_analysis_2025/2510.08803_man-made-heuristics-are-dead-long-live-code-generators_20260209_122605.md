# Man-Made Heuristics Are Dead. Long Live Code Generators!

- arXiv ID: 2510.08803

## Executive Summary
PolicySmith automates the generation of system heuristics using LLMs and evolutionary search, eliminating manual design. Given a policy template and evaluation context, it synthesizes optimized code for specific workloads, hardware, and objectives. Applied to web caching, it discovers heuristics that outperform 14 established baselines on real traces, with instance-optimal policies improving hit rates by 2% over manual oracles. In Linux congestion control, PolicySmith generates safe, compilable eBPF code that achieves up to 98% bandwidth utilization, demonstrating feasibility in safety-critical systems. The approach reduces manual effort, improves performance, and enables dynamic policy adaptation as contexts shift.

## Method Summary
PolicySmith uses an iterative search loop where an LLM generates candidate heuristics based on a policy template, the candidates are evaluated in a simulated context, and top performers are fed back as examples for the next generation. The system separates decision logic from state management in its templates, allowing the LLM to focus on generating smaller, more self-contained code blocks. For kernel deployment, generated code is compiled as eBPF programs that pass through the eBPF verifier for safety checks before execution.

- **Claim:** Separating decision logic from state management in the Template design improves generation success rates
- **Mechanism:** By isolating the policy logic from state handling, the LLM only needs to generate smaller, self-contained code blocks that fit into predefined interfaces, reducing the complexity of each generation task.
- **Core assumption:** Smaller code generation tasks are more likely to succeed than full system implementations.
- **Evidence anchors:**
  - [section 3.2] "PolicySmith templates separate the management of the underlying state from the decision logic, allowing the LLM to focus on generating smaller, more self-contained code blocks"

- **Break condition:** If the template abstraction leaks implementation details that the LLM must guess, or if the interface constraints are too rigid to express optimal policies.
