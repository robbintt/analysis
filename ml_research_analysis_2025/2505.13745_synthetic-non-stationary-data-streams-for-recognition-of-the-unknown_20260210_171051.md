# Synthetic Non-stationary Data Streams for Recognition of the Unknown

## Quick Facts
- arXiv ID: 2505.13745
- Source URL: https://arxiv.org/abs/2505.13745
- Authors: Joanna Komorniczak
- Reference count: 37
- This paper introduces the Open World Data Stream Generator with Concept Non-stationarity (owdsg), a method for generating synthetic data streams that include both concept drifts and the emergence of novel classes.

## Executive Summary
This paper presents the Open World Data Stream Generator with Concept Non-stationarity (owdsg), a novel method for generating synthetic data streams that incorporate both concept drifts and the emergence of novel classes. The generator leverages the Madelon generator to create static class distributions and arranges them into continuous streams, enabling realistic simulation of data stream characteristics including class imbalance and unknown instance proportions. The owdsg method provides a valuable tool for evaluating methods designed for processing data streams in changing, non-stationary environments.

The paper demonstrates the utility of the generated streams through two experiments. The first evaluates unsupervised drift detectors' ability to distinguish between concept drifts and novel class emergence based on method sensitivity. The second experiment examines open set recognition with incremental learning of new classes using a Multilayer Perceptron classifier with decision support thresholding. Results show that controlling drift detector sensitivity can help identify both concept drifts and novel class emergence, while recognition quality varies depending on class novelty and concept drift contexts.

## Method Summary
The owdsg method generates synthetic data streams by using the Madelon generator to create static class distributions, which are then arranged into continuous streams. This approach allows for concept drifts within known classes and the appearance of unknown classes. The generator can simulate various data stream characteristics, including class imbalance and the proportion of unknown instances. By arranging static class distributions from Madelon into a continuous stream, the method creates non-stationary environments that include both gradual concept changes and sudden novel class appearances, making it suitable for evaluating open-world learning algorithms.

## Key Results
- Controlling the sensitivity of drift detectors can help identify both concept drifts and novel class emergence
- Recognition quality changes in the context of class novelty and concept drift
- Incremental training of previously unknown classes affects overall performance in open set recognition scenarios

## Why This Works (Mechanism)
The owdsg method works by creating a controlled environment where both concept drift and novel class emergence can be systematically introduced and studied. By using the Madelon generator as a foundation for creating static class distributions, the method ensures that the underlying data structure remains consistent while allowing for controlled variations in class boundaries and the introduction of new classes. This controlled variability enables researchers to study the behavior of learning algorithms under different scenarios of concept drift and class novelty emergence.

- **Incremental learning**: Why needed - Critical for adapting to new classes in streaming scenarios; Quick check - Monitor model performance degradation over time

## Architecture Onboarding

Component Map:
owdsg Generator -> Data Stream Output -> Drift Detector -> Classifier -> Evaluation Metrics

Critical Path:
Static class distributions (Madelon) -> Stream arrangement -> Concept drift injection -> Novel class emergence -> Evaluation

Design Tradeoffs:
- Generator complexity vs. realism: More complex generators may better simulate real-world scenarios but increase computational overhead
- Drift frequency vs. detection accuracy: Higher drift frequency tests detector robustness but may overwhelm learning algorithms
- Unknown class proportion vs. system stability: More unknown classes challenge open set recognition but may destabilize incremental learning

Failure Signatures:
- Classifier performance degradation when encountering novel classes
- Drift detector false positives/negatives in distinguishing concept drift from novel classes
- Incremental learning failure to adapt to new class distributions

First 3 Experiments:
1. Baseline test with static distributions to verify Madelon generator functionality
2. Single drift scenario to test drift detection capabilities
3. Novel class emergence with known class stability to validate open set recognition

## Limitations
- Reliance on Madelon generator may limit diversity and realism of generated streams for complex real-world applications
- Effectiveness in simulating realistic concept drifts and novel class emergence patterns needs validation against real-world data streams
- Limited scope of experiments focused primarily on unsupervised drift detection and open set recognition

## Confidence
High confidence: The basic functionality of the owdsg generator in creating synthetic data streams with concept drifts and novel class emergence.

Medium confidence: The effectiveness of the generated streams in simulating realistic data stream characteristics and the utility of the generator for method evaluation.

Low confidence: The generalizability of the results to diverse real-world applications and the generator's ability to capture all relevant aspects of open-world data stream scenarios.
