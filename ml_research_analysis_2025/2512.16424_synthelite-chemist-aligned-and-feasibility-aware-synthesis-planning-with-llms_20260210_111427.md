# Synthelite: Chemist-aligned and feasibility-aware synthesis planning with LLMs

## Quick Facts
- **arXiv ID**: 2512.16424
- **Source URL**: https://arxiv.org/abs/2512.16424
- **Reference count**: 40
- **Primary result**: Synthelite achieves up to 95% success rates in synthesis planning tasks while demonstrating chemical feasibility awareness

## Executive Summary
This paper introduces Synthelite, an LLM-driven synthesis planning framework that directly proposes retrosynthetic transformations while enabling expert intervention through natural language prompts. The system employs a two-phase approach where Phase 1 generates synthesis blueprints through iterative LLM planning, and Phase 2 refines these using similarity-based Monte Carlo Tree Search. Experiments show Synthelite achieves 95% success rates in both strategy-constrained and starting-material-constrained tasks, outperforming traditional neural-guided synthesis planning baselines while maintaining chemical plausibility throughout the planning process.

## Method Summary
Synthelite uses a two-phase architecture: Phase 1 iteratively plans synthesis routes through LLM-generated strategy descriptions, while Phase 2 refines these blueprints using MCTS with template similarity weighting. The system grounds LLM text descriptions in a 40,000-template library, incorporates natural language constraints directly into LLM context, and employs self-evaluation with feedback across planning attempts. The framework maps LLM-generated strategies to symbolic templates via vector embeddings and balances strategy alignment against template popularity through an α parameter.

## Key Results
- Achieves up to 95% success rates in strategy-constrained and starting-material-constrained synthesis tasks
- Outperforms traditional neural-guided synthesis planning baselines (43-47% to 95% success rates)
- Demonstrates chemical feasibility awareness with LLM-generated routes receiving higher feasibility scores from expert evaluators compared to conventional approaches

## Why This Works (Mechanism)

### Mechanism 1
Decoupling LLM strategic planning from tree search improves computational efficiency while preserving steering capability. Phase 1 uses LLMs to generate textual strategy descriptions iteratively; Phase 2 grounds these in template space via similarity-based MCTS, avoiding per-node LLM calls during exploration. Core assumption: LLM's latent chemical knowledge encodes valid retrosynthetic reasoning that can be expressed in natural language and mapped to symbolic templates.

### Mechanism 2
Natural language prompting enables flexible constraint incorporation without architectural modification. User constraints are injected directly into the LLM context at each planning step; the LLM adapts its strategy description accordingly, which propagates through template retrieval and selection. Core assumption: LLMs trained on chemical literature have internalized constraint-following behavior for synthesis logic.

### Mechanism 3
Iterative self-evaluation with feedback improves route feasibility across planning attempts. After each Phase 1 attempt, the LLM evaluates its own proposed route for feasibility issues (e.g., selectivity, protecting groups), generating feedback that conditions subsequent attempts. Core assumption: LLMs can reliably critique synthetic routes and identify wet-lab failure modes from SMILES representations.

## Architecture Onboarding

- **Component map**: Template Search Engine (40k SMARTS templates → LLM-generated descriptions → vector embeddings) → Phase 1 Loop (Target + prompt → LLM strategy → text description → template retrieval → LLM selection → update state) → Phase 2 MCTS (Blueprint routes → similarity-weighted expansion policy → stock availability terminal condition) → Self-Evaluation Module (Route → LLM critique → feedback for next attempt)

- **Critical path**: 1. Template embedding (one-time setup) → 2. Phase 1 iterative planning (3 attempts with feedback) → 3. Phase 2 MCTS refinement (300 iterations per blueprint) → 4. Route ranking by blueprint alignment

- **Design tradeoffs**: α parameter (0.5) balances strategy alignment vs. template popularity; higher α increases steering but may miss feasible routes; three Phase 1 attempts trades cost for diversity; template-based grounding avoids SMILES hallucination but limits expressiveness to library coverage

- **Failure signatures**: Premature termination (LLM proposes transformation with no template match); constraint violation (route passes validation script but fails implicit chemical feasibility); false negative evaluation (judge model misreads reaction SMILES, rejects valid route)

- **First 3 experiments**: 1. Reproduce USPTO-190 benchmark with single Phase 1 attempt and no MCTS to establish baseline solve rate (~47-51% per Figure 5); 2. Ablate α parameter (try 0.3, 0.7) on strategy-constrained benchmark to measure steering vs. solve rate tradeoff; 3. Test template coverage failure mode by restricting template library to 10k and measuring termination rate

## Limitations

- **LLM evaluation dependency**: The 95% success rate claims rely heavily on LLM-generated self-evaluation and feasibility judgments from Claude-4.5-Sonnet and Gemini-2.5-Pro, lacking experimental validation or external expert review

- **Template coverage constraints**: The template-based grounding mechanism may fail when LLM-proposed transformations fall outside the 40k-template coverage, though this failure mode is not extensively characterized

- **Generalizability concerns**: While natural language constraint incorporation works for tested examples, the framework's robustness to diverse chemical domains and chemically contradictory prompts remains unproven

## Confidence

- **High confidence**: The architectural framework combining iterative LLM planning with MCTS refinement is well-specified and demonstrates clear improvements over baseline methods (43-47% to 95% success rates)

- **Medium confidence**: The natural language constraint incorporation works as described for the specific examples tested, but generalizability to diverse chemical domains remains unproven

- **Medium confidence**: The feasibility awareness claims based on LLM self-evaluation and judge model scoring are internally consistent but lack external validation

## Next Checks

1. Conduct blind expert evaluation of Synthelite-generated routes against human-synthesized compounds to verify the feasibility scores claimed in the paper

2. Systematically characterize template coverage failure rates by attempting planning on diverse chemical classes and measuring premature termination frequency

3. Test the framework's robustness to chemically contradictory or highly specific constraints by designing prompt adversarial examples that probe the limits of LLM constraint-following ability
