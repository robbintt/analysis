# Rewriting the Budget: A General Framework for Black-Box Attacks Under Cost Asymmetry

## Quick Facts
- arXiv ID: 2506.06933
- Source URL: https://arxiv.org/abs/2506.06933
- Reference count: 40
- Key outcome: General framework for black-box adversarial attacks under asymmetric query costs, achieving up to 40% reduction in total query cost

## Executive Summary
This paper introduces a general framework for decision-based black-box adversarial attacks under asymmetric query costs, where queries of different types incur different costs. The authors propose two key modifications: Asymmetric Search (AS), a cost-aware search strategy that reduces reliance on high-cost queries by splitting intervals in proportion to the cost ratio, and Asymmetric Gradient Estimation (AGREST), which shifts the sampling region toward low-cost query space and reweights queries to improve gradient estimation efficiency. The framework is broadly compatible with existing black-box attacks such as HSJA, GeoDA, SurFree, and CGBA, requiring minimal structural changes.

## Method Summary
The proposed framework addresses cost asymmetry in black-box attacks by modifying two core components: the search strategy and gradient estimation. Asymmetric Search (AS) replaces uniform interval splitting with cost-proportional splitting, reducing the number of high-cost queries while maintaining attack effectiveness. Asymmetric Gradient Estimation (AGREST) improves gradient estimation by sampling more queries from the low-cost region and reweighting them appropriately. The framework is designed to be plug-and-play compatible with existing decision-based attacks, requiring only minimal modifications to their query generation and search procedures. The approach is theoretically grounded and validated across multiple attack methods and benchmark datasets.

## Key Results
- Up to 40% reduction in total query cost compared to baseline methods
- Consistent improvement across various cost ratios, including extreme cases (e.g., 1:100)
- Smaller adversarial perturbations compared to prior approaches, including stealthy attacks
- Framework works with multiple existing black-box attack methods (HSJA, GeoDA, SurFree, CGBA)
- Robust performance across different model architectures including CLIP

## Why This Works (Mechanism)
The framework exploits cost asymmetry by strategically reducing reliance on expensive queries while maintaining attack effectiveness. AS achieves this by splitting search intervals in proportion to the cost ratio, naturally favoring low-cost queries. AGREST further optimizes gradient estimation by shifting the sampling distribution toward low-cost regions and applying appropriate weighting. Together, these modifications allow the attack to achieve comparable or better results with fewer expensive queries, directly addressing the budget constraint problem.

## Foundational Learning
1. **Decision-based black-box attacks**: These attacks only observe the final decision (label) of the target model, not confidence scores or gradients. Understanding this constraint is crucial because it limits the information available for crafting adversarial examples.
2. **Query cost asymmetry**: Different types of queries (e.g., different models, different input transformations) can have vastly different computational costs. Recognizing this allows for optimization of the attack budget.
3. **Gradient estimation in high-dimensional spaces**: Black-box attacks often estimate gradients through finite differences or sampling. The quality of these estimates directly impacts attack success, making efficient estimation techniques critical.
4. **Bayesian Optimization principles**: The framework draws inspiration from Bayesian optimization techniques for efficient search in high-dimensional spaces under budget constraints.
5. **Multi-armed bandit problem**: The cost-aware search can be viewed as a variant of the multi-armed bandit problem, where different "arms" (query types) have different costs and rewards (information gain).
6. **Transferability of adversarial examples**: Understanding how adversarial examples crafted for one model perform on others is relevant for attacks using auxiliary models.

## Architecture Onboarding

**Component Map**: Input Image -> Attack Method (HSJA/GeoDA/SurFree/CGBA) -> Query Generator -> Model Query -> Decision Output -> Cost-aware Search (AS) + Gradient Estimation (AGREST) -> Adversarial Example

**Critical Path**: The critical path is the sequence from query generation through model evaluation to decision output, with AS and AGREST optimizations applied at the query generation stage to minimize expensive queries while maintaining attack effectiveness.

**Design Tradeoffs**: The framework trades off between query cost and attack success rate. AS reduces expensive queries but may increase total query count. AGREST improves gradient estimation quality but requires additional computation for weighting. The balance between these factors depends on the specific cost ratio and attack requirements.

**Failure Signatures**: 
- Poor attack success rate when cost ratio is extreme and AS splits are too aggressive
- Increased total query count when AGREST weighting is miscalibrated
- Degraded performance when combining AS and AGREST if theoretical assumptions don't hold
- Suboptimal results on models with different input/output characteristics (e.g., CLIP vs standard classifiers)

**First Experiments**:
1. Implement AS on HSJA with a 1:10 cost ratio on CIFAR-10 to verify basic functionality
2. Apply AGREST to GeoDA with a 1:100 cost ratio on ImageNet to test extreme asymmetry handling
3. Combine AS and AGREST on SurFree across multiple cost ratios to validate framework compatibility

## Limitations
- Theoretical analysis assumes ideal conditions that may not hold in practice when combining AS and AGREST
- Cost-benefit analysis for AGREST depends heavily on the specific cost ratio, with unclear optimal configurations for intermediate ratios
- The claim of compatibility with "all decision-based attacks" may overstate the generality, as some methods may require more substantial modifications

## Confidence
- High: The basic algorithmic modifications (AS and AGREST) are technically sound and well-explained
- Medium: The empirical results showing cost reductions are convincing, but the generalizability across diverse scenarios needs more validation
- Medium: The theoretical analysis provides useful insights but makes simplifying assumptions that may not hold in practice
