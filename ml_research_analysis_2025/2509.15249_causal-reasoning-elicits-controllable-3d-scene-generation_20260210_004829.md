# Causal Reasoning Elicits Controllable 3D Scene Generation

## Quick Facts
- **arXiv ID**: 2509.15249
- **Source URL**: https://arxiv.org/abs/2509.15249
- **Reference count**: 35
- **Primary result**: CausalStruct achieves CLIP scores of 25.90 and MLLM scores of 21.31, outperforming baseline methods in logical coherence and realistic spatial interactions

## Executive Summary
CausalStruct is a 3D scene generation framework that integrates causal reasoning to produce physically plausible and logically coherent multi-object scenes. The system uses large language models (LLMs) to construct causal graphs representing object relationships and dependencies, then applies causal intervention and PID control to refine object placement and scale. By leveraging 3D Gaussian Splatting and Score Distillation Sampling, the framework generates controllable 3D scenes that maintain consistency with textual descriptions and real-world physics. The method demonstrates superior performance compared to baseline approaches, particularly in handling complex spatial relationships and object interactions.

## Method Summary
CausalStruct processes text descriptions through an LLM to generate an initial scene graph, which is then refined using causal order determination and Bayesian edge estimation. The system applies causal intervention to validate uncertain relationships through MLLM-based scoring, followed by PID controller optimization to iteratively adjust object attributes within bounded limits. The refined layout serves as input for 3D Gaussian Splatting optimization using Score Distillation Sampling from both object-specific (MVDream/Zero123) and scene-level (Stable Diffusion) models. The framework balances computational efficiency with high-quality output, achieving state-of-the-art performance in text-to-3D scene generation.

## Key Results
- CLIP scores: ViT-B/32: 25.90, ViT-L/14: 20.86, ViT-bigL/14: 21.31
- MLLM scores: Gemini: 2.8, GPT-4o: 5.0, Claude: 3.3, Qwen: 6.9, GLM: 4.2
- Outperforms baseline methods in logical coherence and spatial interaction realism
- Demonstrates robust adaptability across diverse scene types

## Why This Works (Mechanism)
The framework's effectiveness stems from integrating causal reasoning into the 3D generation pipeline. By constructing causal graphs where nodes represent objects and edges encode dependencies, the system establishes logical object placement sequences that respect physical constraints. The PID controller provides stable iterative refinement of object attributes, while causal intervention validates relationships through MLLM scoring. The combination of 3D Gaussian Splatting for efficient rendering and Score Distillation Sampling for knowledge transfer from 2D diffusion models enables high-quality 3D output that maintains coherence with both textual descriptions and physical reality.

## Foundational Learning

- **Concept: Structural Causal Models (SCMs)**
  - **Why needed here:** CausalStruct uses SCMs to represent scene structure, with objects as nodes and causal relationships as edges. This enables inference of dependencies and application of interventions.
  - **Quick check question:** In a scene with "table" and "cup," how would an SCM represent their relationship, and which object would causally precede the other?

- **Concept: PID Control Loops**
  - **Why needed here:** The system employs PID control as an optimization engine for stable attribute adjustment. Understanding P, I, and D terms is essential for tuning and stability.
  - **Quick check question:** If object position oscillates wildly around target, which PID component (P, I, or D) is likely set too high?

- **Concept: Score Distillation Sampling (SDS)**
  - **Why needed here:** SDS lifts 2D knowledge from pretrained diffusion models into 3D, serving as the core technique for generating visual object appearance.
  - **Quick check question:** What is the primary source of knowledge for SDS, and how does it use this knowledge to train 3D representations?

## Architecture Onboarding

- **Component map:** Text Prompt -> LLM (Initial Graph) -> Causal Reasoning (Refined Graph) -> PID Control (Final Layout) -> 3DGS Optimization (SDS) -> Final 3D Scene
- **Critical path:** The sequential flow from text input through causal reasoning to final 3D optimization
- **Design tradeoffs:**
  - LLM/MLLM dependency creates quality bottlenecks but enables complex reasoning
  - Iterative PID control provides precision at computational cost
  - 3DGS chosen for efficiency over implicit smoothness of NeRFs
- **Failure signatures:**
  - Floating/misaligned objects indicate causal intervention or PID controller failures
  - Interpenetrating objects suggest incorrect causal order enforcement
  - Oscillating attributes signal poorly tuned PID controller or noisy MLLM scoring
- **First 3 experiments:**
  1. Reproduce ablation study by disabling Causal Reasoning module to observe coherence drop
  2. Systematically vary PID gains (Kp, Ki, Kd) and observe convergence effects on simple scenes
  3. Swap LLM backbone from GPT-4o to smaller distilled model and compare causal graph accuracy

## Open Questions the Paper Calls Out

- **Open Question 1:** How can the causal reasoning pipeline be adapted to maintain high spatial reasoning accuracy when using smaller, distilled models that currently struggle with edge establishment? The paper notes this limitation leads to increased randomness in spatial adjustments and proposes future fine-tuning as mitigation.

- **Open Question 2:** To what extent does the assumption of edge independence in Bayesian Edge Estimation degrade performance in highly complex scenes? The paper acknowledges residual dependencies exist but delegates their handling to the PID controller.

- **Open Question 3:** How sensitive is the PID controller's convergence to noise and inconsistency in MLLM scoring feedback? While the actuator dampens abrupt changes, the framework still relies on stochastic MLLM outputs for error calculations.

## Limitations
- Heavy dependence on LLM reasoning quality creates critical bottlenecks
- PID controller requires careful tuning that may not generalize across diverse scene types
- Framework complexity introduces multiple failure points that can compromise output quality

## Confidence

- **High confidence**: Core causal reasoning framework integration with 3D Gaussian Splatting is technically sound and validated through CLIP score improvements
- **Medium confidence**: PID controller implementation and convergence properties are reasonable but may require scene-specific tuning
- **Low confidence**: Generalizability to complex, real-world scenes beyond demonstrated examples remains uncertain

## Next Checks

1. **Cross-dataset robustness test**: Apply pipeline to scenes from multiple diverse datasets (indoor, outdoor, abstract) to evaluate performance consistency and identify failure modes across different scene types.

2. **Real-time capability assessment**: Measure inference time for complete pipeline on scenes of varying complexity to determine practical deployment feasibility and identify computational bottlenecks.

3. **Human perceptual validation**: Conduct user studies comparing CausalStruct outputs against baseline methods for physical plausibility and logical coherence to complement CLIP/MLLM metrics.
