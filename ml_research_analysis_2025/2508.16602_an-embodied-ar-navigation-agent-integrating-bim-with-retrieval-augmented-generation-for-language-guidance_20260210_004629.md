# An Embodied AR Navigation Agent: Integrating BIM with Retrieval-Augmented Generation for Language Guidance

## Quick Facts
- arXiv ID: 2508.16602
- Source URL: https://arxiv.org/abs/2508.16602
- Reference count: 40
- Primary result: Embodied AR navigation system integrating BIM with multi-agent RAG achieves SUS score of 80.5

## Executive Summary
This paper introduces an embodied AR navigation system that combines Building Information Modeling (BIM) with a multi-agent retrieval-augmented generation (RAG) framework to enable flexible, language-driven navigation in indoor environments. The system leverages three specialized language agents—Triage, Search, and Response—to interpret open-ended user queries and retrieve relevant spatial information from BIM data. Navigation guidance is delivered through an embodied AR agent equipped with voice interaction and locomotion, enhancing user experience and engagement.

The system was evaluated through a real-world user study, achieving a System Usability Scale (SUS) score of 80.5, indicating excellent usability. Comparative evaluations demonstrated that the embodied interface significantly improved users' perception of system intelligence compared to non-embodied alternatives, highlighting the importance of embodiment in AR navigation design.

## Method Summary
The system integrates BIM data with a multi-agent RAG framework consisting of Triage, Search, and Response agents to process natural language queries and retrieve spatial information. This information is then used to guide users through an embodied AR agent that provides voice interaction and locomotion-based navigation assistance. The architecture enables flexible interpretation of open-ended user queries in indoor environments.

## Key Results
- Achieved SUS score of 80.5, indicating excellent usability
- Embodied interface significantly improved user perception of system intelligence compared to non-embodied alternatives
- Successfully demonstrated integration of BIM with RAG for language-driven navigation

## Why This Works (Mechanism)
The system works by combining BIM's structured spatial data with RAG's ability to process and retrieve information from large language models. The three-agent architecture allows for specialized processing: Triage interprets query intent, Search retrieves relevant spatial data from BIM, and Response generates appropriate navigation guidance. The embodied AR agent then delivers this information through natural interaction modalities.

## Foundational Learning
- BIM Integration: Provides structured spatial data for navigation - needed for accurate indoor positioning and routing; quick check: verify BIM data completeness and accuracy
- Multi-agent RAG: Enables flexible query interpretation and information retrieval - needed to handle diverse user queries; quick check: test query understanding across different query types
- Embodied AR Interface: Enhances user engagement through natural interaction - needed for intuitive navigation guidance; quick check: validate voice and locomotion interaction effectiveness

## Architecture Onboarding
**Component Map:** User Query -> Triage Agent -> Search Agent -> Response Agent -> BIM Data -> Embodied AR Agent -> Navigation Guidance

**Critical Path:** User query processing through all three agents to BIM data retrieval and embodied AR delivery

**Design Tradeoffs:** Embodied interface increases engagement but requires more computational resources and hardware complexity compared to non-embodied alternatives

**Failure Signatures:** 
- Query misinterpretation at Triage stage
- Incomplete BIM data retrieval
- Embodied agent movement/positioning issues
- Voice recognition errors

**First Experiments:**
1. Test query interpretation accuracy across different user query types
2. Validate BIM data retrieval completeness and accuracy
3. Evaluate embodied agent navigation precision in controlled environment

## Limitations
- Evaluation limited to single real-world environment, limiting generalizability
- Lack of quantitative navigation performance metrics (accuracy, completion time)
- No long-term user adaptation or effectiveness studies
- Limited demographic and sample size information for user study

## Confidence
- High confidence in technical feasibility of BIM + RAG integration
- Medium confidence in usability findings due to limited user study context
- Low confidence in generalizability of embodiment benefits without objective task performance data
