# RAGFort: Dual-Path Defense Against Proprietary Knowledge Base Extraction in Retrieval-Augmented Generation

## Quick Facts
- arXiv ID: 2511.10128
- Source URL: https://arxiv.org/abs/2511.10128
- Reference count: 3
- Primary result: Dual-path defense framework reduces proprietary knowledge base extraction attacks by over 50% while maintaining answer accuracy

## Executive Summary
RAGFort introduces a comprehensive defense framework against proprietary knowledge base extraction attacks in retrieval-augmented generation systems. The system addresses two critical attack vectors: intra-class extraction targeting fine-grained information within specific topics, and inter-class extraction exploiting semantic relationships across topics. By combining contrastive reindexing for inter-class isolation with constrained cascade generation for intra-class protection, RAGFort achieves significant security improvements while preserving the utility of RAG systems for answering user queries.

The framework demonstrates that addressing both extraction paths simultaneously is essential for robust protection. Evaluation across three diverse datasets shows RAGFort reduces chunk recovery rates by over 50% compared to baseline defenses, with minimal impact on answer accuracy. The approach leverages topic-aware embeddings and a draft-verifier cascade architecture to create a practical defense that can be deployed in real-world RAG applications while maintaining computational efficiency.

## Method Summary
RAGFort employs a dual-path defense strategy combining contrastive reindexing and cascade generation to protect against proprietary knowledge base extraction. The inter-class protection uses HDBSCAN clustering and supervised contrastive learning to create topic-separable embeddings, effectively isolating semantically related chunks. The intra-class protection implements a draft-verifier cascade framework where a draft model generates candidate answers and a verifier model applies optimized rejection rules to suppress sensitive content. The system uses a quality threshold parameter (λ) to balance security and utility, with evaluation showing effective protection across three benchmark datasets while maintaining high answer accuracy.

## Key Results
- RAGFort achieves 0.51× relative mean CRR (chunk recovery rate) versus 0.91× for re-ranking baselines and 0.87× for summarization defenses
- Security improvements exceed 50% reduction in extraction attack success rates across all tested datasets
- Answer accuracy remains high with minimal degradation compared to unprotected RAG systems
- Computational efficiency is maintained despite the additional verification layer in the cascade architecture

## Why This Works (Mechanism)
The dual-path defense framework works by addressing both the semantic isolation of knowledge chunks and the controlled generation of responses. The contrastive reindexing component uses supervised contrastive learning to create embeddings that separate semantically related topics, preventing inter-class knowledge diffusion during retrieval. The cascade generation framework introduces a two-stage generation process where a draft model produces candidate answers and a verifier model applies rejection rules to suppress content that might reveal sensitive information, effectively preventing intra-class extraction while maintaining answer quality.

## Architecture Onboarding

**Component Map:** User Query -> Retriever -> Draft Model -> Verifier Model -> Final Answer
**Critical Path:** Query retrieval through contrastive reindexed embeddings, draft generation, quality verification and rejection
**Design Tradeoffs:** Security vs. utility tradeoff managed through quality threshold λ; computational overhead of double generation balanced against security benefits
**Failure Signatures:** High false rejection rates indicate overly aggressive verification; poor embedding separation suggests inadequate contrastive learning; low CRR reduction indicates ineffective defense

## Limitations
- Focus on single-turn QA tasks may not generalize to multi-turn conversational RAG systems
- Evaluation relies on synthetic attacks rather than real-world extraction attempts
- Computational overhead of draft-verifier cascade not benchmarked against production deployment constraints

## Confidence
- Effectiveness claims: High
- Computational efficiency: Medium
- Generalizability to other RAG tasks: Low
- Performance against adaptive attacks: Low
