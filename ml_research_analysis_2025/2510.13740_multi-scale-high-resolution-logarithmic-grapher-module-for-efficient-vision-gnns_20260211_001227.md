# Multi-Scale High-Resolution Logarithmic Grapher Module for Efficient Vision GNNs

## Quick Facts
- arXiv ID: 2510.13740
- Source URL: https://arxiv.org/abs/2510.13740
- Authors: Mustafa Munir; Alex Zhang; Radu Marculescu
- Reference count: 40
- Primary result: Introduces LSGC for Vision GNNs with logarithmic scaling, reducing computational complexity while preserving long-range connections

## Executive Summary
This paper introduces Logarithmic Scalable Graph Construction (LSGC), a novel graph construction method designed to improve Vision GNNs (ViGs) by reducing computational complexity while preserving long-range connections. LSGC scales logarithmically based on image bit-depth, avoiding over-squashing issues in high-resolution images and outperforming existing approaches like k-nearest neighbors (KNN) and Sparse Vision Graph Attention (SVGA). The authors propose LogViG, a hybrid CNN-GNN architecture that incorporates LSGC and a high-resolution shortcut to achieve superior efficiency and performance on vision tasks.

## Method Summary
The Logarithmic Scalable Graph Construction (LSGC) method constructs graphs for Vision GNNs by connecting pixels based on logarithmic distance scaling relative to image bit-depth. This approach reduces the number of edges while maintaining essential long-range connections between distant pixels. The method is implemented within a hybrid CNN-GNN architecture called LogViG, which combines convolutional layers with graph neural network processing. The high-resolution shortcut connection helps preserve fine-grained spatial information that might otherwise be lost in the graph construction process. The logarithmic scaling ensures that computational complexity grows sublinearly with image resolution, addressing the over-squashing problem common in high-resolution vision tasks.

## Key Results
- LSGC demonstrates logarithmic scaling behavior based on image bit-depth, reducing computational complexity compared to traditional methods
- Outperforms KNN and SVGA in preserving long-range connections while maintaining efficiency
- LogViG architecture with LSGC achieves superior performance on standard vision benchmarks while using fewer computational resources

## Why This Works (Mechanism)
LSGC works by constructing graphs that prioritize connections between pixels based on logarithmic distance scaling rather than fixed distance thresholds. This approach ensures that long-range dependencies are preserved even in high-resolution images where traditional methods would either create prohibitively dense graphs or miss important distant relationships. The logarithmic scaling adapts to the bit-depth of images, maintaining consistent performance across different resolutions. By reducing unnecessary short-range connections while preserving critical long-range ones, LSGC avoids the over-squashing problem where information gets bottlenecked in dense graph structures.

## Architecture Onboarding

Component map:
Input image -> CNN feature extraction -> LSGC graph construction -> GNN layers -> Output prediction
^------------ High-resolution shortcut -------------------^

Critical path:
The core processing path involves feature extraction from the CNN backbone, followed by LSGC graph construction, and then GNN processing layers. The high-resolution shortcut bypasses the graph construction to preserve fine-grained spatial details.

Design tradeoffs:
The main tradeoff involves balancing graph density with computational efficiency. LSGC reduces edges compared to fully connected approaches but must maintain sufficient connectivity for effective message passing. The logarithmic scaling provides a principled way to make this tradeoff adaptive to image resolution.

Failure signatures:
Potential failures include: (1) Insufficient graph connectivity leading to poor long-range information propagation, (2) Oversimplified graph structures that lose important local patterns, (3) Performance degradation on very low-resolution images where logarithmic scaling may be excessive, (4) Suboptimal bit-depth parameter selection affecting graph quality.
