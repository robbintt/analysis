# Understanding GUI Agent Localization Biases through Logit Sharpness

## Quick Facts
- **arXiv ID**: 2506.15425
- **Source URL**: https://arxiv.org/abs/2506.15425
- **Reference count**: 9
- **Primary result**: Introduced Peak Sharpness Score (PSS) metric and Context-Aware Cropping technique to address GUI agent localization hallucinations, showing consistent accuracy improvements across models

## Executive Summary
This paper addresses a critical reliability issue in GUI agents: systematic localization errors or "hallucinations" that occur when models incorrectly identify UI elements. The authors develop a fine-grained evaluation framework that categorizes model predictions into four types (correct, biased hallucination, misleading hallucination, and confusion hallucination) and introduce the Peak Sharpness Score (PSS) as a metric that quantifies the alignment between semantic continuity and logits distribution. They also propose Context-Aware Cropping, a training-free technique that adaptively refines input context to improve localization accuracy. Experimental results demonstrate that PSS effectively distinguishes between hallucination types and that Context-Aware Cropping consistently improves performance across various GUI agent models, with ShowUI-2B showing a 3.9% accuracy improvement from 75.1% to 79.0%.

## Method Summary
The paper introduces a multi-faceted approach to understanding and mitigating localization biases in GUI agents. The core innovation is the Peak Sharpness Score (PSS), which measures how well a model's logits distribution aligns with the semantic continuity of the GUI layout by analyzing the sharpness and concentration of probability distributions over coordinates. The four-category hallucination framework provides a systematic way to classify errors based on their semantic relationship to the ground truth. Context-Aware Cropping is implemented as a training-free inference-time technique that adaptively crops and refines the input context to focus the model's attention on relevant regions. The methodology is validated across multiple GUI agent models using a test set of 15 images from 8 software applications, measuring improvements in localization accuracy and confidence calibration.

## Key Results
- PSS effectively distinguishes between different hallucination types, providing task-relevant confidence measures
- Context-Aware Cropping improved ShowUI-2B accuracy from 75.1% to 79.0%
- Cropping technique showed consistent improvements across all hallucination categories, with biased hallucinations showing the most pronounced gains
- The four-category hallucination framework provides fine-grained error analysis beyond binary correct/incorrect classification

## Why This Works (Mechanism)
The approach works by addressing two fundamental issues in GUI agent localization: confidence miscalibration and context misalignment. PSS captures the relationship between semantic continuity in GUI layouts and the model's internal confidence representation through logits sharpness, which serves as a proxy for how well the model understands the spatial relationships between UI elements. Context-Aware Cropping improves performance by eliminating irrelevant visual information and focusing the model's attention on the most salient regions, reducing the likelihood of the model being distracted by peripheral elements. The combination addresses both the model's internal representation issues and the quality of the input signal.

## Architecture Onboarding

**Component map**: GUI input -> Context-Aware Cropping -> GUI Agent Model -> Logits Distribution -> Peak Sharpness Score -> Prediction + Confidence Score

**Critical path**: The most important sequence is Context-Aware Cropping feeding into the GUI Agent Model, as the cropping directly influences the quality of the logits distribution that PSS analyzes. The PSS calculation itself depends on the complete logits distribution from the model.

**Design tradeoffs**: The training-free approach maximizes applicability but may miss opportunities for deeper integration that could yield larger gains. The fine-grained evaluation framework provides detailed insights but requires more complex analysis compared to simple accuracy metrics. The adaptive cropping adds inference-time overhead but improves accuracy without model retraining.

**Failure signatures**: PSS values that are high but predictions are incorrect suggest confidence miscalibration. Uniform logits distributions indicate model confusion or lack of discriminative features. Context-Aware Cropping that removes relevant information can degrade performance. Limited improvement on certain hallucination types may indicate fundamental model limitations.

## Limitations
- Evaluation dataset limited to 15 test images from 8 software applications, potentially limiting generalizability
- Context-Aware Cropping computational overhead during inference not quantified
- PSS correlation with downstream task performance beyond localization accuracy not established
- Multi-step GUI agent workflows and error propagation not addressed

## Confidence

**High confidence**: The four-category hallucination framework and its empirical validation; the effectiveness of Context-Aware Cropping in improving localization accuracy

**Medium confidence**: The theoretical justification for PSS as a confidence measure; generalizability of findings across diverse GUI contexts

**Low confidence**: Claims about PSS's ability to predict long-term reliability in sequential GUI tasks; the method's robustness to unseen application types
