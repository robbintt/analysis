# Mobile-Agent-RAG: Driving Smart Multi-Agent Coordination with Contextual Knowledge Empowerment for Long-Horizon Mobile Automation

## Quick Facts
- **arXiv ID:** 2511.12254
- **Source URL:** https://arxiv.org/abs/2511.12254
- **Authors:** Yuxiang Zhou; Jichang Li; Yanhao Zhang; Haonan Lu; Guanbin Li
- **Reference count:** 40
- **Primary result:** Hierarchical multi-agent framework with dual-level retrieval augmentation improves long-horizon mobile automation task completion by 11.0% and step efficiency by 10.2%

## Executive Summary
Mobile-Agent-RAG introduces a hierarchical multi-agent framework designed to tackle complex, long-horizon mobile automation tasks through contextual knowledge empowerment. The system employs Manager-RAG for high-level strategic planning and Operator-RAG for low-level UI action execution, each supported by specialized knowledge bases. This dual retrieval augmentation approach addresses the critical challenge of hallucination and execution errors in mobile automation by providing context-aware knowledge integration at both strategic and operational levels.

The framework demonstrates significant performance improvements over state-of-the-art baselines, achieving 11.0% better task completion rates and 10.2% improvement in step efficiency. By decomposing complex automation tasks into manageable subtasks and leveraging domain-specific knowledge, Mobile-Agent-RAG provides a scalable solution for cross-app mobile automation that reduces errors and improves overall task success rates.

## Method Summary
Mobile-Agent-RAG implements a hierarchical multi-agent architecture with dual retrieval augmentation mechanisms. The system consists of Manager-RAG for high-level strategic planning and Operator-RAG for low-level UI action execution, each equipped with specialized knowledge bases. The framework processes long-horizon mobile automation tasks by first breaking them down into manageable subtasks, then using Manager-RAG to develop strategic plans while Operator-RAG handles specific UI interactions. The retrieval augmentation component provides context-aware knowledge integration at both levels, significantly reducing hallucinations and execution errors common in traditional mobile automation approaches.

## Key Results
- Achieved 11.0% improvement in task completion rate over state-of-the-art baselines
- Demonstrated 10.2% increase in step efficiency for mobile automation tasks
- Successfully handled 12 specific long-horizon, cross-app automation tasks with reduced hallucinations and execution errors

## Why This Works (Mechanism)
The hierarchical structure with dual retrieval augmentation addresses the complexity of long-horizon mobile automation by separating strategic planning from tactical execution. Manager-RAG provides high-level task decomposition and planning while Operator-RAG handles the specific UI interactions, with both components drawing from specialized knowledge bases to maintain context and reduce errors. This separation of concerns, combined with knowledge-driven context awareness, enables more reliable execution of complex multi-step automation tasks across different applications.

## Architecture Onboarding

**Component Map:** Mobile-Agent-RAG -> Manager-RAG (Strategic Planning) -> Operator-RAG (UI Execution) -> Specialized Knowledge Bases (Contextual Support)

**Critical Path:** Task Input → Manager-RAG Strategic Planning → Operator-RAG UI Execution → Task Completion, with retrieval augmentation supporting both planning and execution phases

**Design Tradeoffs:** The hierarchical approach trades increased system complexity for improved accuracy and reliability, while specialized knowledge bases provide better context but require ongoing maintenance and updates

**Failure Signatures:** Knowledge base incompleteness leading to planning errors, UI element recognition failures causing execution breakdowns, and strategic misalignment between Manager-RAG and Operator-RAG components

**3 First Experiments:**
1. Single-app automation task execution to validate basic functionality
2. Multi-app task automation to test cross-application coordination
3. Task complexity scaling to measure performance under increasing complexity

## Limitations
- Generalizability across diverse mobile operating systems and app ecosystems remains uncertain
- Performance metrics lack detailed baseline comparisons for proper context
- Specialized knowledge base maintenance overhead and potential knowledge drift over time not fully addressed

- **Low confidence** in long-term scalability and maintenance requirements of specialized knowledge base approach
