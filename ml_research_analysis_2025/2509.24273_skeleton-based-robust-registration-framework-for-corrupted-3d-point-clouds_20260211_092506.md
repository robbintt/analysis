# Skeleton-based Robust Registration Framework for Corrupted 3D Point Clouds

## Quick Facts
- arXiv ID: 2509.24273
- Source URL: https://arxiv.org/abs/2509.24273
- Authors: Yongqiang Wang; Weigang Li; Wenping Liu; Zhiqiang Tian; Jinling Li
- Reference count: 40
- Primary result: Skeleton-based framework achieves significantly lower MSE, RMSE, and MAE than state-of-the-art methods on corrupted 3D point clouds

## Executive Summary
This paper introduces a Skeleton-based Robust Registration Framework (SRRF) that addresses the challenge of aligning corrupted 3D point clouds affected by density variations, noise contamination, and geometric deformations. The framework leverages skeletal representations, which are more stable against corruptions than raw point clouds, to improve registration accuracy. By extracting skeletal structures from both source and target point clouds and using an adaptive weighting scheme based on inlier ratios, SRRF achieves state-of-the-art performance on a constructed corrupted dataset.

## Method Summary
SRRF extracts skeletal structures from corrupted point clouds using a Skeleton Extraction Module (PointNet++ encoder with convex combination), then processes both raw and skeleton data through structural feature extractors and point feature interaction modules. Two transformations are estimated - one from the corrupted point clouds and one from their skeletons - and combined using adaptive weighting based on inlier ratios. A distribution distance loss ensures skeleton consistency. The framework is trained end-to-end with Adam optimizer on synthetic corrupted versions of ModelNet40 dataset.

## Key Results
- SRRF achieves significantly lower MSE, RMSE, and MAE compared to state-of-the-art methods across various corruption types
- Distribution distance loss reduces Chamfer distance from 3.85 to 1.83, improving skeleton consistency
- The framework shows particular effectiveness against Gaussian noise and density variations, though slightly higher errors occur with cutout corruption

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Skeletal representations provide a more stable geometric reference for registration than raw point clouds
- Core assumption: SEM generates topologically consistent skeletons even with density distortions, noise contamination, and geometric deformations
- Evidence: Abstract states skeletons leverage "stability against corruptions"; section 4.2 notes skeleton points are "less susceptible to noise and distortions"
- Break condition: Mechanism fails if corruption is so severe (e.g., massive cutouts) that SEM cannot infer underlying topology

### Mechanism 2
- Claim: Distribution distance loss between source and target skeletons improves registration by ensuring geometric consistency
- Core assumption: Consistent "ground truth" skeleton shape exists that both source and target should approximate despite different corruptions
- Evidence: Section 4.6 describes loss as enforcing "consistency between source and target skeletons"; Table 5 shows Chamfer distance reduction from 3.85 to 1.83
- Break condition: Fails if source and target have non-rigid differences or significant partial overlaps

### Mechanism 3
- Claim: Adaptive fusion of global skeletal alignment with local point cloud alignment achieves robustness
- Core assumption: Inlier ratio is valid proxy for transformation reliability in current corruption context
- Evidence: Abstract mentions "adaptive weighting scheme based on inlier ratios"; section 4.5 explains higher inlier ratio gets greater weight
- Break condition: Breaks if inlier ratio calculation itself is corrupted by noise creating false high-confidence matches

## Foundational Learning

- Why needed: Framework estimates rotation matrix R and translation vector t to align point clouds
- Quick check: Can you derive the rotation matrix R given a set of corresponding points using Singular Value Decomposition (SVD)?

**Medial Axis Transform (MAT)**
- Why needed: Paper's "Skeleton" is based on MAT concepts (centers of maximally inscribed spheres)
- Quick check: How does the Medial Axis differ from a simple centerline, and why is it sensitive to surface noise without regularization?

- Why needed: Geometric Matching module uses probabilistic soft map rather than hard 1-to-1 assignment

## Architecture Onboarding

**Component map:**
Input: Source (X) & Target (Y) Corrupted Point Clouds → SEM (PointNet++ Encoder → Convex Combination → Skeleton Points (X_s, Y_s)) → Feature Branch (SFE + PFI processes both Raw and Skeleton data) → Head (Soft Matching → Differentiable SVD → Transformation (T_s for skeleton, T_c for points)) → Integration (Adaptive Fusion based on inlier ratio λ)

**Critical path:** Skeleton Extraction Module (SEM) is the critical novelty; if SEM outputs garbage, fusion mechanism loses robustness benefit

**Design tradeoffs:** Trades computational efficiency (running two parallel registration branches + feature interaction) for robustness against corruption; trades strict point-level accuracy for topological stability via skeleton

**Failure signatures:**
- Cutout Corruption: Table 1 shows slightly higher rotation errors for "Cutout" vs. "Density_inc", suggesting struggles when structural geometry is physically missing
- Extreme Non-Rigid Deformation: Conclusion explicitly states method may face challenges here as rigid skeletal consistency assumption breaks

**First 3 experiments:**
1. Ablation on SEM: Run registration using only skeleton branch vs. only raw point branch to isolate performance gain specific to skeletal topology on "Gaussian" noise dataset
2. Loss Function sensitivity: Visualize skeleton outputs with and without L_ddl to verify "consistency" claim visually (as hinted in Fig. 11)
3. Fusion Weight Analysis: Plot adaptive weight λ over epochs for different corruption types (e.g., Noise vs. Density) to confirm system shifts reliance to skeleton when noise is high

## Open Questions the Paper Calls Out
- Can integrating learning-based deformation estimation into SRRF improve its adaptability to extreme non-rigid deformations?
- How can adaptive weighting strategies be developed to handle heterogeneous corruptions where distortion levels vary significantly across different regions?
- Does the framework maintain robustness when applied to large-scale, unstructured outdoor scenes rather than object-level CAD models?

## Limitations
- Struggles with cutout corruption where geometric topology is physically missing, preventing valid skeleton extraction
- Cannot handle extreme non-rigid deformations as rigid skeletal consistency assumption breaks
- Assumes relatively uniform corruption levels across the point cloud, limiting effectiveness for heterogeneous corruptions

## Confidence

**High Confidence:** Core mechanism of using skeletal representations for geometric stability is well-supported by experimental results showing consistent improvements across corruption types

**Medium Confidence:** Adaptive fusion approach is logically sound and supported by ablation results, though inlier ratio reliability as weighting proxy in extreme conditions needs further validation

**Low Confidence:** Effectiveness of distribution distance loss for skeleton consistency has minimal direct support in corpus and relies primarily on internal ablation results
