# SAFE-QAQ: End-to-End Slow-Thinking Audio-Text Fraud Detection via Reinforcement Learning

## Quick Facts
- arXiv ID: 2601.01392
- Source URL: https://arxiv.org/abs/2601.01392
- Reference count: 25
- End-to-end audio-text fraud detection framework processing raw audio directly, achieving 88.23 F1 score on TeleAntiFraud-Bench while reducing reasoning chain lengths by 48.87%

## Executive Summary
SAFE-QAQ introduces an innovative end-to-end audio-text fraud detection framework that processes raw audio directly, eliminating transcription errors from automatic speech recognition systems. The system employs rule-based reinforcement learning to develop slow-thinking reasoning capabilities, systematically analyzing fine-grained audio details such as tone, pauses, and environmental cues that fraudsters often conceal. Through intelligent rejection sampling and length-constrained reinforcement learning, SAFE-QAQ achieves significant improvements in inference speed and reasoning efficiency while maintaining high accuracy. Deployed in production analyzing over 70,000 calls daily, the framework demonstrates practical effectiveness in reducing manual audit burdens and preventing financial losses through timely automated intervention.

## Method Summary
SAFE-QAQ operates through a novel end-to-end architecture that bypasses traditional ASR transcription by directly processing raw audio signals alongside text data. The system implements rule-based reinforcement learning to develop "slow-thinking" reasoning capabilities, systematically breaking down complex fraud detection tasks into smaller, analyzable components. Through rejection sampling techniques and length-constrained RL, the model optimizes reasoning chains while maintaining accuracy. The framework's design enables it to capture subtle audio cues like emotional tones, speech patterns, and environmental sounds that are often overlooked by conventional detection systems, making it particularly effective at identifying sophisticated fraud attempts.

## Key Results
- Achieves 88.23 F1 score on TeleAntiFraud-Bench dataset, demonstrating strong detection accuracy
- Reduces reasoning chain lengths by 48.87% through reinforcement learning-based rejection sampling
- Provides 26.34% faster inference compared to baseline models while maintaining detection performance

## Why This Works (Mechanism)
SAFE-QAQ's effectiveness stems from its direct raw audio processing approach, which eliminates the error propagation inherent in ASR transcription pipelines. The rule-based reinforcement learning mechanism enables systematic "slow-thinking" analysis by decomposing complex fraud detection into manageable subtasks, allowing the system to identify subtle patterns that human auditors might miss. The rejection sampling optimization ensures that the model maintains high accuracy while significantly reducing computational overhead, making real-time deployment feasible at scale.

## Foundational Learning
- **Raw audio processing vs ASR**: Direct audio signal analysis eliminates transcription errors but requires more complex signal processing; verify by comparing detection accuracy with and without ASR intermediary
- **Rule-based reinforcement learning**: Combines domain expertise with adaptive learning to create interpretable reasoning chains; check by auditing generated reasoning steps for logical consistency
- **Rejection sampling optimization**: Balances computational efficiency with accuracy by discarding suboptimal reasoning paths; validate by measuring performance degradation when disabled
- **Slow-thinking reasoning**: Deliberate, systematic analysis of multiple audio cues rather than quick pattern matching; test by comparing detection of subtle fraud patterns against baseline models
- **Multi-modal feature fusion**: Integration of audio and text features enables comprehensive fraud detection; verify by evaluating performance when using single modalities
- **Length-constrained reasoning**: Optimizes reasoning depth while preventing overfitting to noise; validate by testing performance across different reasoning depth constraints

## Architecture Onboarding

Component Map:
Audio Signal Processing -> Multi-modal Feature Extraction -> Rule-based RL Engine -> Reasoning Chain Optimization -> Fraud Classification

Critical Path:
Raw audio input → Signal preprocessing → Feature extraction (audio+text) → Rule-based RL reasoning → Reasoning chain optimization → Final classification

Design Tradeoffs:
- Direct audio processing vs ASR transcription accuracy vs computational complexity
- Rule-based reasoning vs pure neural approaches for interpretability vs flexibility
- Reasoning depth optimization vs detection accuracy
- Real-time processing requirements vs thorough analysis depth

Failure Signatures:
- False negatives when fraudsters mimic legitimate audio patterns
- Performance degradation with poor audio quality or background noise
- Potential overfitting to domain-specific fraud patterns
- Computational bottlenecks under high call volume

First Experiments:
1. Benchmark detection accuracy on controlled dataset with varying audio quality levels
2. A/B test rule-based RL reasoning vs traditional neural approaches on same dataset
3. Stress test system under simulated production load to identify performance bottlenecks
