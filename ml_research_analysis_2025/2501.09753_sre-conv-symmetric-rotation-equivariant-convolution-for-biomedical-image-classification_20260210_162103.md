# SRE-Conv: Symmetric Rotation Equivariant Convolution for Biomedical Image Classification

## Quick Facts
- arXiv ID: 2501.09753
- Source URL: https://arxiv.org/abs/2501.09753
- Reference count: 0
- Achieves superior classification accuracy on rotated biomedical images using symmetric rotation-equivariant convolution

## Executive Summary
SRE-Conv introduces a novel convolution operation for biomedical image classification that achieves rotation equivariance without requiring data augmentation. The method parameterizes convolutional kernels using symmetric bands around the center, significantly reducing the number of trainable parameters while maintaining computational efficiency. When evaluated on 16 MedMNISTv2 tasks, SRE-Conv-CNN demonstrated superior performance on rotated test sets compared to standard convolutions and other rotation-equivariant methods, using 70-90% fewer parameters.

## Method Summary
SRE-Conv operates by representing convolutional kernels as symmetric bands around the center pixel, creating a parameter-efficient representation that maintains rotational equivariance. This symmetric band parameterization reduces the trainable parameters from O(C·k²) to O(C·k) where C is the number of channels and k is the kernel size. The method maintains computational complexity while achieving equivariance to rotations through this structured kernel representation. Unlike traditional approaches that rely on data augmentation for rotation robustness, SRE-Conv learns rotation-equivariant features directly through its kernel architecture.

## Key Results
- SRE-Conv-CNN achieved superior classification accuracy on rotated test sets compared to baseline methods
- Parameter reduction of 70-90% compared to standard convolutional layers
- Maintained rotational equivariance without requiring data augmentation
- Evaluated across 16 MedMNISTv2 biomedical image classification tasks

## Why This Works (Mechanism)
The symmetric band parameterization of convolutional kernels enables rotation equivariance by construction. When a kernel is rotated, the symmetric arrangement ensures that the same features are detected regardless of orientation. This structural approach to equivariance is more efficient than data augmentation or complex group convolutions, as it encodes rotational symmetry directly into the kernel representation. The method effectively learns rotation-invariant features by constraining the kernel space to symmetric configurations, which naturally respond consistently to rotated inputs.

## Architecture Onboarding

**Component Map**: Input Image -> Symmetric Rotation Equivariant Convolution -> Feature Extraction -> Classification Head -> Output

**Critical Path**: The symmetric band parameterization is the core innovation. SRE-Conv layers replace standard convolutions in a CNN architecture, where the symmetric kernel representation enables rotation-equivariant feature learning. The classification head remains standard but benefits from rotation-robust features.

**Design Tradeoffs**: The method trades off some representational flexibility for parameter efficiency and equivariance. While standard convolutions can learn any kernel shape, SRE-Conv constrains kernels to symmetric bands, potentially missing some orientation-specific features. However, this constraint is precisely what enables rotation equivariance and parameter savings.

**Failure Signatures**: Performance degradation on non-rotated images compared to standard convolutions, as the symmetric constraint may limit learning of orientation-specific features. Limited generalization to extreme rotation angles (>90 degrees) or combined transformations (rotation + scaling).

**First Experiments**: 1) Train SRE-Conv-CNN on MedMNISTv2 with rotated training data; 2) Compare parameter counts between SRE-Conv and standard convolutions; 3) Evaluate classification accuracy on rotated test sets across different rotation angles.
