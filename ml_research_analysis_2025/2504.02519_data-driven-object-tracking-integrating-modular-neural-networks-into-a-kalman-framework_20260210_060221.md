# Data-Driven Object Tracking: Integrating Modular Neural Networks into a Kalman Framework

## Quick Facts
- arXiv ID: 2504.02519
- Source URL: https://arxiv.org/abs/2504.02519
- Reference count: 40
- Key outcome: Modular tracking framework with <50k parameter NNs achieves 50% RMSE reduction in trajectory prediction and 95% association accuracy in ADAS

## Executive Summary
This paper presents a modular tracking framework that integrates specialized neural networks into a Kalman Filter framework for multi-object tracking in ADAS. The approach addresses challenges in trajectory prediction and sensor object-to-track association by developing three neural networks: SPENT for trajectory prediction, SANT for single sensor object-to-track mapping, and MANTa for multiple associations. Each network contains fewer than 50k trainable parameters and is designed for real-time embedded inference. The evaluation on KITTI demonstrates significant performance improvements while preserving modularity and interpretability.

## Method Summary
The framework integrates three neural networks into a Kalman Filter structure for multi-object tracking. SPENT uses LSTM to predict object trajectories, replacing the standard linear motion model. SANT and MANTa use BiLSTMs to solve data association as a classification problem, eliminating the need for hand-crafted distance thresholds. The system processes normalized state vectors (position, velocity, yaw) from KITTI tracking data, with pre-padding and sorting of sequences to handle variable lengths. Training uses MSE loss for SPENT and cross-entropy for the association networks, with a 562/31/31 train/validation/test split on 624 filtered tracks.

## Key Results
- SPENT reduces RMSE by 50% compared to standard Kalman Filter (0.029 vs 0.066)
- SANT and MANTa achieve up to 95% accuracy in sensor object-to-track assignments
- MANTa performance drops significantly (>6 simultaneous tracks) to 14% accuracy
- All networks contain <50k parameters suitable for real-time embedded inference

## Why This Works (Mechanism)

### Mechanism 1
- **Claim:** Replacing linear motion model with LSTM captures non-linear dynamics
- **Mechanism:** SPENT uses LSTM hidden states as memory for object history, learning to weight past states for future predictions
- **Core assumption:** KITTI training data contains learnable temporal patterns that generalize to ADAS scenarios
- **Evidence anchors:** 50% RMSE reduction vs KF; self-learning approach enables handling non-linearities
- **Break condition:** Performance degrades with highly erratic motion unseen in training

### Mechanism 2
- **Claim:** BiLSTM classification eliminates need for hand-crafted distance thresholds
- **Mechanism:** SANT/MANTa ingest track and sensor object state vectors simultaneously, using BiLSTM to contextualize before outputting association probabilities
- **Core assumption:** Association logic can be inferred from raw state vectors without explicit distance matrices
- **Evidence anchors:** 95% accuracy in assignments; eliminates predefined distance matrix requirement
- **Break condition:** Fails with significant sensor noise shift from training or in swarm scenarios with nearly identical object states

### Mechanism 3
- **Claim:** <50k parameter constraint forces efficient representations for embedded deployment
- **Mechanism:** Compact layer sizes minimize parameters, trading representational power for speed and lower memory footprint
- **Core assumption:** ADAS tracking complexity can be compressed into small neural networks without fidelity loss
- **Evidence anchors:** Designed for real-time embedded environment; preserves modularity and interpretability
- **Break condition:** Underfitting occurs if sensor input dimensionality increases beyond training scope

## Foundational Learning

- **Long Short-Term Memory (LSTM) & BiLSTM**
  - **Why needed here:** LSTMs maintain "hidden states" as memory for object trajectories; understanding gate mechanisms helps diagnose SPENT failures
  - **Quick check question:** How does SPENT LSTM "hidden state" differ from standard Kalman Filter "state vector" in terms of observability?

- **Tracking-by-Detection (TbD) Pipeline**
  - **Why needed here:** System architecture is modular; understanding sequential flow (Detection → Prediction → Association → Update) is required to know where to integrate networks
  - **Quick check question:** Does SPENT replace prediction, update, or both steps of the Kalman Filter?

- **One-Hot Encoding & Softmax Classification**
  - **Why needed here:** SANT/MANTa frame tracking as classification problem; understanding softmax probabilities is essential for tuning association threshold
  - **Quick check question:** If MANTa outputs softmax vector of size 18 for a track, what does index 17 typically represent?

## Architecture Onboarding

- **Component map:** Preprocessing (Normalization + Pre-padding) → SPENT (LSTM→BatchNorm→ReLU→Dropout→FC) → Association (SANT/MANTa: BiLSTM→FC→Softmax) → Track Management

- **Critical path:** Pre-padding and sorting of data is critical failure point; unsorted data leads to excessive noise and prevents convergence

- **Design tradeoffs:** SANT processes 1-to-N associations (simpler, more robust); MANTa processes N-to-N (faster for batches but accuracy drops significantly with >6 tracks)

- **Failure signatures:** MANTa sudden accuracy drop in crowded scenes (>6 tracks); SPENT divergence with normalization shift; identity switching from hard classification vs probabilistic approaches

- **First 3 experiments:**
  1. Baseline RMSE comparison: SPENT vs Standard KF on KITTI test split
  2. Association stress test: MANTa evaluation on 8+ simultaneous object subsets
  3. Noise robustness analysis: SPENT/MANTa performance with 5-10% synthetic noise vs 3% training noise

## Open Questions the Paper Calls Out

- **Open Question 1:** How does performance generalize to diverse autonomous driving datasets beyond KITTI?
  - Basis: Explicit listing of testing on other datasets as primary future research topic
  - Why unresolved: Evaluation restricted to KITTI (cars/vans only)
  - What evidence would resolve it: Reporting RMSE and accuracy on nuScenes, Waymo Open Dataset

- **Open Question 2:** Can uncertainty quantification methods improve safety and reliability?
  - Basis: Explicit identification of uncertainty quantification as future research
  - Why unresolved: Current outputs are deterministic point estimates without confidence intervals
  - What evidence would resolve it: Bayesian neural networks showing correlation between high uncertainty and tracking failures

- **Open Question 3:** How can MANTa be improved for high-density scenarios (>6 tracks)?
  - Basis: 95% accuracy for 1-6 tracks vs 14% for 7-16 tracks due to dataset imbalance
  - Why unresolved: Severe performance drop suggests architecture or training data limitations
  - What evidence would resolve it: Sustained >90% accuracy on 7+ track sequences through architectural scaling or targeted augmentation

- **Open Question 4:** Does multitasking network architecture provide advantages over modular approach?
  - Basis: Proposal to investigate multitasking networks for context-dependent tracking
  - Why unresolved: Current system treats prediction and association as separate modules
  - What evidence would resolve it: Comparative study between modular setup and unified network model

## Limitations
- Performance claims hinge on underspecified architecture details (<50k params mentioned but exact dimensions not provided)
- Evaluation restricted to KITTI cars and vans, leaving generalization to pedestrians/cyclists untested
- Synthetic association data generation method described but implementation details unclear

## Confidence

- **High confidence:** RMSE improvement (50%) for SPENT vs standard KF is well-supported
- **Medium confidence:** 95% association accuracy claims credible but MANTa's 14% drop for >6 tracks reveals clear limitation
- **Low confidence:** Generalization claims to broader ADAS scenarios not substantiated beyond KITTI car/van domain

## Next Checks
