# NavMapFusion: Diffusion-based Fusion of Navigation Maps for Online Vectorized HD Map Construction

## Quick Facts
- **arXiv ID**: 2512.03317
- **Source URL**: https://arxiv.org/abs/2512.03317
- **Reference count**: 39
- **Primary result**: 21.4% relative improvement in mAP for HD map construction using diffusion-based fusion with navigation map priors

## Executive Summary
NavMapFusion introduces a novel diffusion-based framework for online vectorized HD map construction that leverages navigation map priors. The key innovation lies in using diffusion models to naturally interpret discrepancies between prior maps and sensor observations as noise, enabling effective fusion of coarse navigation map information with high-fidelity sensor data. By incorporating both Spatial Cross-Attention for sensor features and Map Cross-Attention for prior map embeddings within a denoising diffusion probabilistic model, the approach achieves state-of-the-art performance on the nuScenes benchmark while maintaining real-time capabilities.

## Method Summary
NavMapFusion employs a denoising diffusion probabilistic model that conditions on navigation map priors (such as OpenStreetMap) to guide the construction of vectorized HD maps from sensor data. The framework uses a U-Net architecture with attention mechanisms to fuse sensor observations and map priors through an iterative denoising process. Spatial Cross-Attention integrates LiDAR and camera features with map information, while Map Cross-Attention embeds the prior map structure into the latent space. The model is trained to predict and correct discrepancies between the prior map and sensor observations, treating these differences as noise to be removed through the diffusion process. This approach allows for incremental map refinement as more sensor data becomes available.

## Key Results
- Achieves 21.4% relative improvement in mean Average Precision (mAP) compared to baseline methods when conditioned on OpenStreetMap road lines
- Maintains real-time performance with 14.7 FPS using 1 denoising step and 8.1 FPS with 5 steps on nuScenes benchmark
- Demonstrates superior performance particularly for long-range mapping where sensor limitations are more pronounced
- Shows robustness to various SD map imperfections through targeted training strategies

## Why This Works (Mechanism)
The diffusion-based approach works effectively because it naturally models the fusion problem as a denoising task where prior map information provides a strong initialization that the model refines using sensor observations. The iterative denoising process allows the model to gradually correct errors and fill gaps in the prior map while respecting the geometric constraints provided by sensor data. By treating map-sensor discrepancies as noise, the framework can leverage well-established diffusion model training techniques while maintaining interpretability of the fusion process.

**Vectorized HD Map Representation**: Why needed - To maintain geometric precision and enable downstream applications like path planning. Quick check - Ensure vector outputs preserve topological relationships and can be efficiently stored and queried.

## Architecture Onboarding

**Component Map**: Sensor data (LiDAR/Camera) -> Spatial Cross-Attention -> U-Net backbone -> Map Cross-Attention -> Prior Map embeddings -> Denoising diffusion process -> Vectorized HD map output

**Critical Path**: Sensor feature extraction → Spatial Cross-Attention fusion → U-Net denoising iterations → Vectorized map generation

**Design Tradeoffs**: The framework balances between leveraging strong prior map information and maintaining flexibility to correct map errors. Using fewer denoising steps improves real-time performance but may reduce map quality, while more steps improve accuracy at computational cost.

**Failure Signatures**: Poor performance when prior maps are severely outdated or missing critical infrastructure; degraded accuracy in areas with complex road geometries not captured in navigation maps; potential drift in map quality over extended operation periods.

**First Experiments**:
1. Baseline comparison with deterministic fusion methods on controlled synthetic datasets
2. Ablation study removing either Spatial or Map Cross-Attention components
3. Performance evaluation with varying numbers of denoising steps to assess speed-quality tradeoff

## Open Questions the Paper Calls Out
The paper highlights several important open questions including the framework's robustness in regions lacking map coverage or experiencing rapid urban changes, the extent of robustness to severe map degradation in real-world scenarios, and the computational requirements for deployment on resource-constrained edge devices commonly used in autonomous vehicles.

## Limitations
- Performance may degrade significantly in areas with poor or outdated prior map coverage
- Computational requirements for real-time operation may be challenging for edge devices
- Long-term map consistency and stability over extended operation periods remains uncertain
- Limited evaluation of performance in rapidly changing urban environments
