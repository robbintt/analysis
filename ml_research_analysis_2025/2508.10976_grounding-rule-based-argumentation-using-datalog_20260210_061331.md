# Grounding Rule-Based Argumentation Using Datalog

## Quick Facts
- arXiv ID: 2508.10976
- Source URL: https://arxiv.org/abs/2508.10976
- Reference count: 17
- Primary result: ANGRY, a Datalog-based grounder for first-order ASPIC+, outperforms the only other system supporting first-order ASPIC+ syntax and produces groundings comparable to state-of-the-art ASP grounder Gringo.

## Executive Summary
This paper introduces ANGRY, a grounder for first-order ASPIC+ argumentation theories that leverages Datalog to produce intelligent groundings. The core approach translates ASPIC+ rules into Datalog programs, queries a Datalog engine to obtain ground substitutions, and uses these to instantiate only relevant rule instances. Optimizations are introduced to further reduce grounding size by filtering out rules that do not affect the reasoning process, specifically through predicate classification (approximated vs. non-approximated) and simplification of strict rules. The system is implemented as a prototype and evaluated against existing systems, demonstrating superior performance on reasoning tasks and comparable grounding quality to established ASP grounders.

## Method Summary
The method centers on translating first-order ASPIC+ theories into Datalog programs to enable intelligent grounding. The translation uses auxiliary predicates and stratified negation to preserve argumentation semantics while filtering out irrelevant rule instances. A predicate classifier identifies "approximated" predicates involved in cyclic negative dependencies, allowing the grounder to exclude certain assumptions and rule instances. The grounded theory is then used with an ASP solver (ASPforASPIC) to compute argumentation extensions. The approach is implemented in a prototype system called ANGRY, built in Rust using the Nemo Datalog engine.

## Key Results
- ANGRY+ASPforASPIC solves 16 instances from the Legal bench, compared to 6 by the only other first-order ASPIC+ system (Arg2P).
- The grounding size produced by ANGRY is comparable and slightly larger than that produced by the state-of-the-art ASP grounder Gringo.
- The optimizations preserve acceptable conclusions under complete, grounded, preferred, and stable semantics, but not admissible semantics.

## Why This Works (Mechanism)

### Mechanism 1
Translating first-order ASPIC+ theories into Datalog programs enables intelligent grounding that preserves argumentation semantics while reducing grounding size. The Datalog engine is queried to obtain ground substitutions, which are used to instantiate only relevant rule instances. This filtering is based on the systematic translation of ASPIC+ rules to Datalog rules using auxiliary predicates. The core assumption is that this translation is semantics-preserving, meaning the arguments and attacks derivable from the grounded theory are identical to those from naive grounding.

### Mechanism 2
Predicate classification (approximated vs. non-approximated) and strict-rule simplification further reduce grounding size while preserving acceptable conclusions under complete, grounded, preferred, and stable semantics. Predicates involved in cyclic negative dependencies are marked as "approximated"; only non-approximated predicates appear in negated bodies of Datalog rules. This filters out assumptions and rule instances that are certainly attacked by non-attacked arguments. Additionally, strict rules whose bodies become fully grounded facts are simplified by adding their heads to the fact base and removing the rules.

### Mechanism 3
The ANGRY grounder, when coupled with the ASPforASPIC solver, outperforms the existing first-order ASPIC+ system Arg2P and produces groundings of size comparable to the state-of-the-art ASP grounder Gringo. This is demonstrated through empirical evaluation using three scenarios: reasoning on legal-domain ASPIC+ instances, scalability on randomly generated instances, and grounding quality on ASP-to-ASPIC+ translated instances.

## Architecture Onboarding

- **Component map:** Input ASPIC+ theory → Predicate Classifier → Transformer (Transformation 1 & 2) → Datalog Engine (Nemo) → Grounder Core (Algorithm 1 & 2) → Output grounded ASPIC+ theory
- **Critical path:** The translation to Datalog (`T → PT`) followed by querying. The correctness of the entire pipeline hinges on the semantic equivalence proven in Theorems 1 and 2.
- **Design tradeoffs:**
  - Datalog over ASP: Chosen for simplicity, transparency, and modularity. ASP grounders have hard-coded optimizations for stable models that may not suit ASPIC+ extension enumeration.
  - Semantics Preservation Scope: Optimizations preserve conclusions for `com, grd, prf, stb` semantics but *not* for `adm`. This is a deliberate trade-off for grounding size reduction.
  - Prototype Implementation (ANGRY): Built in Rust using Nemo. Performance is acceptable but not yet on par with mature ASP grounders like Gringo.
- **Failure signatures:**
  - Incorrect Grounding: If `grDL(T)` yields extensions with different conclusions than `gr(T)`, check: (a) predicate classification, (b) transformation of contraries, (c) implementation of stratified negation in Nemo.
  - Excessive Grounding Size: If `grDL(T)` is much larger than expected, verify that non-approximated predicates are correctly identified and that Algorithm 2 is properly simplifying strict rules.
  - Performance Bottleneck: If grounding is slow, profile the Datalog query step. The number of variables per rule has the largest impact (Table 1).
- **First 3 experiments:**
  1. Reproduce Scenario S1: Run ANGRY+ASPforASPIC on the Legal bench instances. Compare completion rate and time against the paper's reported results for both ANGRY and Arg2P.
  2. Test Optimization Impact: On a mid-sized instance from S2, run grounding with and without the non-approximated predicate optimization. Measure the difference in grounding size and verify that extension conclusions remain the same (for non-admissible semantics).
  3. Grounding Size Profile: Take an instance from S3 (e.g., GraphColouring). Compare the grounding size and time produced by ANGRY (on the ASPIC+ translation) versus Gringo (on the original ASP program).

## Limitations
- The semantic preservation proofs rely on the correctness of the translation and predicate classification, which are not formally verified in the paper.
- The grounding size comparison with Gringo is based on translated ASPIC+ instances, not directly comparable ASP programs, which may introduce overhead unrelated to the grounding approach itself.
- Admissible semantics is explicitly not preserved by the optimizations, limiting the method's applicability.
- The evaluation uses a relatively small set of benchmark instances, which may not be representative of all ASPIC+ usage patterns.

## Confidence
- **High Confidence:** The core claim that ANGRY outperforms Arg2P on first-order ASPIC+ instances is well-supported by the empirical results in Scenario S1.
- **Medium Confidence:** The semantic preservation theorems are stated but their proofs are not detailed in the paper, relying on external references and the reader's trust in the transformation definitions.
- **Medium Confidence:** The grounding size being "comparable and slightly larger" than Gringo is based on a specific set of translated instances and may not generalize to all ASPIC+ theories.
