# GeoSR: Cognitive-Agentic Framework for Probing Geospatial Knowledge Boundaries via Iterative Self-Refinement

## Quick Facts
- arXiv ID: 2508.04080
- Source URL: https://arxiv.org/abs/2508.04080
- Authors: Jinfan Tang; Kunming Wu; Ruifeng Gongxie; Yuya He; Yuankai Wu
- Reference count: 33
- Key outcome: Agentic self-refinement framework improving LLM geospatial predictions through iterative refinement and Tobler's First Law integration, achieving up to 67.9% Spearman correlation gains and 96.8% bias reduction without model retraining

## Executive Summary
This paper introduces GeoSR, an agentic self-refinement framework that enhances large language model (LLM) geospatial predictions by embedding Tobler's First Law of Geography into an iterative process. The framework employs three collaborating agents: one selects relevant auxiliary variables at target locations, another retrieves reference predictions from nearby locations, and a third coordinates the iterative refinement. Evaluated on both physical and socioeconomic prediction tasks, GeoSR significantly improves accuracy and reduces geographic bias across multiple LLM architectures without requiring model retraining.

## Method Summary
GeoSR implements an agentic self-refinement framework that iteratively improves LLM geospatial predictions by embedding Tobler's First Law of Geography. The system uses three specialized agents working in coordination: a Variable Selection Agent that identifies relevant auxiliary variables at the target location, a Reference Prediction Retrieval Agent that gathers predictions from geographically proximate locations, and a Coordination Agent that manages the iterative refinement process. This multi-agent approach allows the framework to systematically incorporate spatial context and improve predictions without requiring model retraining, addressing both accuracy and geographic bias concerns.

## Key Results
- Achieved up to 67.9% improvement in Spearman correlation for geospatial predictions
- Reduced geographic bias by up to 96.8% across multiple LLM architectures
- Demonstrated consistent performance improvements across physical and socioeconomic prediction tasks

## Why This Works (Mechanism)
The framework leverages Tobler's First Law of Geography by incorporating spatial context through nearby reference predictions and auxiliary variables. The iterative self-refinement process allows the model to progressively improve its understanding of geospatial relationships by combining local information with spatial patterns from surrounding areas. The multi-agent architecture enables specialized processing of different aspects of the geospatial prediction problem, with each agent focusing on a specific task that contributes to the overall refinement.

## Foundational Learning
- **Tobler's First Law of Geography**: Everything is related to everything else, but near things are more related than distant things. Why needed: Provides theoretical foundation for incorporating spatial context into predictions. Quick check: Verify that nearby locations actually show higher correlation in prediction errors.
- **Agentic AI systems**: Multiple specialized agents working in coordination to solve complex problems. Why needed: Enables decomposition of the geospatial prediction task into manageable subtasks. Quick check: Test performance with different numbers of agents.
- **Iterative refinement**: Repeated cycles of prediction improvement using feedback from previous iterations. Why needed: Allows progressive enhancement of predictions through accumulated spatial knowledge. Quick check: Measure convergence rate of the refinement process.
- **Geospatial bias**: Systematic errors in predictions that vary across geographic locations. Why needed: Identifies a key limitation of static LLM predictions in geospatial contexts. Quick check: Analyze prediction error distribution across different regions.
- **Spearman correlation**: Non-parametric measure of rank correlation. Why needed: Evaluates the quality of predictions when absolute values may not be as important as relative rankings. Quick check: Compare results with Pearson correlation for validation.
- **Auxiliary variables**: Additional contextual information that can improve prediction accuracy. Why needed: Provides relevant context beyond the primary prediction task. Quick check: Test performance with different sets of auxiliary variables.

## Architecture Onboarding

Component map: Variable Selection Agent -> Reference Prediction Retrieval Agent -> Coordination Agent -> Refined Prediction

Critical path: The system takes a target location and prediction task, then the Variable Selection Agent identifies relevant auxiliary variables, the Reference Prediction Retrieval Agent gathers nearby predictions, and the Coordination Agent manages iterative refinement until convergence or maximum iterations reached.

Design tradeoffs: The framework trades computational overhead for improved accuracy and reduced bias. The multi-agent approach adds complexity but enables specialized processing. The iterative nature requires more inference steps but allows progressive improvement without retraining.

Failure signatures: The system may fail to converge if nearby locations provide contradictory information, if auxiliary variables are poorly chosen, or if the coordination agent cannot effectively balance competing signals. Performance may degrade in areas with sparse data or rapidly changing spatial patterns.

## Limitations
- Evaluation primarily based on comparisons against static LLM baselines without external validation on truly independent datasets
- Focus on prediction accuracy gains without thorough exploration of computational overhead or real-world deployment constraints
- Geographic bias reduction analysis lacks qualitative investigation into potential introduction of new bias forms or artifacts

## Confidence
- High confidence in reported accuracy improvements (up to 67.9% Spearman correlation gains) based on multiple datasets and LLM architectures
- Medium confidence in geographic bias reduction claims (up to 96.8%) as bias metrics may not capture all dimensions of geographic bias
- Low confidence in framework's scalability and computational efficiency as these aspects are not deeply explored

## Next Checks
1. Replicate the framework on an entirely independent geospatial dataset not used in the original training or validation process to test generalizability
2. Conduct ablation studies to isolate the contribution of each agent (variable selection, reference prediction retrieval, coordination) to quantify their individual impact on performance
3. Perform computational overhead analysis comparing the total inference time and resource usage of GeoSR against baseline LLM predictions to assess practical deployment feasibility
