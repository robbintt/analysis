# DiffusionLight-Turbo: Accelerated Light Probes for Free via Single-Pass Chrome Ball Inpainting

## Quick Facts
- arXiv ID: 2507.01305
- Source URL: https://arxiv.org/abs/2507.01305
- Reference count: 40
- Single-pass chrome ball inpainting with diffusion models reduces HDR lighting estimation time from 30 minutes to 30 seconds

## Executive Summary
DiffusionLight-Turbo is a method for estimating high dynamic range (HDR) lighting from a single low dynamic range (LDR) image by inpainting a chrome ball using a pre-trained diffusion model. It builds upon the earlier DiffusionLight approach, which required multiple denoising passes, by training a Turbo LoRA to predict average chrome balls directly, achieving a 60× speedup while maintaining competitive performance. The method leverages Stable Diffusion XL and introduces a LoRA swapping technique to apply different LoRA modules at different stages of the denoising process, further optimizing speed and quality.

## Method Summary
The method estimates HDR lighting by inpainting a chrome ball in a single pass using a pre-trained diffusion model (Stable Diffusion XL). Unlike the original DiffusionLight, which required multiple denoising steps, DiffusionLight-Turbo trains a Turbo LoRA to predict average chrome balls directly, drastically reducing inference time. A LoRA swapping technique is introduced, where the Turbo LoRA is applied early in the denoising process and an Exposure LoRA later, achieving a 60× speedup. The method generalizes well to diverse in-the-wild scenes and produces convincing light estimates, outperforming existing approaches in many cases.

## Key Results
- Reduces HDR lighting estimation time from 30 minutes to 30 seconds using Turbo LoRA
- Achieves 60× speedup with minimal quality loss via LoRA swapping technique
- Outperforms state-of-the-art methods on standard benchmarks while generalizing to diverse in-the-wild scenes

## Why This Works (Mechanism)
The method works by leveraging the generative capabilities of diffusion models to inpaint missing or corrupted regions (chrome balls) in images, which are then used to estimate HDR lighting. By training a Turbo LoRA to predict average chrome balls directly, the method bypasses the need for multiple denoising passes, significantly accelerating inference. The LoRA swapping technique optimizes the denoising process by applying different LoRA modules at different stages, balancing speed and quality. The pre-trained Stable Diffusion XL model provides a strong prior for generating realistic chrome ball appearances, which are crucial for accurate lighting estimation.

- **Chrome Ball as Light Probe**: Reflective sphere used to capture environmental lighting; needed as a proxy for scene illumination; quick check: ensure chrome ball inpainting preserves reflection details.
- **Single-Pass Inference**: Completing the task in one forward pass; needed to drastically reduce computation time; quick check: measure latency improvement over multi-pass methods.
- **LoRA Swapping**: Dynamically switching LoRA modules during denoising; needed to optimize for both speed and quality; quick check: test if swapping improves metrics over fixed LoRA.

## Architecture Onboarding

**Component Map**
LDR Image -> Chrome Ball Detection -> Stable Diffusion XL with Turbo LoRA -> HDR Lighting Estimation

**Critical Path**
Image input → Chrome ball detection → Turbo LoRA-based inpainting → Lighting estimation pipeline

**Design Tradeoffs**
- Speed vs. quality: Multi-pass methods are slower but potentially more accurate; single-pass with Turbo LoRA sacrifices some quality for 60× speedup.
- Generalization vs. specialization: Pre-trained SDXL offers broad applicability but may not be optimal for all lighting scenarios.
- Complexity vs. performance: LoRA swapping adds implementation complexity but yields better results than fixed LoRA.

**Failure Signatures**
- Inaccurate chrome ball detection leading to poor inpainting
- Turbo LoRA generating unrealistic reflections under complex lighting
- LoRA swapping causing artifacts if modules are not well-aligned
- Generalization issues in scenes with rare or extreme lighting conditions
