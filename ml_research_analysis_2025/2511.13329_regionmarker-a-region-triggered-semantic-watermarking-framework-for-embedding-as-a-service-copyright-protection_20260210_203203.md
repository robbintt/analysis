# RegionMarker: A Region-Triggered Semantic Watermarking Framework for Embedding-as-a-Service Copyright Protection

## Quick Facts
- arXiv ID: 2511.13329
- Source URL: https://arxiv.org/abs/2511.13329
- Reference count: 40
- Primary result: RegionMarker provides comprehensive protection for EaaS against model extraction attacks through region-triggered semantic watermarking

## Executive Summary
RegionMarker addresses the vulnerability of Embedding-as-a-Service (EaaS) platforms to model extraction attacks, where attackers query APIs to steal proprietary embedding models. The framework introduces a region-triggered semantic watermarking approach that embeds watermarks in low-dimensional semantic spaces rather than raw embeddings, making detection and removal significantly more difficult for attackers. By combining a secret dimensionality reduction matrix with random trigger region selection, RegionMarker achieves robust copyright protection while maintaining embedding quality.

## Method Summary
RegionMarker operates by defining trigger regions in a low-dimensional semantic space and injecting watermarks into text embeddings associated with these regions. The framework uses a secret dimensionality reduction matrix to project high-dimensional embeddings into a lower-dimensional semantic space where watermark injection occurs. Random trigger region selection further enhances security by preventing attackers from easily identifying and removing watermarks. This semantic-space approach contrasts with traditional methods that embed watermarks directly in high-dimensional embedding spaces, making RegionMarker more resistant to watermark removal attacks.

## Key Results
- Detection p-values consistently below 0.05 across four datasets, indicating strong watermark presence
- ∆cos and ∆l2 metrics meet or exceed copyright protection thresholds
- Effective protection against paraphrasing and dimension-perturbation attacks
- Comprehensive protection where previous methods failed

## Why This Works (Mechanism)
RegionMarker's effectiveness stems from its fundamental shift in watermarking strategy from high-dimensional to low-dimensional semantic spaces. By embedding watermarks in semantic regions rather than raw embedding coordinates, the framework exploits the inherent structure of semantic spaces where similar meanings cluster together. The secret dimensionality reduction matrix ensures that only authorized parties can detect watermarks, while random trigger region selection prevents attackers from reverse-engineering the watermarking pattern. This dual-layer security makes watermark removal exponentially more difficult compared to traditional approaches.

## Architecture Onboarding

Component Map: Input Text -> Embedding Model -> Dimensionality Reduction -> Semantic Space -> Trigger Region Detection -> Watermark Injection -> Output Watermarked Embedding

Critical Path: The most critical path is the dimensionality reduction and trigger region detection sequence, as any failure here prevents watermark injection entirely. This path must maintain semantic fidelity while enabling reliable watermark detection.

Design Tradeoffs: RegionMarker trades computational overhead (from dimensionality reduction) for enhanced security. The random trigger region selection improves security but may reduce watermark detection reliability if regions are poorly chosen.

Failure Signatures: Common failures include poor semantic preservation during dimensionality reduction, trigger regions that don't capture meaningful semantic distinctions, and watermark patterns that become detectable through statistical analysis.
