# Phase-OTDR Event Detection Using Image-Based Data Transformation and Deep Learning

## Quick Facts
- arXiv ID: 2512.05830
- Source URL: https://arxiv.org/abs/2512.05830
- Authors: Muhammet Cagri Yeke; Samil Sirin; Kivilcim Yuksel; Abdurrahman Gumus
- Reference count: 40
- One-line primary result: Image-based Phase-OTDR event detection achieves 98.84% accuracy using EfficientNetB0 and 98.24% using DenseNet121.

## Executive Summary
This study addresses the challenge of event detection and classification in optical fibers using the Phase-OTDR system, which traditionally involves complex 1D signal processing. The core method introduces a novel approach that transforms 1D Phase-OTDR data into multi-channel RGB images using mathematical techniques: Gramian Angular Difference Field (GADF), Gramian Angular Summation Field (GASF), and Recurrence Plot (RP). These image representations are then analyzed using transfer learning models to enhance classification accuracy and efficiency. The primary results demonstrate high classification accuracies: 98.84% and 98.24% for EfficientNetB0 and DenseNet121 models respectively using holdout validation, with test accuracies of 99.07% and 98.68% confirmed through 5-fold cross-validation. The study also achieves a significant reduction in dataset size from 2.03 GB to 180 MB, improving data management efficiency while maintaining performance. This image-based approach offers a robust and efficient solution for interpreting complex fiber optic sensing data, advancing the accuracy and reliability of fiber optic monitoring systems.

## Method Summary
The study proposes a novel image-based approach for Phase-OTDR event detection that transforms 1D signal data into visual representations using GADF, GASF, and Recurrence Plot techniques. These mathematical transformations convert the time-series data into multi-channel RGB images that capture temporal and structural patterns. The resulting image datasets are then analyzed using transfer learning models (EfficientNetB0 and DenseNet121) that have been pre-trained on ImageNet. The approach includes comprehensive data preprocessing, normalization, and augmentation to enhance model robustness. The system was evaluated using both holdout validation and 5-fold cross-validation on the Cao et al. dataset containing 15,612 samples across 7 event classes.

## Key Results
- EfficientNetB0 achieved 98.84% classification accuracy and 99.07% test accuracy on Phase-OTDR data
- DenseNet121 achieved 98.24% classification accuracy and 98.68% test accuracy on Phase-OTDR data
- Dataset size was reduced from 2.03 GB to 180 MB through image-based representation

## Why This Works (Mechanism)
The image-based transformation approach works by converting complex 1D Phase-OTDR signals into structured visual representations that deep learning models can process more efficiently. GADF and GASF techniques encode temporal dependencies and phase relationships into spatial patterns, while Recurrence Plots capture the recurrence properties of the signal dynamics. These transformations reveal hidden patterns in the data that are difficult to extract through traditional 1D signal processing methods. By leveraging transfer learning from pre-trained models, the system benefits from learned feature representations that generalize well to the Phase-OTDR domain, resulting in high classification accuracy with reduced computational complexity.

## Architecture Onboarding

Component Map:
1D Phase-OTDR Signal -> GADF/GASF/RP Transformation -> RGB Image Dataset -> Transfer Learning Model (EfficientNetB0/DenseNet121) -> Classification Output

Critical Path:
The critical path involves transforming raw 1D Phase-OTDR signals into RGB images using the three mathematical techniques, then feeding these images into pre-trained deep learning models. The transformation step is crucial as it converts complex temporal patterns into spatial representations that deep learning architectures can process efficiently.

Design Tradeoffs:
The approach trades computational complexity during transformation for improved classification accuracy and reduced model training requirements. While the image transformation adds preprocessing overhead, it enables the use of powerful pre-trained models that would be difficult to train from scratch on Phase-OTDR data. The choice of three different transformation techniques provides complementary information but increases storage requirements and processing time.

Failure Signatures:
Classification performance may degrade when applied to datasets with significantly different noise characteristics or fiber types than those used in training. The approach may struggle with rare or novel event types not represented in the training data. Poor quality Phase-OTDR signals with low signal-to-noise ratios may produce images that lack discriminative features.

First Experiments:

3. Compare classification performance when using individual transformation techniques (GADF only, GASF only, RP only) versus their combination

## Open Questions the Paper Calls Out
### Open Question 1
- Question: Does the proposed image-based transformation method maintain high classification performance when applied to diverse or larger Phase-OTDR datasets outside of the single controlled environment used in this study?
- Basis in paper: [explicit] The Conclusion identifies the "reliance on a single, controlled dataset" as a key limitation and states that future research must focus on "validating the generalizability of this methodology across diverse and larger Phase-OTDR datasets."
- Why unresolved: The experimental validation was restricted to the Cao et al. dataset (15,612 samples), and the authors did not test the model on data from different fiber setups or environmental conditions.
- What evidence would resolve it: Reporting classification accuracy and F1 scores when the EfficientNetB0/DenseNet121 models are trained and tested on independent Phase-OTDR datasets with varying signal-to-noise ratios or fiber lengths.

### Open Question 2
- Question: Can the image transformation and deep learning pipeline be optimized to meet the latency requirements of real-time event detection systems?
- Basis in paper: [explicit] The authors explicitly list "exploring real-time processing implementations" as a direction for future research in the Conclusion.
- Why unresolved: While the study highlights a reduction in dataset size (storage efficiency), it provides no analysis regarding the computational latency of the GADF/GASF/RP transformations or the inference speed required for real-time monitoring.
- What evidence would resolve it: Benchmarks of end-to-end processing time (transformation + inference) per sample and an analysis of throughput on standard CPU/GPU hardware.

### Open Question 3
- Question: To what extent does fine-tuning specific image transformation parameters, such as the Recurrence Plot threshold, improve the robustness of the classification in noisy environments?
- Basis in paper: [explicit] The Conclusion states that future work involves "optimizing image transformation parameters for enhanced robustness in varied operational environments."
- Why unresolved: The study applies GADF, GASF, and RP techniques using standard parameters but does not perform an ablation study to determine if specific parameter tuning improves discrimination between events with similar strain characteristics.
- What evidence would resolve it: A comparative analysis showing how varying the threshold ($\epsilon$) in Recurrence Plots affects the classification accuracy for difficult classes (e.g., digging vs. background) under different noise levels.

## Limitations
- Reliance on a single, controlled dataset limits generalizability to diverse real-world conditions
- No analysis of computational latency for real-time deployment scenarios
- Limited exploration of optimal parameters for image transformation techniques
