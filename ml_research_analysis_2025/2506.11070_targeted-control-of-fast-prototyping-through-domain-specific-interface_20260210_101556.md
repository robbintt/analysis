# Targeted control of fast prototyping through domain-specific interface

## Quick Facts
- arXiv ID: 2506.11070
- Source URL: https://arxiv.org/abs/2506.11070
- Reference count: 40
- Primary result: Domain-specific interface bridges designer language and modeling commands using hierarchical probabilistic model and MCMC sampling guided by LLMs

## Executive Summary
This paper presents a novel interface for fast prototyping that addresses the communication gap between designers' natural language and technical modeling commands. The system employs a domain-specific language (DSL) as an intermediate representation, automatically adapted to different design domains through a hierarchical probabilistic model and MCMC sampling guided by large language models. The approach enables high-level, ambiguous designer intentions to be translated into precise, low-level modeling commands while maintaining creative control.

Human studies across eight product domains demonstrate the interface's effectiveness, showing significant improvements in rendering consistency (1.0-3.0 scale) and information clarity compared to baseline methods. The research addresses a critical bottleneck in the design-to-engineering workflow by reducing communication overhead while preserving the designer's creative intent.

## Method Summary
The proposed method introduces a domain-specific language (DSL) that serves as an intermediate representation between designers' natural language and modeling commands. The system automatically adapts to different design domains using a hierarchical probabilistic model and MCMC sampling guided by large language models. This architecture allows the translation of high-level, ambiguous designer intentions into precise, low-level modeling commands. The interface is evaluated through human studies across eight product domains, comparing rendering consistency and information clarity against baseline approaches.

## Key Results
- Interface significantly outperforms baselines in rendering consistency (1.0-3.0 scale) with p < 0.001 statistical significance
- Information clarity improved across all eight product domains tested
- Automatic domain adaptation achieved through hierarchical probabilistic model and MCMC sampling
- Maintains designer creative control while reducing communication costs between designers and engineers

## Why This Works (Mechanism)
The system works by establishing a DSL as an intermediate layer that captures domain-specific semantics while remaining abstract enough to handle ambiguous designer input. The hierarchical probabilistic model enables context-aware translation by learning domain-specific patterns and relationships. MCMC sampling guided by LLMs provides a robust mechanism for exploring the solution space when dealing with ambiguous or incomplete designer specifications, effectively bridging the gap between natural language and precise modeling commands.

## Architecture Onboarding

Component map: Designer Input -> DSL Parser -> Hierarchical Model -> MCMC Sampler -> LLM Guidance -> Modeling Commands -> Rendering Output

Critical path: Designer Intent → DSL Translation → Domain-Specific Adaptation → Precise Command Generation → Prototype Rendering

Design tradeoffs:
- Flexibility vs precision in DSL design
- Computational cost vs translation quality in MCMC sampling
- Domain specificity vs generalization capability

Failure signatures:
- Misinterpretation of designer intent due to DSL limitations
- Convergence issues in MCMC sampling for highly ambiguous inputs
- Domain adaptation failures when encountering novel design patterns

## Limitations
- Generalizability beyond eight tested product domains remains unproven
- Long-term maintenance and scalability of automatically generated interfaces not evaluated
- Performance bottlenecks in real-time translation not addressed
- Error handling for invalid or contradictory inputs not explored
