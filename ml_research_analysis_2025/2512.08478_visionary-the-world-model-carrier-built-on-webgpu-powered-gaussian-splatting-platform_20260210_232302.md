# Visionary: The World Model Carrier Built on WebGPU-Powered Gaussian Splatting Platform
## Quick Facts
- arXiv ID: 2512.08478
- Source URL: https://arxiv.org/abs/2512.08478
- Reference count: 40
- Presents a web-native platform for real-time 3D Gaussian Splatting rendering with up to 135× speed-up over WebGL

## Executive Summary
Visionary is a web-native platform that enables real-time rendering of various 3D Gaussian Splatting variants and meshes directly in browsers. Built on WebGPU with per-frame ONNX inference, it addresses the deployment gap in neural rendering by allowing dynamic neural processing while maintaining a lightweight "click-to-run" experience. The platform's key innovation is the Gaussian Generator contract - a standardized ONNX I/O schema that enables plug-and-play integration of 3DGS algorithms without modifying the rendering pipeline.

## Method Summary
Visionary leverages WebGPU for GPU-accelerated rendering and introduces a standardized ONNX-based interface called the Gaussian Generator contract. This contract defines a unified I/O schema for 3DGS algorithms, allowing different variants to be integrated without modifying the core rendering pipeline. The platform supports real-time rendering of MLP-based 3DGS, 4DGS for dynamic scenes, neural avatars, and generative post-processing networks, serving as a unified World Model Carrier for both reconstructive and generative paradigms.

## Key Results
- Achieves up to 135× speed-up over WebGL-based viewers like SparkJS due to GPU-based primitive sorting
- Supports multiple 3DGS variants including MLP-based 3DGS, 4DGS for dynamic scenes, and neural avatars
- Enables plug-and-play integration of 3DGS algorithms through the standardized Gaussian Generator contract
- Provides unified platform for both reconstructive and generative neural rendering paradigms

## Why This Works (Mechanism)
Visionary works by combining WebGPU's hardware acceleration capabilities with a standardized ONNX interface for neural processing. The WebGPU backend enables efficient GPU-based primitive sorting and rendering, while the Gaussian Generator contract abstracts the complexity of different 3DGS algorithms into a common interface. This separation of concerns allows the rendering pipeline to remain constant while supporting diverse neural rendering approaches through dynamic ONNX model execution.
