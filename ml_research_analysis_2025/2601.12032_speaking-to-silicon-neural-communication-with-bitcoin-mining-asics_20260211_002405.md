## Why This Works (Mechanism)

### Mechanism 1: Thermodynamic Reservoir Computing via Timing Jitter
Bitcoin mining ASICs function as physical reservoir computers, achieving NARMA-10 NRMSE of 0.8661 through the Single-Word Handshake (SWH) protocol. The timing jitter and thermal variations in mining hardware encode exploitable information about ongoing computations. By enforcing strict one-to-one correspondence between input nonces and response times (SWH), the coefficient of variation (CV) of timing measurements acts as an order parameter. At CV≈0.092 ("SYNC" state), reduced variability optimizes the reservoir for computation. A readout layer trained via ridge regression on timing features maps reservoir states to target outputs.

### Mechanism 2: Thermodynamic Probability Filter (TPF) for Early-Abort
The TPF predicts SHA-256 hash failures from early-round thermodynamic signatures, achieving a theoretical 92.19% energy reduction. SHA-256's hierarchical Merkle-Damgård structure creates information-theoretic constraints. Early-round thermodynamic signatures (timing, temperature, voltage) encode a probability of ultimate hash failure. A machine-verified theorem establishes that if a predictor beats baseline, the system is non-independent, allowing a classifier to decide "likely failure" after round k (e.g., k=5 of n=64), enabling early abort.

### Mechanism 3: Virtual Block Manager (VBM) for Latency Elimination
The VBM eliminates network latency, achieving a +25% effective hashrate. In standard mining, ASICs idle while waiting for new block templates from the pool. VBM implements local block template prefetching, ensuring the ASIC always has valid work. By reducing network wait time toward zero, the time efficiency approaches 1.0.

## Architecture Onboarding

**Component map**: ASIC Hardware -> SWH Controller -> Feature Extractor -> TPF Classifier -> Early-Abort Decision (for TPF); VBM -> Local Block Assembly -> ASIC (for VBM)

**Critical path**: SWH Controller -> ASIC -> Feature Extractor -> TPF Classifier -> Early-Abort Decision. For VBM: VBM -> Local Block Assembly -> ASIC. The formalization provides offline validation.

**Design tradeoffs**:
- SWH vs. Pipelined Stratum: SWH provides precise timing but reduces throughput. Pipelining increases throughput but introduces timing ambiguity.
- TPF Energy Savings vs. False Abort Risk: Aggressive early abort (low `k`) increases theoretical savings but risks aborting valid shares.
- SYNC (CV=0.092) vs. OPTIMAL (CV=0.586): SYNC optimizes reservoir computing consistency; OPTIMAL maximizes thermodynamic sensing entropy.

**Failure signatures**:
- High classifier false-positive rate: Would cause valid shares to be aborted, directly reducing mining revenue.
- `leakage_eq_zero_of_independent` holds: If thermodynamic signatures are truly independent of hash outcome, TPF cannot work.
- VBM orphans: If prefetched templates become invalid, the ASIC performs wasted work.

## Limitations

**Hardware Dependence**: The core findings rely on specific ASIC models (Antminer S9, Lucky Miner LV06, Goldshell LB-Box) with custom firmware modifications for SWH. Generalization to other mining hardware or newer models is untested.

**Measurement Precision**: Achieving microsecond-level timestamp accuracy requires precise control over Stratum proxy behavior and ASIC response timing. Without exact implementation details, external validation faces substantial technical hurdles.

**Information Theory Assumption**: The TPF mechanism fundamentally depends on the unproven assumption that thermodynamic signatures violate the independence assumption required by the avalanche effect. The Lean formalization proves a theorem about predictability, but physical validation of this side-channel remains empirical.
