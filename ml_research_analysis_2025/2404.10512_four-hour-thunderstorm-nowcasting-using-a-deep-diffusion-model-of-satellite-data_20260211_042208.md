# Four-hour thunderstorm nowcasting using a deep diffusion model of satellite data

## Quick Facts
- **arXiv ID**: 2404.10512
- **Source URL**: https://arxiv.org/abs/2404.10512
- **Reference count**: 0
- **Primary result**: DDMS achieves CSI scores of 0.33-0.36 at 4-km resolution for 4-hour convective cloud nowcasting

## Executive Summary
This study introduces a deep diffusion model for satellite data (DDMS) that achieves 4-hour convective cloud nowcasting using FengYun-4A satellite brightness temperature data. The model combines a satellite predictor with a temporal diffusion module to capture spatiotemporal evolution patterns of convective clouds. Tested on data from 2018-2023, DDMS outperforms state-of-the-art baselines including NowcastNet, PredRNN-v2, PySTEPS, and persistence methods in predicting convective cloud growth, dissipation, and intensity over extended lead times.

## Method Summary
The DDMS approach employs a two-component architecture: a satellite predictor that extracts spatiotemporal features from brightness temperature data, and a temporal diffusion module that generates future frames through iterative denoising. The model is trained on historical data spanning 2018-2021 and validated on 2022-2023 data. The diffusion process progressively refines predictions by reversing a noise-adding process, allowing the model to capture complex cloud evolution dynamics. The method specifically targets the challenging task of 4-hour nowcasting, which is critical for severe weather warning systems.

## Key Results
- DDMS achieves CSI scores of 0.33-0.36 at 4-km resolution across evaluation months
- Outperforms baseline methods including NowcastNet, PredRNN-v2, PySTEPS, and persistence
- Accurately predicts convective cloud growth, dissipation, and intensity over 4-hour lead times
- Demonstrates effectiveness in capturing spatiotemporal evolution patterns of severe weather systems

## Why This Works (Mechanism)
The diffusion model architecture enables effective learning of the complex spatiotemporal dynamics in convective cloud systems. By iteratively denoising through the temporal diffusion module, the model can capture both short-term and long-term dependencies in cloud evolution patterns. The satellite predictor component effectively extracts relevant features from brightness temperature data, while the diffusion process allows for generation of realistic future states that account for the stochastic nature of severe weather development.

## Architecture Onboarding
**Component map**: Satellite Predictor -> Temporal Diffusion Module -> Output Generator

**Critical path**: Input brightness temperature frames → Feature extraction → Diffusion process → Denoised future frames

**Design tradeoffs**: 
- Diffusion steps vs. computational efficiency: More steps improve quality but increase inference time
- Resolution vs. coverage: 4-km resolution balances detail with computational tractability
- Training data duration vs. model generalization: Longer historical data improves robustness

**Failure signatures**: 
- Mode collapse in generated clouds (overly uniform predictions)
- Temporal inconsistencies between consecutive frames
- Overestimation of cloud intensity in stable weather conditions
- Underestimation during rapid convective development
