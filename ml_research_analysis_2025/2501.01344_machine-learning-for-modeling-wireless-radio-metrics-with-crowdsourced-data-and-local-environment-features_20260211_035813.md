# Machine Learning for Modeling Wireless Radio Metrics with Crowdsourced Data and Local Environment Features

## Quick Facts
- **arXiv ID**: 2501.01344
- **Source URL**: https://arxiv.org/abs/2501.01344
- **Reference count**: 25
- **Primary result**: CRC-ML-4G models achieve RMSE performances of 9.76–11.69 dB for RSRP, 2.90–3.23 dB for RSRQ, and 9.50–10.36 dB for RSSI across three Canadian cities

## Executive Summary
This paper introduces CRC-ML-4G radio metric models that predict RSRP, RSRQ, and RSSI in 4G wireless environments using crowdsourced data and local environmental features. The models employ a two-stage approach combining path loss estimation with deep neural network refinement to achieve accurate predictions across both indoor and outdoor urban settings. Evaluated on over 300,000 data points from Toronto, Montreal, and Vancouver, the models demonstrate robust performance for network planning and quality of service optimization in complex Canadian urban environments.

## Method Summary
The CRC-ML-4G models use a two-stage approach: first estimating path loss using simplified formulas, then refining predictions with a deep neural network. The system incorporates crowdsourced data from three major Canadian cities along with local environmental features including land use, building characteristics, and terrain information. Temporal features are included to capture environmental variations, improving RSRQ and RSSI predictions. The models are trained and validated on datasets containing over 300,000 measurements, with separate test sets for each city to evaluate generalization performance.

## Key Results
- RSRP RMSE: 9.76–11.69 dB across Toronto, Montreal, and Vancouver
- RSRQ RMSE: 2.90–3.23 dB showing consistent performance across cities
- RSSI RMSE: 9.50–10.36 dB with temporal features improving prediction accuracy
- Model robustness demonstrated across both indoor and outdoor urban environments

## Why This Works (Mechanism)
The two-stage approach effectively combines physical radio propagation principles with machine learning to capture complex urban propagation effects. The initial path loss estimate provides a physically meaningful baseline, while the deep neural network learns to model deviations caused by local environmental features, building characteristics, and temporal variations. By incorporating crowdsourced data, the models learn from real-world measurements that capture the complexity of urban wireless environments better than traditional propagation models. The inclusion of temporal features allows the system to adapt to diurnal and seasonal variations in radio propagation conditions.

## Architecture Onboarding

**Component Map:**
Data Preprocessing -> Path Loss Estimation -> Deep Neural Network -> Output Predictions

**Critical Path:**
Crowdsourced measurements → Environmental feature extraction → Path loss calculation → DNN refinement → Radio metric predictions

**Design Tradeoffs:**
The two-stage approach adds computational complexity but improves accuracy by separating physical and environmental effects. Using crowdsourced data provides real-world coverage but introduces sampling bias. Including temporal features improves prediction accuracy but increases model complexity and data requirements.

**Failure Signatures:**
- High RMSE in areas with sparse crowdsourced data
- Poor performance in environments with features not represented in training data
- Degradation in temporal predictions during unusual weather conditions
- Overfitting to specific building types or urban layouts

## Limitations
- Limited geographic scope (three Canadian cities) may not represent global urban diversity
- Crowdsourced data introduces potential sampling bias based on user behavior patterns
- Two-stage approach complexity may limit real-time deployment efficiency
- Environmental feature set may not capture all relevant urban characteristics
