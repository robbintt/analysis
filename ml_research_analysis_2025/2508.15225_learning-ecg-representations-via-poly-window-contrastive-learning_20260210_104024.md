# Learning ECG Representations via Poly-Window Contrastive Learning

## Quick Facts
- arXiv ID: 2508.15225
- Source URL: https://arxiv.org/abs/2508.15225
- Reference count: 25
- Primary result: Poly-window CL achieves AUROC 0.891 and F1 0.680 on PTB-XL with 4x faster training

## Executive Summary
Poly-window contrastive learning introduces a novel self-supervised approach for ECG representation learning that extracts multiple temporal windows from single recordings and maximizes agreement across them. This method leverages the inherent temporal redundancy of ECG signals to learn physiologically meaningful, temporally invariant features. By creating multiple views from one recording through cropping, the approach significantly improves training efficiency and downstream classification performance compared to standard two-view contrastive methods.

The technique addresses the challenge of learning meaningful representations from biomedical time-series data without requiring extensive manual annotation. Through systematic exploration of temporal windows within ECG recordings, the method captures diverse physiological patterns while maintaining computational efficiency. The approach demonstrates practical utility for multi-label classification tasks on the PTB-XL dataset, achieving state-of-the-art performance with substantially reduced training requirements.

## Method Summary
The method extracts multiple temporal windows from single ECG recordings and applies contrastive learning to maximize agreement across these views. Rather than the standard two-view approach in contrastive learning, poly-window CL creates several cropped views from each recording, treating them as positive pairs in the contrastive loss. The cropping strategy generates views with different temporal positions and scales, enabling the model to learn features invariant to temporal variations while preserving physiological information. The approach is trained using standard contrastive loss functions but with multiple positive pairs per anchor, increasing the efficiency of representation learning from limited labeled data.

## Key Results
- Achieves AUROC of 0.891 and F1 score of 0.680 on PTB-XL multi-label superclass classification
- Reduces wall clock training time by up to 14.8% compared to standard contrastive methods
- Requires four times fewer pre-training epochs (32 vs 128) to reach comparable performance
- Ablation studies identify crop size and window multiplicity as the most influential hyperparameters

## Why This Works (Mechanism)
The approach works by exploiting the inherent temporal redundancy in ECG signals. ECG recordings contain repeating physiological patterns and temporal structures that can be captured through multiple cropped views. By creating several positive pairs from a single recording, the model is forced to learn representations that are invariant to temporal shifts and cropping while preserving diagnostic features. This multi-view strategy increases the effective training signal without requiring additional data, making the representation learning process more sample-efficient. The temporal invariance learned through poly-window contrastive learning aligns well with the requirements of ECG analysis, where physiological patterns should be recognizable regardless of their exact timing within the recording.

## Architecture Onboarding

**Component Map:** ECG recording -> Multiple temporal crop views -> Encoder network -> Feature representations -> Contrastive loss (multiple positive pairs) -> Optimized model

**Critical Path:** Data preprocessing (cropping) -> Encoder backbone (ResNet/CNN) -> Projection head -> Contrastive loss computation -> Parameter updates

**Design Tradeoffs:** Multiple windows increase computational overhead per batch but reduce total epochs needed; larger crop sizes capture more context but may miss fine-grained details; more windows improve representation quality but increase memory requirements.

**Failure Signatures:** Poor downstream performance indicates loss of physiological information in cropping; unstable training suggests inappropriate temperature parameter in contrastive loss; high variance across runs suggests insufficient crop diversity or suboptimal hyperparameter tuning.

**First Experiments:** 1) Implement basic two-view contrastive learning baseline on PTB-XL; 2) Add poly-window cropping with varying window counts; 3) Systematically vary crop sizes to identify optimal temporal resolution.
