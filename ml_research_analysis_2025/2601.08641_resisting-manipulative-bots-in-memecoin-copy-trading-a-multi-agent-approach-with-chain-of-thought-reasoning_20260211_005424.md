# Resisting Manipulative Bots in Memecoin Copy Trading: A Multi-Agent Approach with Chain-of-Thought Reasoning

## Quick Facts
- arXiv ID: 2601.08641
- Source URL: https://arxiv.org/abs/2601.08641
- Reference count: 40
- Multi-agent system achieves 73% precision in identifying high-quality meme coin projects and 70% precision in identifying key opinion leader wallets

## Executive Summary
This paper addresses the challenge of manipulative bots in meme coin copy trading, where automated trading bots exploit copy traders by creating fake trading activity and sentiment to extract value. The authors propose an explainable multi-agent system with chain-of-thought reasoning that decomposes the task into specialized agents for coin evaluation, wallet selection, and timing assessment. Using a dataset of 1,000 meme coin projects, the system demonstrates significant improvements over traditional machine learning models and single LLM approaches, achieving strong precision in identifying profitable trading opportunities while resisting manipulation.

## Method Summary
The system employs three specialized LLM agents (Coin, Wallet, and Timing) using few-shot chain-of-thought prompting to evaluate meme coin projects. It processes on-chain transaction data from Flipside and pump.fun comments, applying bot detection algorithms to identify manipulative patterns. Agent outputs are combined using weighted aggregation optimized for AUC. The approach contrasts with traditional statistic-driven baselines (LASSO, XGBoost, neural networks) and aims to overcome the "imitation penalty" inherent to bonding curve mechanics where copiers always pay higher prices than target wallets.

## Key Results
- 73% precision in identifying high-quality meme coin projects
- 70% precision in identifying key opinion leader (KOL) wallets
- Selected KOLs collectively generated approximately $500,000 in profits
- Multi-agent approach with CoT prompting outperforms both traditional ML models and single LLM baselines

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Decomposing copy trading into specialized sub-tasks improves robustness against manipulation
- Mechanism: The system mirrors human asset management teams by assigning distinct roles to separate LLM agents, allowing granular reasoning about asset quality, trader skill, and entry timing independently
- Core assumption: Copy-trading problem space is factorable into semi-independent dimensions that can be evaluated separately
- Evidence anchors: Abstract describes specialized agents for coin evaluation, wallet selection, and timing assessment; multi-agent frameworks validated in related work
- Break condition: Performance degrades if sub-tasks are highly interdependent in non-linear ways

### Mechanism 2
- Claim: Few-shot Chain-of-Thought prompting enables detection of manipulative patterns base LLMs miss
- Mechanism: CoT injects domain expertise by providing explicit reasoning steps and examples of manipulation tactics, guiding the model to reason like an expert
- Core assumption: LLM possesses sufficient underlying reasoning capabilities to follow provided logical steps
- Evidence anchors: Abstract mentions acquiring professional trading knowledge through CoT; efficacy supported by related meme detection work
- Break condition: Failure if manipulation strategies evolve faster than few-shot examples or if LLM hallucinates reasoning

### Mechanism 3
- Claim: Weighted aggregation of agent confidence scores yields higher predictive accuracy than individual agents
- Mechanism: Agent confidence scores (derived from logits) are combined using optimized weight vectors to maximize AUC
- Core assumption: LLM confidence scores are well-calibrated indicators of prediction probability
- Evidence anchors: Section 5.1.3 describes weight vector optimization for AUC; section 5.2 shows CoT improves precision and F1
- Break condition: If agent confidence scores are systematically overconfident on out-of-distribution data

## Architecture Onboarding

**Component map:** Data Layer (Flipside Solana data + pump.fun comments) -> Feature Layer (bot detection algorithms + statistical metrics) -> Agent Layer (three GPT-4o agents with CoT prompts) -> Aggregation Layer (weighted confidence scores)

**Critical path:**
1. Extract transaction history and calculate statistical features
2. Run bot detection algorithms to generate boolean flags
3. Inject features + bot flags + multimodal data into specialized agents
4. Extract True/False logits to form probability vector
5. Apply optimized weights to get final profitability prediction

**Design tradeoffs:**
- Interpretability vs. Latency: CoT approach provides explainable decisions but introduces higher latency and token costs
- Recall vs. Precision: System optimizes for AUC balancing trade-off, whereas simple copy-trading often prioritizes high recall

**Failure signatures:**
- Gradual Bundle False Negatives: Harder to detect, may require visual pattern recognition the agent might miss
- Imitation Penalty Exhaustion: Model may correctly predict profitability but "imitation penalty" exceeds gross return

**First 3 experiments:**
1. Baseline Validation: Reproduce "Imitation Penalty" calculation on sample trades to verify bonding curve math
2. Prompt Sensitivity Test: Run Coin Agent on known "Bundle Bot" coins with and without CoT prompt
3. Ablation Study: Run full system on test set, systematically removing one agent at a time to observe AUC drop

## Open Questions the Paper Calls Out

**Open Question 1:** How can the framework be extended to accurately detect "gradual bundle" bots that obfuscate temporal and attribution signals?
- Basis: Section 4.3.5 states gradual bundles "obscure temporal signals and attribution, making gradual bundles much harder to detect accurately"
- Why unresolved: Paper lacks rigorous detection metric for gradual bundling, relying on subjective interpretation
- What evidence would resolve it: Formalized detection algorithm with quantifiable precision/recall metrics

**Open Question 2:** Can the structural "Imitation Penalty" be minimized through alternative execution strategies rather than immediate replication?
- Basis: Theorem 2 proves copier "strictly overpays" due to bonding curve mechanics; statistic-driven models yield negative copier returns
- Why unresolved: Current implementation assumes immediate replication; doesn't explore if dynamic entry could reduce price impact penalty
- What evidence would resolve it: Comparative analysis of different trade execution strategies showing reduced gap between smart money and copier returns

**Open Question 3:** Does the multi-agent system maintain effectiveness on blockchain architectures with different transaction ordering rules (e.g., Ethereum)?
- Basis: Methodology relies heavily on Solana-specific features like sub-second block times and absence of public mempool
- Why unresolved: Bot strategies are likely dependent on infrastructure specifics; unclear if same heuristics apply to EVM-based chains
- What evidence would resolve it: Evaluation on Ethereum or Base dataset with AMM liquidity pools

## Limitations
- Temporal generalization risk due to rapidly evolving crypto manipulation tactics
- Economic mechanism validation uncertainty due to vague copier implementation details
- Prompt engineering opacity making approach difficult to reproduce and evaluate
