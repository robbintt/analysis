# Denoising and Adaptive Online Vertical Federated Learning for Sequential Multi-Sensor Data in Industrial Internet of Things

## Quick Facts
- arXiv ID: 2501.01693
- Source URL: https://arxiv.org/abs/2501.01693
- Reference count: 40
- Proposes DAO-VFL algorithm achieving O(√T + Tβd) regret bound for industrial IoT sequential multi-sensor data

## Executive Summary
This paper introduces the Denoising and Adaptive Online Vertical Federated Learning (DAO-VFL) algorithm to address the challenges of communication noise and sensor heterogeneity in industrial IoT environments with sequential multi-sensor data. The approach combines denoising autoencoders for noise reduction with an adaptive local iteration mechanism controlled by deep reinforcement learning to optimize communication efficiency and learning accuracy. Theoretical analysis establishes a regret bound of O(√T + Tβd), demonstrating the algorithm's convergence properties under communication noise conditions.

The proposed framework is validated through experiments on both image classification (CIFAR-10) and real-world industrial datasets (C-MAPSS), showing superior performance compared to baseline federated learning approaches. The DAO-VFL algorithm effectively reduces noise-induced errors while maintaining communication efficiency, making it particularly suitable for resource-constrained industrial IoT scenarios where sensor data quality and communication bandwidth are critical concerns.

## Method Summary
DAO-VFL addresses the challenge of online vertical federated learning in industrial IoT environments where sequential multi-sensor data arrives continuously from distributed sensors with varying quality and communication constraints. The algorithm employs denoising autoencoders to process and clean noisy sensor data before federated aggregation, while a deep reinforcement learning controller dynamically adjusts local iteration counts at each sensor to balance communication efficiency against learning accuracy. The framework operates in an online learning setting where models are updated incrementally as new data arrives, with theoretical regret bounds established for the convergence behavior under communication noise conditions.

## Key Results
- Achieves O(√T + Tβd) regret bound in presence of communication noise
- Demonstrates superior noise reduction and learning accuracy on CIFAR-10 and C-MAPSS datasets
- Reduces total communication latency compared to benchmark federated learning approaches
- Shows effective handling of sensor heterogeneity through adaptive local iteration control

## Why This Works (Mechanism)
The algorithm's effectiveness stems from its dual approach to addressing the core challenges of industrial IoT federated learning. First, the denoising autoencoders learn to filter out communication noise and sensor-specific distortions, creating cleaner feature representations that improve model convergence. Second, the deep reinforcement learning controller optimizes the trade-off between local computation and global communication by adaptively adjusting iteration counts based on current network conditions, sensor data quality, and model convergence state. This adaptive mechanism prevents both underfitting (from insufficient local updates) and overfitting (from excessive local updates without global synchronization), while minimizing unnecessary communication overhead.

## Foundational Learning
**Online Learning**: Sequential decision-making framework where models update incrementally as new data arrives; needed because industrial IoT data streams are continuous and non-stationary; quick check: verify model adapts to concept drift in sequential data.

**Vertical Federated Learning**: Collaborative learning approach where different parties hold different feature sets for the same sample; needed because industrial sensors often capture complementary aspects of the same physical phenomena; quick check: confirm feature alignment across heterogeneous sensor data.

**Denoising Autoencoders**: Neural networks trained to reconstruct clean data from noisy inputs; needed because communication channels and sensor hardware introduce various noise patterns; quick check: measure noise reduction effectiveness on corrupted sensor data.

**Deep Reinforcement Learning**: Framework for learning optimal sequential decision policies through interaction with environment; needed to dynamically balance local computation and global communication; quick check: validate policy convergence and generalization across different noise conditions.

**Regret Analysis**: Theoretical framework for measuring online algorithm performance relative to optimal offline solution; needed to establish convergence guarantees under communication noise; quick check: verify regret bounds hold under practical noise distributions.

## Architecture Onboarding

**Component Map**: Sensor Data → Denoising Autoencoder → Local Model Updates → DRL Controller → Global Aggregation → Model Updates

**Critical Path**: Raw sensor data flows through denoising autoencoders, undergoes local model updates, passes through DRL controller for iteration decisions, and reaches global aggregation for synchronization. The DRL controller acts as the decision point that determines when local models should synchronize with the global model.

**Design Tradeoffs**: The framework balances noise reduction quality (more complex denoising autoencoders) against computational overhead, and communication frequency (adaptive iterations) against learning accuracy. Higher local iteration counts reduce communication but risk model divergence, while aggressive denoising may remove useful signal along with noise.

**Failure Signatures**: Communication bottlenecks trigger excessive local iterations leading to model divergence; overly aggressive denoising creates feature loss degrading model performance; DRL controller convergence failure results in suboptimal iteration decisions and poor learning efficiency.

## Limitations
- Theoretical regret bounds assume idealized communication noise conditions that may not reflect real-world industrial environments
- Scalability concerns with the DRL-based adaptive mechanism when handling large sensor networks
- Limited empirical validation with only two datasets and absence of ablation studies
- Computational overhead of denoising autoencoders may be prohibitive in resource-constrained industrial settings
