# The Cognitive Bandwidth Bottleneck: Shifting Long-Horizon Agent from Planning with Actions to Planning with Schemas
## Quick Facts
- arXiv ID: 2510.07091
- Source URL: https://arxiv.org/abs/2510.07091
- Reference count: 40
- Primary result: Identified inflection point (35-500 actions) where planning with schemas outperforms planning with actions due to EU overload

## Executive Summary
This paper investigates optimal action representation for long-horizon agents as environmental action spaces scale. The authors introduce the Cognitive Bandwidth Perspective framework to decompose agent workflows into stages and diagnose performance bottlenecks. Through systematic experiments across four environments with varying action space sizes, they identify an inflection point where planning with schemas (PwS) becomes superior to planning with actions (PwA) due to cognitive overload from parsing long action lists. The study reveals that stronger planning proficiency shifts this inflection point rightward while better schema instantiation shifts it leftward, providing actionable insights for building more capable autonomous agents.

## Method Summary
The authors develop a framework that decomposes agent workflows into action parsing and schema instantiation phases to identify cognitive bottlenecks. They conduct controlled experiments across four environments: TextCraft (~7 actions), WebShop (~20 actions), ALFWorld (~35 actions), and SciWorld (~500 actions). The study compares PwA versus PwS approaches, measuring performance as environmental action space scales. Ablation studies systematically vary model capabilities in planning proficiency and schema instantiation to understand how these factors shift the performance inflection point. The experimental design isolates the impact of action representation choices on agent performance across different task complexities.

## Key Results
- PwA outperforms PwS in ALFWorld (35 actions) but PwS becomes superior in SciWorld (500 actions)
- Identified inflection point between 35 and 500 actions where representation choice shifts
- Model planning proficiency shifts inflection point rightward; schema instantiation proficiency shifts it leftward
- PwS offers better asymptotic scalability for real-world autonomy applications

## Why This Works (Mechanism)
The cognitive bandwidth bottleneck emerges when agents must parse increasingly long action lists, creating an EU (Efficiency-Utility) overload that degrades performance. PwA requires continuous parsing of action spaces, while PwS abstracts actions into reusable schemas that reduce cognitive load. The framework decomposes workflows to isolate where bottlenecks occur, revealing that action parsing becomes increasingly expensive as action space grows. Model capabilities modulate this relationship: stronger planning can handle larger action spaces before bottlenecking, while better schema instantiation reduces reliance on raw action parsing.

## Limitations
- Framework assumes clear separation between action parsing and schema instantiation phases
- Inflection point identification based on only four environments may not generalize
- Scaling hypothesis for PwS superiority remains theoretical beyond tested action space sizes

## Confidence
**High Confidence**: Experimental results showing PwA outperforming PwS in ALFWorld and PwS outperforming PwA in SciWorld are well-supported.
**Medium Confidence**: Generalization to other domains and action space structures beyond tested environments.
**Low Confidence**: Precise numerical location of inflection point may vary significantly across different task types.
