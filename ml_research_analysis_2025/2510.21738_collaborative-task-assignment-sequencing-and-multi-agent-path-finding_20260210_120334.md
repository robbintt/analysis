# Collaborative Task Assignment, Sequencing and Multi-agent Path-finding

## Quick Facts
- arXiv ID: 2510.21738
- Source URL: https://arxiv.org/abs/2510.21738
- Reference count: 36
- Primary result: CBS-TS algorithm achieves >90% success rate for up to 20 agents/tasks on 8×8 grids, outperforming CBSS baseline which drops to 5.88% for 30 agents/tasks

## Executive Summary
This paper introduces CBS-TS, a two-level algorithm that simultaneously solves task assignment, sequencing, and multi-agent pathfinding (TSPF) while minimizing flowtime. The method combines a high-level Mixed-Integer Linear Programming (MILP) formulation for optimal task sequencing with a low-level Conflict-Based Search (CBS) using Multi-Label A* (MLA*) for collision-free path planning. Experiments demonstrate CBS-TS outperforms the baseline CBSS method in success rate and solution quality across various agent-task configurations on grid maps with obstacles.

## Method Summary
CBS-TS employs a hierarchical approach where an MILP optimizer generates optimal task sequences at the high level, while a Multi-Label A* planner finds collision-free paths at the low level. The algorithm maintains a search forest of partial solutions, expanding nodes by either generating new task sequences via MILP with exclusion constraints or resolving conflicts by splitting into child nodes with additional constraints. Two variants exist: CBS-TS_d applies direct exclusion constraints while CBS-TS_i uses iterative constraint generation. The method handles agent-task compatibility constraints and minimizes total flowtime through careful sequencing and path coordination.

## Key Results
- CBS-TS achieves success rates above 90% for up to 20 agents and 20 tasks on 8×8 grids with 20% obstacles
- CBSS baseline success rate drops to 5.88% for 30 agents and 30 tasks under same conditions
- CBS-TS consistently finds optimal solutions while CBSS fails on larger problem instances
- Runtime is dominated by MILP optimization (over 80% of total time in most cases)

## Why This Works (Mechanism)
The algorithm's effectiveness stems from its two-level decomposition that leverages the strengths of both optimization and search-based planning. The MILP formulation provides globally optimal task sequencing considering compatibility constraints and sub-tour elimination, while the CBS-MLA* combination efficiently handles the combinatorial explosion of collision-free path planning. By maintaining a search forest of partial solutions and alternating between sequence generation and conflict resolution, the method systematically explores the solution space without getting trapped in local optima.

## Architecture Onboarding

**Component Map:**
MILP Optimizer -> Search Forest Manager -> MLA* Path Planner -> Conflict Detector

**Critical Path:**
Task Sequence Generation (MILP) -> Path Planning (MLA*) -> Conflict Detection -> Constraint Application -> New Sequence Generation

**Design Tradeoffs:**
The two-level approach balances optimality (MILP) against computational tractability (CBS). While MILP guarantees optimal sequences, it becomes computationally expensive for large P_sets, necessitating the CBS framework to manage solution space exploration. The choice between CBS-TS_d and CBS-TS_i affects constraint propagation efficiency.

**Failure Signatures:**
- MILP infeasibility when P_set grows large, preventing new root nodes
- Runtime dominated by MILP optimization (>80% of total time)
- MLA* failure under heavy constraints, returning None paths
- Success rate degradation on larger agent-task configurations

## Open Questions the Paper Calls Out

**Open Question 1:**
How can the CBS-TS framework be extended to guarantee dynamic feasibility for non-holonomic agents directly within the planning phase, rather than relying on post-processing? The authors state: "For future work, we plan to incorporate nonlinear optimization techniques to smooth trajectories, ensuring dynamic feasibility for non-holonomic agents." The current method plans paths on a grid assuming holonomic movement and uses a separate Model Predictive Controller (MPC) afterward to time-parameterize the trajectory, which creates a disconnect between the collision-free path found and the actual dynamic constraints. Evidence would resolve it: A variation of the low-level Multi-Label A* that utilizes motion primitives satisfying kinodynamic constraints, ensuring the output path is executable without smoothing modifications.

**Open Question 2:**
How does CBS-TS perform in highly constrained topologies, such as narrow corridors where deadlock prevention is required? The authors list as future work: "testing in constrained environments, such as narrow corridors where only one agent can pass at a time." The experiments were conducted on 8x8 and 16x16 grids with 20% random obstacles, which generally allow for alternative routing; the behavior in strictly bounded spaces where agents must actively yield or reverse is not evaluated. Evidence would resolve it: Benchmark results on map sets featuring bottlenecks (e.g., warehouse aisles or maze-like structures) comparing success rates and flowtime against the baseline CBSS.

**Open Question 3:**
Can the current formulation be modified to allow for active repositioning after task completion to facilitate the movement of other agents? The authors identify the need to evaluate: "scenarios requiring agents to reposition after completing all tasks to allow others to proceed." The current problem definition enforces a terminal condition where an agent stops exactly at its last assigned goal location and remains there, potentially blocking high-traffic areas in dense scenarios. Evidence would resolve it: An altered MILP objective or terminal constraint that minimizes "occlusion" or maximizes clearance, demonstrating that agents autonomously move to designated waiting zones after task fulfillment.

## Limitations
- Runtime dominated by MILP optimization, becoming prohibitive for large agent-task configurations
- Limited evaluation to grid maps with random obstacles, not testing narrow corridors or highly constrained environments
- Current formulation doesn't guarantee dynamic feasibility for non-holonomic agents, requiring post-processing
- Success rate degrades significantly for scenarios with 30+ agents/tasks due to computational complexity
