# Accelerating Outlier-robust Rotation Estimation by Stereographic Projection

## Quick Facts
- arXiv ID: 2502.06337
- Source URL: https://arxiv.org/abs/2502.06337
- Authors: Taosi Xu; Yinlong Liu; Xianbo Wang; Zhi-Xin Yang
- Reference count: 40
- Primary result: Robust rotation estimation with 0.01° error in 0.07s on GPU, handling 90% outliers

## Executive Summary
This paper introduces a novel method for rotation estimation from 3D point sets with heavy outlier contamination. The key innovation is decoupling rotation into axis and angle components, then using stereographic projection to transform the 3D sphere axis estimation problem into a 2D plane problem. This enables efficient voting-based identification of optimal rotation axes. The method demonstrates exceptional speed and accuracy, processing 10^6 points with 90% outliers in 0.07 seconds with 0.01° angular error on GPU, while achieving 100% success rate on the 3DMatch dataset.

## Method Summary
The method operates by first decoupling rotation into axis and angle components. For axis estimation, it employs stereographic projection to map the 3D rotation axis (lying on a unit sphere) to a 2D plane, transforming the problem into a 2D clustering task. This allows efficient voting-based identification of optimal rotation axes. The angle is then estimated separately using least-squares fitting. The approach can simultaneously estimate multiple rotations and is specifically designed to handle severely corrupted data with up to 90% outliers. GPU acceleration is leveraged for the voting and clustering operations, enabling the reported 0.07-second runtime for large datasets.

## Key Results
- Processes 10^6 points with 90% outliers in 0.07 seconds with 0.01° angular error on GPU
- Achieves 100% success rate on 3DMatch dataset while being 30× faster than closest competitors
- Superior accuracy and speed versus state-of-the-art methods including ARCS+OR, RANSAC, FGR, and deep learning approaches

## Why This Works (Mechanism)
The stereographic projection transforms the non-linear 3D sphere clustering problem into a 2D plane clustering problem, which can be solved efficiently using standard voting schemes. By decoupling rotation into axis and angle components, the method avoids the computational complexity of direct 3D rotation search while maintaining robustness to outliers through the voting mechanism.

## Foundational Learning
- Stereographic projection: A mapping from sphere to plane that preserves angles and circles. Needed for transforming 3D axis estimation to 2D clustering. Quick check: Verify that circles on sphere map to circles or lines on plane.
- Rotation decomposition: Breaking rotation into axis-angle representation rather than quaternion/matrix form. Needed for isolating the geometrically meaningful component for outlier rejection. Quick check: Confirm that axis and angle can be recovered from rotation matrix.
- Voting-based clustering: Accumulating evidence for hypothesis in parameter space. Needed for identifying dominant rotation axes in presence of outliers. Quick check: Test voting performance with synthetic data of known rotation and varying outlier ratios.

## Architecture Onboarding

Component map: Point pairs -> Rotation candidates -> Stereographic projection -> 2D voting map -> Axis clustering -> Angle estimation

Critical path: The stereographic projection and 2D voting steps form the computational bottleneck. The method requires efficient GPU implementation of voting accumulation and peak detection in the projected space.

Design tradeoffs: The decoupling approach sacrifices the ability to directly optimize rotation error in favor of computational efficiency. The method assumes rotation axes are sufficiently separated for clustering, which may limit performance when multiple similar rotations exist.

Failure signatures: Poor performance when rotation axes are close together (below clustering resolution), when outliers are not uniformly distributed, or when the stereographic projection introduces excessive distortion.

First experiments: 1) Test axis estimation accuracy with synthetic rotations of known axes, 2) Benchmark voting resolution sensitivity by varying grid size, 3) Compare computational time breakdown between CPU and GPU implementations.

## Limitations
- Computational efficiency claims heavily depend on GPU implementation, with unverified CPU performance
- Exceptional 0.01° angular error lacks statistical confidence intervals or sensitivity analysis across different conditions
- 100% success rate claim on 3DMatch dataset appears overly optimistic given dataset variability
- Limited validation of behavior under correlated outliers, systematic errors, or non-uniform sampling

## Confidence
- High: The core mathematical framework using stereographic projection for axis estimation is sound and novel
- Medium: Computational efficiency gains over traditional methods are likely but exact speedups depend on implementation details
- Low: Generalization claims to arbitrary outlier distributions and real-world datasets lack sufficient validation
