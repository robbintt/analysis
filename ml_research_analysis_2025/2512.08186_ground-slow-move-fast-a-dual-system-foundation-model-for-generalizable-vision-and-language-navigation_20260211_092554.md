# Ground Slow, Move Fast: A Dual-System Foundation Model for Generalizable Vision-and-Language Navigation

## Quick Facts
- arXiv ID: 2512.08186
- Source URL: https://arxiv.org/abs/2512.08186
- Reference count: 14
- Key outcome: DualVLN achieves 70.7% success rate on VLN-CE and 64.3% SPL, outperforming prior methods

## Executive Summary
This paper introduces DualVLN, a dual-system foundation model for generalizable vision-and-language navigation (VLN). It decouples high-level semantic reasoning from low-level action execution: System 2 (a VLM-based planner) "grounds slowly" by predicting mid-term pixel goals via image-grounded reasoning, while System 1 (a lightweight diffusion-based policy) "moves fast" by generating smooth trajectories from high-frequency RGB inputs and latent features. This design enables robust real-time control and adaptive obstacle avoidance. DualVLN achieves state-of-the-art performance on VLN benchmarks (VLN-CE and VLN-PE), outperforming prior methods with 70.7% success rate on VLN-CE and 64.3% SPL. Real-world experiments demonstrate strong generalization across robot platforms and dynamic obstacle avoidance in diverse scenarios. The approach also introduces Social-VLN, a new benchmark for evaluating navigation in dynamic environments with multiple humanoid agents.

## Method Summary
DualVLN employs a dual-system architecture where System 2 (a 7B parameter VLM) operates at low frequency (2 Hz) to perform semantic reasoning and predict mid-term pixel goals through image-grounded instruction understanding. System 1 (a lightweight Diffusion Transformer) operates at high frequency (30 Hz) to generate smooth trajectories from real-time RGB inputs and latent features extracted from the frozen VLM. The systems are trained sequentially - System 2 is first fine-tuned on large datasets and frozen, then System 1 is trained on goal-reaching data conditioned on System 2's outputs. This decoupling preserves the VLM's generalization capabilities while enabling efficient real-time control. The policy conditions on both explicit pixel goals and implicit latent features from the VLM's hidden states to provide rich, task-relevant guidance.

## Key Results
- Achieves 70.7% success rate on VLN-CE benchmark, outperforming prior methods
- Obtains 64.3% SPL (Success weighted by Path Length) on VLN-CE
- Demonstrates strong real-world generalization across different robot platforms
- Introduces Social-VLN benchmark, showing significant performance drop in dynamic environments (Success Rate decreases by approx. 27%)

## Why This Works (Mechanism)

### Mechanism 1
Decoupled, sequential training preserves VLM generalization while enabling lightweight diffusion policy to specialize for real-time control. The 7B parameter VLM (System 2) is fine-tuned on large datasets and frozen, then a small Diffusion Transformer (System 1) is trained on goal-reaching data conditioned on frozen VLM outputs. This prevents low-level control from degrading VLM's pre-trained semantic reasoning abilities. Core assumption: high-level planning and low-level control tasks have different optimization landscapes that benefit from separation. Evidence: "By decoupling training, the VLM retains its generalization, while System 1 achieves interpretable and effective local navigation." Break condition: If joint training achieves better overall performance without sacrificing System 2's generalization.

### Mechanism 2
Asynchronous dual-system design enables continuous smooth trajectories and dynamic obstacle avoidance by decoupling planning latency from control frequency. System 2 operates at 2 Hz providing mid-term pixel goals, while System 1 operates at 30 Hz continuously generating smooth trajectories based on latest goal and real-time RGB inputs. This allows agent to react to immediate changes without waiting for System 2's slower deliberation. Core assumption: System 2's 2 Hz frequency is sufficient for goal updates, and latency is low enough to not cause instability. Evidence: "System 1 encodes both the RGB features corresponding to the last frame from System 2 at time t and the current observation at time t+k... This asynchronous inference enables continuous and smooth navigation process." Break condition: If System 2 latency causes System 1 to pursue outdated goals, leading to erratic motion or collisions.

### Mechanism 3
Conditioning low-level policy on implicit latent goal improves performance by leveraging rich task-relevant features from VLM. After System 2 predicts pixel goal, learnable latent queries are passed through frozen VLM to extract compact latent feature vector encoding implicit information from VLM's hidden states. System 1's diffusion policy conditions on both pixel goal (for interpretability) and latent goal (for richer guidance). Core assumption: VLM's hidden states contain task-relevant information beyond 2D pixel location. Evidence: "Building upon this, implicit latent features further provide richer and more adaptive guidance for System 1..." Break condition: If extracted latent features are noisy or irrelevant and don't add predictive power beyond pixel goal.

## Architecture Onboarding

**Component map:**
System 2 (VLM) -> Latent Goal Extractor -> System 1 (Diffusion Transformer)

**Critical path:** System 2 grounds instruction into pixel goal. This goal anchors Latent Goal Extractor, which produces implicit feature vector. System 1 takes this feature vector and real-time vision to generate executable trajectory. Failure in System 2's grounding cascades into incorrect goal for System 1.

**Design tradeoffs:**
- Latency vs. Reasoning: System 2 is slow but semantically rich; System 1 is fast but locally focused
- Explicit vs. Implicit Guidance: Pixel goals are interpretable but may be imprecise; latent goals provide richer but black-box guidance; model uses both
- Asynchronous vs. Synchronous: Decoupling loops enables smooth motion but requires System 1 to handle potentially outdated goals

**Failure signatures:**
- "Fragmented Motion": Failure in System 1's diffusion or trajectory tracking
- "Missed Goal": Failure in System 2's pixel-goal grounding or instruction understanding
- "Collision with Dynamic Obstacle": Failure in System 1's reactive policy or System 2 guiding into obstacle's predicted path

**First 3 experiments:**
1. **System 2 Ablation:** Run System 2 alone on validation set. Evaluate pixel-goal grounding accuracy (e.g., L2 distance to ground-truth) to isolate performance.
2. **Latent Goal Ablation:** Train System 1 conditioned only on pixel goal. Compare success rate and trajectory smoothness against full model to measure latent goal's contribution.
3. **Asynchronous Stress Test:** In simulation, introduce fast-moving obstacles and vary System 2's update frequency. Measure collision rate and task success to find critical planning frequency.

## Open Questions the Paper Calls Out

### Open Question 1
How can the dual-system architecture be enhanced to close performance gap between static VLN and dynamic Social-VLN benchmark? Basis: Authors state "there remains considerable room for improvement on this task," noting significant performance drop (Success Rate decreases by approx. 27%) in dynamic settings. Why unresolved: Current policy trained on A*-augmented trajectories may not fully capture social interactions or task recovery after detouring around humanoids. What evidence would resolve it: Developing method that significantly improves Social-VLN success rates or demonstrating specific failure modes in social task recovery.

### Open Question 2
Is System 1's performance strictly bounded by System 2's accuracy, and can this bottleneck be removed without re-coupling systems? Basis: Data Scaling Analysis suggests System 1 saturates because "performance upper bound of System 2 has been reached," implying low-level controller limited by high-level planner's errors. Why unresolved: Unclear if System 1 can learn to be robust to System 2 errors beyond certain threshold or if performance gains must come from improving System 2. What evidence would resolve it: Ablation studies showing System 1 performance improving independently of System 2 improvements, or analysis quantifying error propagation from System 2 to System 1.

### Open Question 3
Can framework maintain robust navigation if System 2's update frequency drops significantly below 2 Hz due to computational constraints? Basis: Paper highlights asynchronous design (System 2 at 2 Hz vs. System 1 at 30 Hz) but relies on specific latency setup; robustness to severe degradation of "slow" system's responsiveness is untested. Why unresolved: In real-world deployments, VLM inference can be variable; policy's ability to interpolate using "outdated" latent goals over longer horizons is untested. What evidence would resolve it: Stress-testing Success Rate and Collision Rate while systematically reducing System 2 update frequency.

## Limitations

- Dual-system architecture introduces significant complexity that may limit real-world deployment
- Performance heavily depends on frozen VLM's ability to generalize across diverse environments
- Latent goal mechanism is not adequately justified - insufficient ablation showing meaningful contribution beyond pixel goal

## Confidence

- **High Confidence:** Architectural framework of decoupling high-level planning from low-level execution is sound and supported by strong benchmark results (70.7% success rate on VLN-CE, 64.3% SPL)
- **Medium Confidence:** Claim that conditioning on latent features improves performance is plausible but under-validated
- **Low Confidence:** Assertion that dual-system design inherently enables better dynamic obstacle avoidance lacks rigorous validation

## Next Checks

1. **Cross-Environment Generalization Test:** Evaluate DualVLN on completely unseen environments with varying layouts, textures, and lighting conditions to assess true generalization capabilities.

2. **Dynamic Obstacle Stress Test:** Systematically vary obstacle speed, size, and predictability in simulation while measuring collision rates, success rates, and system latency. Compare performance against high-frequency single-system baseline.

3. **Latent Feature Analysis:** Perform ablation studies removing latent goal conditioning to quantify contribution. Additionally, analyze latent features using visualization techniques to determine if they encode meaningful semantic information beyond pixel goal coordinates.
