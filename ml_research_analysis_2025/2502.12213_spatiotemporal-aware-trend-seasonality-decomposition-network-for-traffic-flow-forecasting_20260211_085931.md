# Spatiotemporal-aware Trend-Seasonality Decomposition Network for Traffic Flow Forecasting

## Quick Facts
- arXiv ID: 2502.12213
- Source URL: https://arxiv.org/abs/2502.12213
- Authors: Lingxiao Cao; Bin Wang; Guiyuan Jiang; Yanwei Yu; Junyu Dong
- Reference count: 11
- Achieves state-of-the-art performance on three real-world traffic flow datasets, including the newly released urban JiNan dataset

## Executive Summary
This paper introduces the Spatiotemporal-aware Trend-Seasonality Decomposition Network (STDN), a novel approach for traffic flow forecasting that addresses the limitations of existing methods in modeling complex spatial and temporal dynamics. The method constructs a dynamic graph structure to represent traffic flow and incorporates novel spatio-temporal embeddings to capture global traffic dynamics. A key innovation is the trend-seasonality decomposition module, which disentangles the trend-cyclical and seasonal components for each traffic node at different times within the graph. These components are then processed through an encoder-decoder network to generate final predictions. The model achieves state-of-the-art performance on three real-world datasets, including a newly released urban dataset named JiNan.

## Method Summary
STDN employs a dynamic graph learning framework to capture evolving traffic relationships. It introduces a novel spatio-temporal embedding mechanism that leverages both static and dynamic graph properties to encode node features. The core innovation is a trend-seasonality decomposition module that separates traffic patterns into trend-cyclical and seasonal components using Hadamard-product-based operations on hidden states and embeddings. These decomposed components are then processed through an encoder-decoder network architecture to generate final traffic flow predictions. The approach is evaluated on three real-world datasets, including a newly released urban dataset named JiNan.

## Key Results
- Achieves state-of-the-art performance on three real-world traffic flow datasets
- Consistently outperforms various competing baselines across multiple evaluation metrics
- Demonstrates remarkable computation cost efficiency while maintaining superior accuracy
- Ablation study confirms the effectiveness of each proposed module in the architecture

## Why This Works (Mechanism)
The method works by explicitly modeling the complex interplay between spatial dependencies and temporal patterns in traffic flow. By constructing dynamic graphs, it captures evolving relationships between traffic nodes. The trend-seasonality decomposition allows the model to separately learn long-term trends and recurring patterns, which are then recombined through the encoder-decoder network. This separation of concerns enables more accurate modeling of both persistent and periodic traffic behaviors while filtering out noise.

**Why needed:** Traffic patterns evolve over time, requiring adaptive graph structures rather than fixed relationships.

## Architecture Onboarding

### Component Map
Dynamic Graph Construction -> Spatio-Temporal Embeddings Learning -> Trend-Seasonality Decomposition -> Encoder-Decoder Network -> Prediction

### Critical Path
The critical path flows through the trend-seasonality decomposition module, as it directly influences the quality of inputs to the encoder-decoder network. The decomposition must accurately separate trend and seasonal components for optimal performance.

### Design Tradeoffs
The model trades computational complexity for improved accuracy by explicitly modeling trend and seasonal components separately. While this adds processing steps, it enables more precise capture of traffic patterns compared to end-to-end approaches.

### Failure Signatures
The model may struggle with irregular events that break standard periodicity assumptions, particularly holidays or major events that disrupt typical traffic patterns. Additionally, rapid structural changes in the road network may not be captured if the static graph components are too rigid.

## Limitations
- Limited evaluation on larger-scale traffic networks, with experiments focused on medium-sized urban datasets
- Potential computational bottlenecks in dynamic graph construction for very large networks
- Reliance on strict periodic temporal features may limit performance during irregular events or holidays
