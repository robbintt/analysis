# Noise Robust One-Class Intrusion Detection on Dynamic Graphs

## Quick Facts
- **arXiv ID**: 2508.14192
- **Source URL**: https://arxiv.org/abs/2508.14192
- **Reference count**: 8
- **Primary result**: RTGN-SVDD achieves 84.4% ROC-AUC at 50% noise vs 57.4% for baseline TGN-SVDD

## Executive Summary
This paper addresses intrusion detection in dynamic network graphs under noisy conditions by extending the Temporal Graph Network Support Vector Data Description (TGN-SVDD) model. The proposed Robust TGN-SVDD (RTGN-SVDD) introduces probabilistic modeling of network events through Gaussian distribution parameters, enabling robust detection in environments with significant noise. The model employs a dual-score system: one to identify noisy events and another for attack detection. Extensive experiments on the CIC-IDS2017 dataset with synthetic noise demonstrate substantial performance improvements over the baseline, particularly at high noise levels.

## Method Summary
The RTGN-SVDD model extends the TGN-SVDD framework by predicting Gaussian distribution parameters for each network event rather than deterministic embeddings. This probabilistic approach enables the model to quantify uncertainty and distinguish between noise and actual attacks. The model uses two distinct scores: a noise score based on the divergence between predicted and observed distributions, and an attack score based on the SVDD objective. During training, the model learns to minimize both scores while maintaining separation between normal and attack patterns. The Gaussian parameterization allows the model to naturally handle variability in network traffic patterns while maintaining robustness to noise injection.

## Key Results
- RTGN-SVDD achieves 84.4% ROC-AUC at 50% synthetic noise levels on Tuesday CIC-IDS2017 data
- Outperforms baseline TGN-SVDD (57.4% ROC-AUC) by 27 percentage points at same noise level
- Performance advantage increases with noise level, demonstrating enhanced robustness
- Model shows consistent improvements across multiple days of CIC-IDS2017 dataset

## Why This Works (Mechanism)
The model's effectiveness stems from its probabilistic treatment of network events through Gaussian distribution prediction. By modeling each event as a distribution rather than a point embedding, the model can naturally capture uncertainty and variability in network traffic. The dual-score architecture allows independent optimization for noise detection and attack detection, preventing these competing objectives from interfering with each other. The Gaussian parameterization provides a principled way to measure similarity between events, with the model learning to assign similar distributions to events from the same class while maintaining separation between normal and attack patterns.

## Foundational Learning
**Gaussian distribution parameterization**: Predicts mean and variance for each network event to capture uncertainty - needed for robust noise handling, quick check: verify parameter prediction accuracy
**Temporal Graph Networks (TGNs)**: Processes dynamic graph data with memory mechanisms for node representations - needed for handling evolving network topology, quick check: ensure temporal dependencies are captured
**Support Vector Data Description (SVDD)**: One-class classification technique that finds hypersphere enclosing normal data - needed for anomaly detection without labeled attacks, quick check: verify sphere boundary optimization
**Dual-score architecture**: Separate scores for noise and attack detection - needed to prevent objective interference, quick check: validate score independence
**Distribution divergence metrics**: Measures difference between predicted and observed distributions - needed for noise quantification, quick check: confirm appropriate divergence choice

## Architecture Onboarding
**Component map**: Graph data → TGN encoder → Gaussian parameter predictor → Distribution divergence calculator → Noise score → SVDD scorer → Combined decision
**Critical path**: Input events → Temporal encoding → Distribution prediction → Noise score calculation → Attack score calculation → Final classification
**Design tradeoffs**: Probabilistic modeling vs computational overhead, dual scores vs model complexity, Gaussian assumption vs flexibility
**Failure signatures**: High noise score with low attack score (false positive), low noise score with high attack score (missed detection), parameter prediction collapse
**First experiments**: 1) Test Gaussian parameter prediction accuracy on clean data, 2) Validate noise score sensitivity to synthetic noise injection, 3) Benchmark attack detection performance at varying noise levels

## Open Questions the Paper Calls Out
The paper suggests future work extending the model to additional datasets and integrating its noise quantification with other detection systems, though specific open questions are not explicitly enumerated in the provided content.

## Limitations
- Evaluation limited to single dataset (CIC-IDS2017) with synthetic noise
- Gaussian distribution assumption may not capture all network traffic patterns
- Computational overhead of probabilistic modeling may impact real-time deployment
- Synthetic noise generation methodology not detailed, questioning ecological validity
