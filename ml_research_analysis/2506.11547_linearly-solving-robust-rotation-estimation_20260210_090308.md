# Linearly Solving Robust Rotation Estimation
## Quick Facts
- arXiv ID: 2506.11547
- Source URL: https://arxiv.org/abs/2506.11547
- Authors: Yinlong Liu; Tianyu Huang; Zhi-Xin Yang
- Reference count: 40
- Key outcome: Novel linear formulation for robust rotation estimation achieving O(N) complexity

## Executive Summary
This paper introduces a fundamentally different approach to robust rotation estimation by transforming the problem into a linear model fitting task. The method represents rotation constraints as great circles (quaternion circles) on the unit quaternion sphere, where each observation pair generates a circle and the solution corresponds to their intersection. By leveraging stereographic projection to map these circles to 3D curves, the approach enables efficient voting-based computation that can handle massive datasets with severe corruption.

The proposed method demonstrates remarkable performance, solving rotation estimation problems with 10^6 points and 99% outlier ratios in under 0.5 seconds using GPU acceleration. This represents a significant advance over existing methods, particularly for large-scale applications where computational efficiency and robustness to corruption are critical requirements.

## Method Summary
The core innovation transforms rotation estimation into a linear model fitting problem by representing rotation constraints as great circles on the quaternion sphere. Each observation pair generates a quaternion circle, and the rotation solution is found at the intersection of these circles. The method employs stereographic projection to map quaternion circles to 3D curves, enabling efficient voting-based computation. This geometric reformulation allows the algorithm to achieve linear time complexity O(N) while maintaining robustness against severe outliers, with GPU acceleration enabling processing of 10^6 points in under 0.5 seconds.

## Key Results
- Achieves linear time complexity O(N) for rotation estimation
- Handles 10^6 points with 99% outlier ratio in under 0.5 seconds (GPU)
- Demonstrates superior accuracy and efficiency compared to state-of-the-art methods

## Why This Works (Mechanism)
The method works by geometrically reformulating rotation constraints as great circles on the unit quaternion sphere. Each observation pair defines a quaternion circle, and the true rotation lies at the intersection of all these circles. Stereographic projection maps these 4D quaternion circles to 3D curves while preserving their intersection properties. The voting mechanism efficiently identifies the intersection point by aggregating evidence across all curves, with GPU parallelization enabling the linear time complexity despite the massive number of votes cast.

## Architecture Onboarding
Component map: Observation pairs -> Quaternion circles -> Stereographic projection -> 3D curves -> Voting accumulation -> Rotation solution

Critical path: The stereographic projection and voting accumulation stages are most critical for performance. Any numerical instability in projection mapping directly impacts voting accuracy, while inefficient voting implementation prevents achieving O(N) complexity.

Design tradeoffs: The method trades geometric elegance for computational efficiency, using stereographic projection to convert a complex 4D intersection problem into tractable 3D voting. This introduces projection-related numerical errors but enables massive parallelization.

Failure signatures: Poor outlier rejection occurs when quaternion circles become ill-conditioned or when projection mapping loses intersection fidelity. Performance degradation happens with adversarial outliers that create artificial voting peaks away from the true solution.
