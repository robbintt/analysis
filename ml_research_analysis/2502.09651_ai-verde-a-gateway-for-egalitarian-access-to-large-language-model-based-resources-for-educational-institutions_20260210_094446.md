# AI-VERDE: A Gateway for Egalitarian Access to Large Language Model-Based Resources For Educational Institutions

## Quick Facts
- arXiv ID: 2502.09651
- Source URL: https://arxiv.org/abs/2502.09651
- Reference count: 40
- One-line primary result: University-focused platform enabling seamless access to LLMs for academic use, deployed at University of Arizona with 78 users, 110M+ tokens, and 97K+ API calls over six months

## Executive Summary
AI-VERDE is a university-focused platform enabling seamless access to large language models (LLMs) for academic use. It addresses barriers like privacy, technical complexity, and limited access by providing a unified gateway to both commercial and open-source models, with features such as on-premises deployment, RAG support, automated access control, and budget management. Deployed at the University of Arizona, it supported 78 users across courses and research projects, handling over 110 million tokens and 97,000 API calls in six months. By integrating authentication systems, offering tailored support, and reducing reliance on expensive commercial services, AI-VERDE democratizes AI access and supports diverse academic needs while ensuring data privacy and compliance.

## Method Summary
AI-VERDE was implemented as a unified gateway platform at the University of Arizona to provide academic users with access to both commercial and open-source LLMs. The system integrates with institutional authentication, enforces automated access control, and manages budget allocation for different user groups. The deployment included support for Retrieval-Augmented Generation (RAG) and on-premises hosting options to address privacy and compliance needs. Usage was monitored across courses and research projects over a six-month period, with token usage and API call counts recorded.

## Key Results
- Supported 78 users across multiple courses and research projects
- Processed over 110 million tokens and 97,000 API calls in six months
- Reduced reliance on expensive commercial APIs through integrated access management and budget controls

## Why This Works (Mechanism)
AI-VERDE works by centralizing access to LLM resources behind a unified gateway that abstracts the complexity of interacting with multiple models and providers. The platform integrates institutional authentication to ensure secure access, applies automated access control to enforce usage policies, and manages budgets to prevent overspending. By supporting both commercial and open-source models, it offers flexibility while addressing privacy and compliance through on-premises deployment options. RAG capabilities enhance the utility of LLMs for academic research, and tailored support ensures usability for non-technical users.

## Architecture Onboarding
- **Component map**: User Interface -> Authentication Service -> Access Control Engine -> Budget Manager -> LLM Gateway -> Commercial/Open-source Models
- **Critical path**: User login → Authentication → Policy check → Budget validation → Model selection → Response delivery
- **Design tradeoffs**: On-premises vs. cloud deployment (privacy vs. scalability), commercial vs. open-source models (performance vs. cost)
- **Failure signatures**: Authentication errors (access denied), budget overruns (service interruption), model unavailability (fallback required)
- **First experiments**:
