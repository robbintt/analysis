# GSMT: Graph Fusion and Spatiotemporal TaskCorrection for Multi-Bus Trajectory Prediction

## Quick Facts
- arXiv ID: 2508.09227
- Source URL: https://arxiv.org/abs/2508.09227
- Reference count: 37
- Primary result: 88.12% short-term, 66.12% long-term prediction accuracy on Kuala Lumpur dataset

## Executive Summary
This paper presents GSMT, a hybrid model for multi-bus trajectory prediction in urban environments that relies solely on GPS data. The model combines Graph Attention Networks (GAT) with sequence-to-sequence Recurrent Neural Networks (RNN) and introduces a task correction module that clusters historical trajectories to identify motion patterns and refine predictions. GSMT treats the entire multi-route bus fleet as a unified trajectory planning system, capturing both direct and indirect bus interactions to enable accurate future trajectory prediction.

The system is specifically designed for regions with limited multimodal data, addressing the challenge of predicting bus movements in complex urban environments. By fusing dynamic bus information with static station data and applying secondary refinement through the task corrector after initial predictions, GSMT demonstrates superior performance in identifying issues like bus bunching and enabling timely interventions for urban transit management.

## Method Summary
GSMT employs a hybrid architecture combining Graph Attention Networks with sequence-to-sequence RNNs to predict bus trajectories. The model processes GPS data from multiple bus routes simultaneously, using GAT to capture spatial relationships between buses and stations while RNNs handle temporal dependencies in movement patterns. The innovative task correction module clusters historical trajectories to identify common motion patterns, then fine-tunes initial predictions based on these learned patterns. This two-stage approach first generates predictions using the hybrid model, then applies secondary refinement through the task corrector to improve accuracy. The entire system treats the multi-route bus fleet as an interconnected network rather than isolated routes.

## Key Results
- Achieved 88.12% accuracy for short-term bus trajectory predictions
- Reached 66.12% accuracy for long-term predictions on Kuala Lumpur dataset
- Successfully identified bus bunching scenarios and enabled timely interventions
- Outperformed existing approaches on the tested dataset

## Why This Works (Mechanism)
The model's effectiveness stems from treating the entire bus fleet as an interconnected system rather than isolated routes. By using GAT, GSMT captures both direct spatial relationships (buses at the same station) and indirect interactions (buses affecting each other's routes through traffic patterns). The task correction module adds a critical refinement layer by identifying historical motion patterns and adjusting predictions accordingly, addressing the limitation of relying solely on real-time GPS data which may miss contextual patterns.

## Architecture Onboarding

Component map: GPS data -> GAT layer -> RNN sequence encoder -> Initial prediction -> Task corrector (clustering + refinement) -> Final prediction

Critical path: The most critical sequence is GPS input through GAT spatial processing, then RNN temporal encoding, followed by the task correction module. The clustering step in the corrector is particularly crucial as it identifies the motion patterns that drive the final refinement.

Design tradeoffs: The model trades computational complexity for accuracy by adding the task correction stage. While this increases inference time, it provides the necessary refinement for handling complex urban scenarios. The choice to use only GPS data limits the model's contextual awareness but makes it more practical for deployment in resource-constrained environments.

Failure signatures: The model may struggle with sudden route changes not present in historical data, GPS signal loss or high noise, and scenarios involving multiple simultaneous disruptions. Poor clustering in the task corrector can lead to inappropriate pattern matching and degraded predictions.

## Limitations
- Narrow geographic scope - tested only on Kuala Lumpur dataset, limiting generalizability to other cities
- Performance claims lack benchmarking against other urban datasets and real-world transit scenarios
- Does not address model behavior under data sparsity or sensor failure conditions
- Reliance on GPS data alone may miss richer contextual information from other sensors or traffic signals

## Confidence
High for technical innovation (GAT + RNN integration with task correction), Medium for performance claims (limited external validation), Low for generalizability (single-dataset evaluation)
