# SCoPE VLM: Selective Context Processing for Efficient Document Navigation in Vision-Language Models

## Quick Facts
- arXiv ID: 2510.21850
- Source URL: https://arxiv.org/abs/2510.21850
- Reference count: 40
- 2.38× higher memory efficiency while maintaining competitive accuracy

## Executive Summary
SCoPE VLM addresses the challenge of efficient long-context visual understanding in vision-language models for document navigation and GUI control tasks. The system introduces a Chain of Scroll framework that enables selective, recursive document navigation by focusing only on relevant segments rather than processing entire documents. To train this decision-making capability, the authors create a specialized dataset with pseudo-labeled trajectories and introduce Episodic Group Relative Policy Optimization (EGRPO), a reinforcement learning method that optimizes both terminal and penultimate steps while balancing exploration and exploitation. Experimental results demonstrate significant memory efficiency gains and competitive accuracy across multiple document benchmarks.

## Method Summary
The SCoPE VLM system employs a Chain of Scroll framework that enables selective document navigation by recursively focusing on relevant segments rather than processing entire documents. The training methodology uses pseudo-labeled trajectories combined with Episodic Group Relative Policy Optimization (EGRPO), a reinforcement learning approach that optimizes decision-making at both terminal and penultimate steps. This approach balances exploration and exploitation during training, allowing the model to learn efficient navigation strategies while maintaining accuracy in document understanding and GUI control tasks.

## Key Results
- Achieves 2.38× higher memory efficiency compared to baseline approaches
- Maintains competitive accuracy across multiple document benchmarks
- Demonstrates strong generalization to GUI control tasks with improved navigation and task completion performance

## Why This Works (Mechanism)
The Chain of Scroll framework works by enabling the model to make sequential decisions about which document segments to process, rather than requiring full document context at once. This selective processing approach reduces memory overhead while preserving task-relevant information. The EGRPO training method optimizes the decision-making process by considering both the final outcome and intermediate steps, allowing the model to learn efficient navigation strategies that balance immediate gains with long-term task completion.

*Why needed:* Reduces computational requirements by focusing on relevant document segments
*Quick check:* Memory usage scales sublinearly with document length

## Architecture Onboarding

**Component Map**
Vision Encoder -> Feature Extractor -> Decision Network -> Action Selector -> Memory Manager

**Critical Path**
Vision encoder processes document segments → Feature extractor creates contextual embeddings → Decision network evaluates navigation options → Action selector chooses next segment → Memory manager tracks visited regions and maintains context

**Design Tradeoffs**
Selective processing vs. completeness: Prioritizing efficiency may miss relevant context
Exploration vs. exploitation: Balancing discovery of new information with exploiting known useful paths
Memory efficiency vs. accuracy: Reducing context may impact understanding quality

**Failure Signatures**
Getting stuck in loops when revisiting the same document segments
Missing critical information due to overly aggressive context reduction
Poor generalization to novel document layouts or GUI interfaces

## Limitations
