# Synthetic Data for Robust Runway Detection

## Quick Facts
- **arXiv ID:** 2510.20349
- **Source URL:** https://arxiv.org/abs/2510.20349
- **Reference count:** 29
- **Primary result:** Synthetic data integration improves runway detection model robustness, particularly for nighttime conditions

## Executive Summary
This paper addresses the challenge of limited real-world training data for runway detection in autonomous landing systems. The authors propose generating synthetic runway images using a commercial flight simulator and integrating them with a small real dataset. They evaluate multiple training strategies, including simple mixing and a customized domain adaptation method based on CARE. Their experiments show that synthetic data significantly enhances model robustness, especially for nighttime conditions not present in the real training set. The best results were achieved using the CARE method, improving Average Precision by up to 8.9% on real nighttime images compared to training on real data alone.

## Method Summary
The authors developed a synthetic data generation pipeline using Microsoft Flight Simulator to create diverse runway images under various lighting and weather conditions. They trained a YOLOv8 object detection model using three approaches: training on real data alone, simple mixing of real and synthetic data, and a customized CARE domain adaptation method. The CARE approach fine-tunes a pre-trained model on synthetic data before combining it with real data training. The synthetic dataset included 10,000 images spanning day, night, and dusk conditions, while the real dataset contained 400 images from multiple airports. All models were evaluated on a held-out test set of real images from the same airports.

## Key Results
- Synthetic data integration improved Average Precision by 8.9% on real nighttime images compared to real data only
- The CARE domain adaptation method outperformed simple mixing of real and synthetic data
- Models trained with synthetic data showed improved robustness across different airports and lighting conditions
- Nighttime detection performance improved significantly despite nighttime images being absent from the real training set

## Why This Works (Mechanism)
Synthetic data effectively expands the training distribution to cover rare scenarios like nighttime runway conditions. The commercial flight simulator generates photorealistic images with controllable environmental parameters, allowing targeted augmentation of underrepresented conditions. Domain adaptation methods like CARE help bridge the reality gap by aligning feature distributions between synthetic and real domains, enabling better generalization.

## Foundational Learning
- **Domain adaptation**: Methods for reducing distribution shift between source (synthetic) and target (real) domains - needed to prevent synthetic data from degrading real data performance
- **Object detection with YOLOv8**: Single-stage detector architecture for runway localization - provides the baseline model architecture
- **Photorealistic rendering**: Flight simulator capabilities for generating realistic runway images - enables creation of diverse synthetic training data
- **Transfer learning**: Pre-training on synthetic data before fine-tuning on real data - improves convergence and performance when real data is limited

## Architecture Onboarding

**Component Map:** Flight Simulator -> Synthetic Image Generation -> YOLOv8 Training Pipeline -> Domain Adaptation Module -> Model Evaluation

**Critical Path:** The core pipeline involves generating synthetic runway images in the flight simulator, feeding them through the YOLOv8 training framework, applying domain adaptation techniques, and evaluating on real test data.

**Design Tradeoffs:** Using a commercial flight simulator provides photorealism but limits control over exact environmental parameters compared to custom rendering engines. The choice of YOLOv8 balances detection speed with accuracy requirements for real-time runway detection.

**Failure Signatures:** Poor synthetic-real domain alignment manifests as degraded performance on real test data despite good synthetic data performance. Overfitting to synthetic data characteristics can occur if domain adaptation is insufficient.

**First Experiments:**
1. Train YOLOv8 on real data only as baseline
2. Train on synthetic data only to assess data quality
3. Compare simple mixing vs. CARE domain adaptation performance

## Limitations
- Evaluation based on relatively small real-world dataset (400 images) limiting generalizability
- Synthetic data generation depends on single commercial flight simulator constraining environmental variability
- Domain adaptation method (CARE) tested against only two baseline approaches

## Confidence
- **Core findings: High** - Methodology is sound, experimental design is rigorous, results show consistent improvements
- **Generalizability: Medium** - Promising results but specific application domain may have unique characteristics
