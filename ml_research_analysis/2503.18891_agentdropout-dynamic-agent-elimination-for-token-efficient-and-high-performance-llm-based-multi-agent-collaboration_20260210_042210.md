# AgentDropout: Dynamic Agent Elimination for Token-Efficient and High-Performance LLM-Based Multi-Agent Collaboration

## Quick Facts
- **arXiv ID:** 2503.18891
- **Source URL:** https://arxiv.org/abs/2503.18891
- **Reference count:** 40
- **Primary result:** Reduces prompt tokens by 21.6% and completion tokens by 18.4% while improving task performance by 1.14 points

## Executive Summary
This paper introduces AgentDropout, a dynamic optimization method for multi-agent systems (MAS) that eliminates redundant agents and communication links across rounds. The approach draws inspiration from efficient team role assignment, performing two key operations: Node Dropout (removing less contributory agents) and Edge Dropout (pruning low-impact communication edges). AgentDropout demonstrates significant improvements in token efficiency and task performance while maintaining robustness across different domains.

## Method Summary
AgentDropout implements a two-phase elimination strategy for LLM-based multi-agent collaboration. The method first identifies and removes agents with minimal contribution to task outcomes (Node Dropout), then prunes communication links between agents that provide little value (Edge Dropout). This dynamic pruning occurs across rounds, allowing the system to adapt to changing task requirements and agent effectiveness. The approach uses contribution metrics to evaluate agent and edge importance, enabling targeted elimination that preserves system functionality while reducing computational overhead.

## Key Results
- Reduces prompt token consumption by 21.6% compared to state-of-the-art baselines
- Reduces completion tokens by 18.4% while maintaining system functionality
- Improves task performance by an average of 1.14 points across benchmark tasks
- Demonstrates strong domain transferability and structural robustness

## Why This Works (Mechanism)
AgentDropout works by recognizing that not all agents and communication links contribute equally to task outcomes in multi-agent systems. By dynamically identifying and eliminating redundant components, the method reduces computational overhead while preserving essential collaborative functions. The elimination process is guided by contribution metrics that assess each agent's and communication edge's impact on task performance. This selective pruning approach maintains system capability while significantly reducing token consumption, effectively balancing efficiency with performance.

## Architecture Onboarding

**Component map:** AgentPool -> ContributionEvaluator -> NodeDropout -> EdgeDropout -> OptimizedAgentNetwork

**Critical path:** Agent contributions are evaluated each round, followed by sequential elimination of low-contribution nodes and edges, resulting in a streamlined agent network for subsequent rounds.

**Design tradeoffs:** The method trades potential information loss from elimination against significant token savings and performance improvements; aggressive elimination may reduce redundancy but could compromise robustness in unexpected scenarios.

**Failure signatures:** Over-elimination leading to task failure, premature elimination of potentially useful agents, or retention of redundant agents due to poor contribution metric design.
