# Ontology Creation and Management Tools: the Case of Anatomical Connectivity

## Quick Facts
- arXiv ID: 2509.15780
- Source URL: https://arxiv.org/abs/2509.15780
- Reference count: 29
- Authors present ApiNATOMY framework for multiscale physiological circuit maps

## Executive Summary
ApiNATOMY is a framework designed to represent topological and semantic relationships in multiscale physiological circuit maps. The system enables physiology experts to capture interactions between anatomical entities and convert abstractions into detailed physiological models. By using JSON-LD, the framework ensures FAIR compliance and integrates with external ontologies and knowledge graphs. The platform provides visual knowledge management tools including model viewers, control panels, and resource editors for various physiological components.

## Method Summary
The framework employs a knowledge representation model and management tools to enable physiology experts to work with anatomical connectivity data. It uses JSON-LD to structure application-specific data in a FAIR-compliant manner, integrating with external ontologies and knowledge graphs. The system provides visual tools including model viewers, control panels, relationship graphs, and resource editors for materials, lyphs, chains, and coalescences. Models can be combined through merging, joining, importing, and adding scaffolds, with validation mechanisms to prevent inconsistencies.

## Key Results
- ApiNATOMY framework enables multiscale physiological circuit mapping through topological and semantic representation
- System integrates with SPARC knowledge graph for peripheral nervous system research
- Framework supports model combination through merging, joining, importing, and scaffolding operations
- Visual knowledge management tools include model viewers, control panels, relationship graphs, and resource editors
- JSON-LD implementation ensures FAIR compliance for application-specific data structures

## Why This Works (Mechanism)
The framework works by providing a structured approach to representing anatomical connectivity through topological and semantic modeling. JSON-LD serves as the foundation for FAIR compliance, allowing structured data to be linked and discoverable. The integration with external ontologies and knowledge graphs enables semantic interoperability, while visual tools lower the barrier for physiology experts to work with complex models. The combination of validation mechanisms and model management operations ensures consistency across complex physiological representations.

   - Why needed: Captures structural organization of physiological systems
   - Quick check: Confirm entity relationships maintain topological consistency

3. **Semantic Integration**: Connection with external ontologies and knowledge graphs
   - Why needed: Enables cross-referencing and standardization of terminology
   - Quick check: Validate ontology mappings resolve correctly

4. **Model Validation**: Mechanisms to prevent inconsistent physiological representations
   - Why needed: Ensures scientific accuracy of anatomical circuit models
   - Quick check: Test validation catches known inconsistencies

5. **Visual Knowledge Management**: Interactive tools for non-technical experts
   - Why needed: Lowers barrier for domain experts to work with complex data
   - Quick check: Assess tool usability through domain expert feedback

## Architecture Onboarding

**Component Map**: User Interface -> JSON-LD Parser -> Knowledge Graph -> Validation Engine -> External Ontologies

**Critical Path**: Model Creation → Validation → Integration → Visualization

**Design Tradeoffs**: Flexibility in model representation versus standardization for FAIR compliance; rich visual tools versus potential learning curve for non-technical users; integration capabilities versus dependency management complexity.

**Failure Signatures**: Inconsistent entity relationships, ontology mapping failures, JSON-LD validation errors, visualization rendering issues, model combination conflicts.

**First Experiments**:
1. Create a simple anatomical circuit model and validate JSON-LD output structure
2. Test integration with a basic external ontology and verify semantic mappings
3. Attempt model combination operations (merge, join) with small test datasets

## Limitations
- JSON-LD reliance may limit adoption among researchers unfamiliar with linked data standards
- Integration with external ontologies introduces potential dependencies and compatibility challenges
- Visual tools may present steep learning curve for non-technical physiology experts
- Validation mechanisms may not catch all inconsistencies in complex models
