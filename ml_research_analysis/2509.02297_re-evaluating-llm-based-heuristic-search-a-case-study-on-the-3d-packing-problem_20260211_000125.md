# Re-evaluating LLM-based Heuristic Search: A Case Study on the 3D Packing Problem

## Quick Facts
- **arXiv ID**: 2509.02297
- **Source URL**: https://arxiv.org/abs/2509.02297
- **Reference count**: 40
- **Primary result**: LLM-driven heuristic design requires extensive scaffolding due to fragility, discovering competitive scoring functions but struggling with constraint complexity

## Executive Summary
This paper investigates the application of LLM-based heuristic search to the 3D Packing Problem, revealing significant challenges in directly applying LLM-driven Evolution of Heuristics. The approach proved fragile, generating frequent logical and geometric errors that necessitated two key interventions: constraint scaffolding through verified geometric APIs and iterative self-correction mechanisms. The LLM ultimately discovered a heuristic focused on refining a scoring function within a simple greedy framework, achieving competitive performance with human-designed methods on the base problem and rivaling state-of-the-art solvers when integrated into a metaheuristic. However, performance notably degraded as problem constraints increased, highlighting fundamental limitations in the approach's scalability and generalizability.

## Method Summary
The researchers employed a scaffolded evolutionary framework where LLMs iteratively generated and refined heuristic code for the 3D packing problem. The process incorporated constraint scaffolding via verified geometric APIs to ensure correctness, combined with iterative self-correction cycles to repair errors. The evolutionary approach focused on discovering and optimizing scoring functions within a greedy framework rather than generating entirely novel heuristic structures. This method allowed the LLM to operate within a constrained but effective design space, ultimately producing heuristics that could be integrated into metaheuristic solvers for competitive performance.

## Key Results
- LLM-driven heuristic discovery requires extensive scaffolding due to frequent logical and geometric errors
- The discovered scoring function was competitive with human-designed greedy methods on the base problem
- Performance degraded significantly as problem constraints increased, limiting scalability
- The approach yielded incremental improvements rather than fundamental innovation in heuristic design

## Why This Works (Mechanism)
The mechanism relies on LLMs operating within a carefully constrained framework that compensates for their inherent fragility. By providing verified geometric APIs and iterative correction mechanisms, the approach channels LLM creativity toward productive heuristic discovery while preventing catastrophic errors. The evolutionary framework allows for gradual refinement of promising heuristics rather than requiring perfect initial generation.

## Architecture Onboarding

**Component Map**: Problem Definition -> Constraint Scaffolding -> LLM Code Generation -> Iterative Self-Correction -> Heuristic Evaluation -> Metaheuristic Integration

**Critical Path**: The most critical sequence is Problem Definition -> Constraint Scaffolding -> LLM Code Generation -> Heuristic Evaluation, as errors at any early stage cascade through the entire process.

**Design Tradeoffs**: The approach trades potential for fundamental innovation against reliability and correctness, choosing a constrained design space that ensures LLMs can contribute meaningfully without generating catastrophic errors.

**Failure Signatures**: Common failure modes include logical errors in geometric reasoning, violation of problem constraints, and inability to handle increased complexity. These manifest as incorrect solutions, crashes, or severe performance degradation.
