# Multi-Agent Collaborative Filtering: Orchestrating Users and Items for Agentic Recommendations

## Quick Facts
- arXiv ID: 2511.18413
- Source URL: https://arxiv.org/abs/2511.18413
- Reference count: 17
- Best hit rates achieved (e.g., 0.5238 on Amazon Clothing dataset)

## Executive Summary
This paper introduces Multi-Agent Collaborative Filtering (MACF), a framework that integrates collaborative filtering principles with LLM-based multi-agent systems for agentic recommendations. MACF addresses the limitation of existing agentic recommenders by instantiating user and item agents corresponding to similar users and relevant items in the target user's history. A central orchestrator agent manages dynamic agent recruitment and personalized collaboration instructions across multiple discussion rounds. Experimental results on Amazon Clothing, Beauty, and Music datasets show MACF outperforms strong baselines, achieving best hit rates and competitive NDCG scores.

## Method Summary
MACF operates through a multi-round discussion process where user and item agents interact to suggest recommendation candidates. The framework begins with a target user's history, which is used to instantiate similar user agents and relevant item agents. A central orchestrator agent manages the entire process, including dynamic recruitment of agents and providing personalized collaboration instructions. Through multiple discussion rounds, these agents exchange information and generate recommendations. The orchestrator maintains control over the collaboration process, ensuring that discussions remain focused and productive while leveraging the collective knowledge of instantiated agents.

## Key Results
- MACF achieved best hit rates across all tested datasets, with 0.5238 on Amazon Clothing
- Competitive NDCG scores were obtained, demonstrating strong ranking performance
- Outperformed strong baseline models in both hit rate and NDCG metrics

## Why This Works (Mechanism)
The effectiveness of MACF stems from its ability to leverage collaborative filtering principles within an agentic framework. By instantiating agents that represent similar users and relevant items from the target user's history, MACF creates a collaborative environment where multiple perspectives can be considered simultaneously. The orchestrator agent ensures that these discussions remain focused and productive, while the multi-round discussion format allows for deeper exploration of recommendation candidates. This approach effectively combines the strengths of traditional collaborative filtering (leveraging user-item interactions) with the flexibility and reasoning capabilities of LLM-based agents.

## Architecture Onboarding

**Component Map**: User History -> Agent Instantiation -> Orchestrator -> Discussion Rounds -> Recommendations

**Critical Path**: Target User History → Agent Recruitment → Orchestrator Instructions → Multi-round Discussions → Candidate Generation → Final Recommendations

**Design Tradeoffs**: The framework trades computational overhead (multiple agents, discussion rounds) for potentially more nuanced recommendations that leverage collective reasoning. The dynamic recruitment approach balances thoroughness with efficiency, while the orchestrator ensures quality control but may introduce a single point of coordination complexity.

**Failure Signatures**: Performance degradation may occur if agent instantiation fails to capture relevant similar users/items, if the orchestrator provides poor collaboration instructions, or if discussion rounds become unproductive or diverge from recommendation goals. Baseline methods may outperform if agent interactions introduce noise rather than signal.
