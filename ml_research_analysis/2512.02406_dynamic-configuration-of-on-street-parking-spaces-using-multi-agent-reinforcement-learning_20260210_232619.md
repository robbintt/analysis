# Dynamic Configuration of On-Street Parking Spaces using Multi Agent Reinforcement Learning

## Quick Facts
- arXiv ID: 2512.02406
- Source URL: https://arxiv.org/abs/2512.02406
- Reference count: 38
- Multi-agent RL framework reduces travel time by up to 47% with minimal walking distance increase

## Executive Summary
This paper introduces a multi-agent reinforcement learning framework for dynamically configuring on-street parking spaces to minimize urban traffic congestion. The approach employs lane-level agents that make parking restriction decisions based on local traffic conditions, enhanced by LSTM and GAT networks to capture temporal and spatial correlations. Block-level agents coordinate these decisions to ensure sufficient parking availability while optimizing traffic flow. Experiments using real-world Melbourne data demonstrate significant improvements over static clearways and other RL baselines, with up to 47% reduction in average travel time loss and less than 2 meters increase in walking distance.

## Method Summary
The framework implements a hierarchical multi-agent reinforcement learning system with lane-level and block-level agents. Lane-level agents observe local traffic conditions and make parking restriction decisions, while block-level agents coordinate these decisions across their jurisdiction. LSTM networks capture temporal dependencies in traffic patterns, and GAT networks model spatial correlations between adjacent lanes. The system operates in episodes where agents receive rewards based on traffic flow optimization, with penalties for excessive walking distances. Training occurs through centralized learning with decentralized execution, allowing agents to learn from collective experience while making independent decisions during operation.

## Key Results
- Up to 47% reduction in average travel time loss compared to static clearways
- Less than 2 meters increase in average walking distance
- Outperforms static clearways and other RL baselines in traffic flow optimization
- Demonstrates emission reduction benefits through improved traffic patterns

## Why This Works (Mechanism)
The framework succeeds by capturing both temporal and spatial traffic dynamics through specialized neural network architectures. LSTM networks identify patterns in how traffic conditions evolve over time, allowing agents to anticipate future congestion. GAT networks model the influence between adjacent lanes, ensuring coordinated decision-making that accounts for spillover effects. The hierarchical structure enables fine-grained local decisions while maintaining global optimization through block-level coordination. This combination allows the system to adapt parking configurations in real-time based on changing traffic conditions, unlike static approaches that cannot respond to dynamic urban environments.

- Graph Attention Networks: Why needed? To model spatial relationships between adjacent parking lanes and account for traffic spillover effects. Quick check: Validate that spatial correlation capture improves decision quality over independent lane modeling.

## Architecture Onboarding

Component Map:
Centralized Training Coordinator -> Lane-level RL Agents -> Block-level Coordination Agents -> Parking Control Interface

Critical Path:
Training data collection -> Centralized model updates -> Decentralized execution -> Traffic monitoring -> Reward calculation -> Model refinement

Design Tradeoffs:
- Centralized vs decentralized learning: Centralized learning enables better coordination but requires more computational resources
- Temporal vs spatial emphasis: Balancing LSTM and GAT contributions affects model performance in different traffic scenarios
- Reward function design: Trade-off between traffic flow optimization and parking availability satisfaction

Failure Signatures:
- Oscillation in parking restrictions when traffic patterns are highly variable
- Suboptimal coordination between adjacent blocks leading to local optima
- Overfitting to training data causing poor generalization to new traffic patterns

First Experiments:
1. Single-lane simulation with varying traffic patterns to validate temporal modeling
2. Two-block coordination test to verify spatial correlation capture
3. Static vs dynamic configuration comparison using simplified traffic scenarios
