# FastMap: Fast Queries Initialization Based Vectorized HD Map Reconstruction Framework

## Quick Facts
- arXiv ID: 2503.05492
- Source URL: https://arxiv.org/abs/2503.05492
- Reference count: 31
- FastMap achieves 68.1 mAP on nuScenes with 3.2x faster decoder and 40.3% improved inference speed over baseline

## Executive Summary
FastMap introduces an efficient vectorized HD map reconstruction framework that addresses computational bottlenecks in multi-layer DETR decoder architectures. The method replaces random query initialization with heatmap-guided query generation and employs a single-layer, two-stage transformer architecture. By incorporating a geometry-constrained point-to-line loss, FastMap effectively mitigates issues with homogeneous feature points while maintaining high accuracy in map element detection.

## Method Summary
FastMap presents a novel approach to vectorized HD map reconstruction that combines efficient query initialization with geometric constraints. The framework utilizes a single-layer, two-stage transformer architecture where queries are initialized through heatmap-guided generation rather than random assignment. This approach is complemented by a geometry-constrained point-to-line loss function that addresses the challenge of homogeneous feature points in map reconstruction. The method demonstrates state-of-the-art performance on nuScenes and Argoverse2 datasets while achieving significant computational efficiency improvements.

## Key Results
- Achieves 68.1 mAP on nuScenes dataset with FastMap-large configuration
- Decoder operates 3.2x faster than baseline methods
- Improves inference speed by 40.3% over MapTRv2 while maintaining or improving accuracy

## Why This Works (Mechanism)
The method's efficiency stems from replacing computationally expensive multi-layer DETR decoders with a single-layer architecture that uses intelligently initialized queries. Heatmap-guided query generation provides more informed starting positions compared to random initialization, reducing the search space for the transformer decoder. The geometry-constrained point-to-line loss ensures that detected map elements maintain proper geometric relationships, addressing the fundamental challenge of representing continuous map features from discrete point clouds.

## Architecture Onboarding

**Component Map:** Raw Sensor Data -> Feature Extraction -> Heatmap Generation -> Query Initialization -> Single-layer Transformer -> Geometry-constrained Loss -> Vectorized Output

**Critical Path:** The core processing pipeline flows through heatmap-guided query generation into the single-layer transformer, with the geometry-constrained loss providing feedback for optimization.

**Design Tradeoffs:** Single-layer transformer sacrifices potential refinement depth for significant speed gains; heatmap guidance requires additional computation but reduces decoder search space.

**Failure Signatures:** Poor heatmap quality leads to suboptimal query initialization; homogeneous point clusters may still confuse the geometry constraints in dense urban environments.
