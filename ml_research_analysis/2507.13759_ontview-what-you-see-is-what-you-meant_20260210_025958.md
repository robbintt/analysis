# OntView: What you See is What you Meant

## Quick Facts
- arXiv ID: 2507.13759
- Source URL: https://arxiv.org/abs/2507.13759
- Reference count: 19
- Primary result: An open-source ontology viewer that visualizes inferred knowledge including GCIs and anonymous classes, enabling efficient navigation of large ontologies like DBpedia in under 10 seconds

## Executive Summary
OntView is an innovative ontology visualization tool designed to address the challenge of effectively visualizing complex ontologies. Unlike traditional ontology viewers that only display explicit knowledge, OntView implements a "What you see is what you meant" approach by using a Description Logic reasoner to present inferred knowledge, including General Concept Inclusions (GCIs) and anonymous classes. The tool offers interactive navigation capabilities, allowing users to explore and dynamically hide/show ontology branches, while providing summarization methods to assess concept importance through algorithms like PageRank and KeyConceptExtraction.

## Method Summary
OntView is implemented in Java using OWLAPI for ontology manipulation and JavaFX for the graphical interface. The tool employs a Description Logic reasoner to compute inferred knowledge, presenting a comprehensive view of both explicit and implicit ontology content. Users can interact with the ontology through incremental expansion with user-defined detail levels, enabling efficient navigation of large ontologies. The system includes two summarization algorithms - PageRank and KeyConceptExtraction - to help users identify the most important concepts within an ontology. OntView supports the loading of ontologies from files or URIs and provides multiple visualization layouts including radial and vertical tree arrangements.

## Key Results
- Successfully visualizes complex ontologies like DBpedia in under 10 seconds
- Implements inferred knowledge visualization including General Concept Inclusions and anonymous classes
- Provides interactive navigation with dynamic hiding/showing of ontology branches
- Includes summarization algorithms (PageRank and KeyConceptExtraction) for assessing concept importance

## Why This Works (Mechanism)
OntView works by leveraging a Description Logic reasoner to compute and visualize both explicit and inferred knowledge within ontologies. This approach ensures that users see not just what is explicitly stated in the ontology, but also what can be logically deduced, providing a more complete understanding of the domain. The incremental expansion feature allows users to explore ontologies at their own pace, revealing details progressively rather than overwhelming them with the entire structure at once. The summarization algorithms help users quickly identify the most important concepts by analyzing the ontology structure and relationships.

## Architecture Onboarding

Component Map: User Interface (JavaFX) -> OWLAPI -> Description Logic Reasoner -> Visualization Engine -> Display

Critical Path: User interaction -> Ontology loading -> Reasoning computation -> Visualization generation -> Display update

Design Tradeoffs: Prioritized comprehensive knowledge visualization over real-time performance, chose JavaFX for cross-platform compatibility over native performance, implemented multiple summarization algorithms to provide flexibility rather than optimizing a single approach.

Failure Signatures: Slow loading times indicate reasoning bottlenecks; incomplete visualizations suggest ontology parsing errors; incorrect hierarchy displays point to reasoner configuration issues.

## Limitations

- Lacks quantitative validation through user studies comparing comprehension with existing tools
- Performance claims for large ontologies lack detailed methodology and scalability testing
- Current limitation to class hierarchies excludes property assertions, which are important for many ontology applications
- Summarization algorithms effectiveness not compared against alternative ranking methods
