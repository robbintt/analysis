# On-line learning of dynamic systems: sparse regression meets Kalman filtering

## Quick Facts
- arXiv ID: 2511.11178
- Source URL: https://arxiv.org/abs/2511.11178
- Reference count: 40
- Authors: Gianluigi Pillonetto, Akram Yazdani, Aleksandr Aravkin
- Key outcome: Combines sparse regression (Sindy) with Kalman filtering to enable real-time identification of nonlinear dynamical systems, treating unknown parameters as states and recovering parsimonious models from streaming data

## Executive Summary
This paper introduces the Sindy Kalman Filter (SKF), a novel framework that bridges sparse regression techniques with Kalman filtering for real-time identification of nonlinear dynamical systems. The key innovation treats unknown system parameters as states in a dynamic system, allowing the filter to estimate parsimonious models from streaming data. SKF unifies both frameworks and extends sparsity-driven approaches to real-time learning, enabling three key capabilities: recovering time-invariant systems, identifying smoothly time-varying systems, and detecting abrupt changes in switching systems.

The method demonstrates strong performance on chaotic Lorenz systems with drifting or switching parameters and shows effectiveness in real-time identification of a sparse nonlinear aircraft model from actual flight data. The approach provides sparse, interpretable models with strong predictive capability while maintaining linear computational complexity with respect to dataset size.

## Method Summary
The Sindy Kalman Filter treats unknown system parameters as states in an augmented dynamic system, where the original system states and parameters evolve together. The filter uses prediction error minimization to automatically estimate sparsity levels, variance parameters, and switching instants without requiring validation data. For time-varying parameters, the filter models them as stochastic processes (e.g., random walk or Ornstein-Uhlenbeck), while for switching systems it uses discrete modes with unknown switching times. The sparse regression component identifies relevant basis functions from a library, while the Kalman filtering component handles the estimation of both states and parameters in real-time.

## Key Results
- Recovers time-invariant systems matching classical Sindy performance while operating in real-time
- Identifies smoothly time-varying systems with parameters evolving as stochastic processes
- Detects abrupt changes in switching systems and estimates switching instants automatically

## Why This Works (Mechanism)
The method works by unifying sparse regression (which identifies parsimonious model structures) with Kalman filtering (which handles noisy measurements and parameter uncertainty). By treating parameters as states, the filter can propagate uncertainty and update estimates recursively as new data arrives. The prediction error minimization provides a unified criterion for joint estimation of model structure, noise parameters, and switching events, eliminating the need for separate validation procedures.

## Foundational Learning
- Kalman filtering: Recursive state estimation under uncertainty
  - Why needed: Handles noisy measurements and provides uncertainty quantification
  - Quick check: Verify state estimation accuracy on linear systems with known noise characteristics
- Sparse regression (Sindy): Identifying parsimonious model structures from basis function libraries
  - Why needed: Prevents overfitting and produces interpretable models
  - Quick check: Test on systems with known sparse representations
- Stochastic process modeling: Representing time-varying parameters
  - Why needed: Captures smooth parameter evolution without assuming constancy
  - Quick check: Validate parameter tracking on systems with known parameter drift rates

## Architecture Onboarding
Component map: Measurements -> Kalman Filter -> Prediction Error -> Sparse Regression -> Parameter Estimates -> Model Output

Critical path: The core loop involves prediction using current parameter estimates, measurement update incorporating new data, sparse regression to refine model structure, and parameter update. The prediction error drives both the Kalman gain calculations and the sparse regression step.

Design tradeoffs: Real-time operation vs. estimation accuracy, model sparsity vs. predictive performance, computational complexity vs. system dimensionality. The method prioritizes interpretability and real-time capability over maximum accuracy.

Failure signatures: Poor performance indicates either insufficient excitation in the data, inappropriate basis function library, or violation of noise assumptions. Rapid parameter changes may cause tracking lag if the stochastic process model is too restrictive.

## Limitations
- Assumption of locally constant parameters within prediction steps may fail for rapidly varying systems
- Noise variance estimation relies on prediction error minimization, which could be sensitive to initial parameter guesses
- Assumes Gaussian process noise and measurement noise, limiting applicability to systems with non-Gaussian disturbances

## Confidence
