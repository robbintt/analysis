# Large Language Models for Automating Clinical Data Standardization: HL7 FHIR Use Case

## Quick Facts
- **arXiv ID**: 2507.03067
- **Source URL**: https://arxiv.org/abs/2507.03067
- **Reference count**: 40
- **Primary result**: LLM-driven FHIR transformation achieved 100% resource-level identification and up to 73.88% attribute-level mapping accuracy

## Executive Summary
This study introduces a semi-automated approach using large language models (LLMs) to transform structured clinical data into HL7 FHIR format. The method combines embedding techniques, clustering algorithms, and semantic retrieval to map tabular fields to corresponding FHIR resources. Tested on the MIMIC-IV dataset with GPT-4o and Llama 3.2 405b, the approach demonstrated high resource-level accuracy while maintaining robust attribute-level mapping in real-world conditions.

## Method Summary
The methodology employs a multi-stage pipeline integrating embedding techniques, clustering algorithms, and semantic retrieval to transform structured clinical data into HL7 FHIR format. The approach uses LLMs to interpret field semantics and generate corresponding FHIR resources, with evaluation conducted on the MIMIC-IV dataset using both GPT-4o and Llama 3.2 405b models. The system achieves resource-level mapping through semantic analysis while attribute-level mapping requires additional validation steps.

## Key Results
- Achieved 100% resource-level identification accuracy for FHIR mappings
- Attained up to 73.88% attribute-level mapping accuracy under ideal conditions
- Maintained 67.74%-69.86% accuracy in real-world simulations (95% CI for GPT-4o)
- GPT-4o outperformed Llama 3.2 405b in all evaluation metrics

## Why This Works (Mechanism)
The approach leverages LLMs' contextual understanding capabilities to interpret clinical data semantics and map them to standardized FHIR structures. By combining semantic retrieval with clustering algorithms, the system can identify patterns and relationships in clinical data that align with FHIR resource definitions. The semi-automated nature allows human oversight while reducing manual effort in data standardization.

- **Semantic Retrieval**: Why needed - Find relevant FHIR resources based on field semantics; Quick check - Measure retrieval precision and recall
- **Large Language Models**: Why needed - Interpret complex clinical data semantics and generate FHIR mappings; Quick check - Evaluate model performance on clinical domain-specific tasks
- **MIMIC-IV Dataset**: Why needed - Provides realistic clinical data for validation; Quick check - Verify dataset completeness and relevance to target use cases

## Architecture Onboarding
- **Component Map**: Clinical Data -> Embedding Layer -> Clustering -> Semantic Retrieval -> LLM Mapping -> FHIR Output
- **Critical Path**: Data preprocessing → Embedding generation → Cluster formation → Semantic matching → LLM resource generation → Attribute mapping → Validation
- **Design Tradeoffs**: Accuracy vs. processing time (complex embeddings improve accuracy but increase latency); Model choice impacts performance (GPT-4o vs. Llama 3.2); Semi-automation balances efficiency with quality control
- **Failure Signatures**: Hallucinations in LLM output; Missing metadata causing mapping errors; Cluster misclassification leading to wrong resource assignment; Semantic retrieval failures due to vocabulary mismatches
- **First Experiments**: 1) Test embedding quality on sample clinical fields vs. FHIR resources; 2) Validate cluster formation with known clinical data groupings; 3) Evaluate semantic retrieval precision on small test set

## Limitations
- Results based on single dataset (MIMIC-IV) limiting generalizability across diverse clinical sources
- Binary resource-level accuracy metric may not capture partial or context-dependent mappings
- Acknowledged but unquantified impact of hallucinations and missing metadata on clinical safety
- Room for improvement in attribute-level accuracy (73.88% ideal, 67.74%-69.86% real-world)

## Confidence
- **High confidence**: LLM feasibility for FHIR transformation is well-supported by experimental results
- **Medium confidence**: Comparative performance between GPT-4o and Llama 3.2 405b is valid for tested task and dataset
- **Medium confidence**: Identified challenges (hallucinations, missing metadata) likely persist across clinical datasets

## Next Checks
1. Replicate methodology across diverse clinical datasets beyond MIMIC-IV to assess generalizability
2. Implement formal hallucination detection mechanism to quantify error rates in real-world deployment
3. Conduct expert clinician review of LLM-generated FHIR mappings to assess clinical validity and safety implications
