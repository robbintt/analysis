# PyFCG: Fluid Construction Grammar in Python

## Quick Facts
- arXiv ID: 2505.12920
- Source URL: https://arxiv.org/abs/2505.12920
- Reference count: 13
- Primary result: Python package wrapping Fluid Construction Grammar for seamless NLP ecosystem integration

## Executive Summary
PyFCG introduces a pip-installable Python wrapper for the Fluid Construction Grammar (FCG) system, which was originally implemented in Common Lisp. This integration bridges the gap between FCG's formal linguistic capabilities and Python's extensive NLP ecosystem, enabling researchers to leverage Python tools while working with construction grammar formalisms. The library provides a native Pythonic interface for agents, grammars, and constructions, making construction grammar research more accessible to Python developers.

The paper demonstrates three key applications: grammar formalisation and testing, learning from semantically annotated corpora, and agent-based emergent communication experiments. By maintaining compatibility with the original FCG codebase while improving accessibility and usability, PyFCG addresses the practical limitations that have historically restricted FCG's adoption among computational linguistics researchers who primarily work in Python environments.

## Method Summary
The paper presents PyFCG as a Python wrapper around the Common Lisp reference implementation of Fluid Construction Grammar. The approach involves creating a pip-installable package that exposes FCG's core functionality through a Pythonic API, allowing users to interact with agents, grammars, and constructions using familiar Python constructs. The implementation maintains bidirectional compatibility with the original Common Lisp codebase while providing seamless integration with Python's NLP ecosystem.

## Key Results
- Provides pip-installable Python package wrapping Fluid Construction Grammar implementation
- Enables seamless integration with Python's NLP ecosystem through native Pythonic interface
- Demonstrates three practical use cases: grammar formalisation, corpus learning, and agent communication experiments

## Why This Works (Mechanism)
PyFCG works by creating a Python interface layer that communicates with the underlying Common Lisp FCG implementation. This wrapper approach preserves the formal linguistic capabilities of FCG while exposing them through Python's object-oriented paradigms and extensive library ecosystem. The design leverages Python's mature packaging infrastructure and NLP tool support to make construction grammar research more accessible and practical for modern computational linguistics workflows.

## Architecture Onboarding

**Component map**: Python wrapper -> Common Lisp FCG core -> Construction grammar formalism

**Critical path**: User Python code → PyFCG API → CFFI bridge → Common Lisp FCG → Processing results → Python response

**Design tradeoffs**: Performance vs accessibility - the Python wrapper introduces overhead but dramatically improves usability and ecosystem integration compared to direct Common Lisp usage.

**Failure signatures**: 
- Import errors when Common Lisp dependencies are missing
- Performance bottlenecks due to inter-language communication overhead
- Compatibility issues when FCG API changes aren't reflected in Python wrapper
