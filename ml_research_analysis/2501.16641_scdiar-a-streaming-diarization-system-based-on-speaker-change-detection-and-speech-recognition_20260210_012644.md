# SCDiar: a streaming diarization system based on speaker change detection and speech recognition

## Quick Facts
- arXiv ID: 2501.16641
- Source URL: https://arxiv.org/abs/2501.16641
- Authors: Naijun Zheng; Xucheng Wan; Kai Liu; Zhou Huan
- Reference count: 29
- Primary result: Outperforms previous streaming systems by up to 53.6% accuracy on real-world meeting data with 10+ speakers

## Executive Summary
SCDiar addresses the challenge of real-time speaker diarization in long meetings with many participants. The system combines speaker change detection with token-level segmentation and an optimization process to select the best available segment for each speaker. It significantly outperforms existing streaming diarization methods while maintaining real-time processing capabilities, achieving cpWER of 10.66 and WDER of 15.36 on in-house datasets with 10+ speakers.

## Method Summary
SCDiar processes speech by first detecting speaker changes to create token-level segments. It then employs a novel optimization algorithm to select the most appropriate segment for each speaker, balancing the trade-offs between speaker identification accuracy and diarization completeness. The system operates in a streaming fashion, processing audio incrementally without requiring full recording access, making it suitable for real-time applications.

## Key Results
- Achieves 53.6% accuracy improvement over previous streaming diarization systems
- cpWER of 10.66 and WDER of 15.36 on in-house dataset with 10+ speakers
- Significantly narrows the performance gap between online and offline diarization systems
- Maintains real-time processing capabilities on extended meeting recordings

## Why This Works (Mechanism)
The system's effectiveness stems from its dual approach: precise speaker change detection creates meaningful segmentation boundaries, while the optimization process intelligently selects segments that maximize speaker identification accuracy. By operating at the token level rather than utterance or fixed window boundaries, SCDiar captures fine-grained speaker transitions that traditional methods miss, particularly important in complex multi-speaker scenarios.

## Architecture Onboarding

**Component Map**: Audio Stream -> Speaker Change Detection -> Token Segmentation -> Optimization Engine -> Diarization Output

**Critical Path**: The optimization engine is the computational bottleneck, as it must process token segments in real-time while maintaining streaming constraints.

**Design Tradeoffs**: Token-level precision versus computational complexity; real-time processing versus batch optimization accuracy; streaming constraints versus offline performance potential.

**Failure Signatures**: Poor speaker change detection leads to segment boundary errors; optimization failures result in speaker identity confusion; speech recognition errors propagate into diarization mistakes.
