# Temporal Knowledge Graph Hyperedge Forecasting: Exploring Entity-to-Category Link Prediction

## Quick Facts
- arXiv ID: 2510.24240
- Source URL: https://arxiv.org/abs/2510.24240
- Reference count: 9
- Primary result: TLogic with Max+ aggregation generally outperforms category-specific C-TLogic on entity prediction, though category prediction remains feasible with high accuracy.

## Executive Summary
This work extends the TLogic framework for temporal knowledge graph (TKG) link prediction by incorporating entity categories and exploring category-specific predictions. The proposed C-TLogic format integrates entity types into rule mining and grounding, limiting rule application to relevant entities. A data-driven LLM-based method generates categories for datasets lacking them, and an alternative aggregation method (Max+) is introduced alongside the original Noisy-OR. Experiments on ICEWS18 and FinDKG datasets show that TLogic with Max+ aggregation generally outperforms C-TLogic, suggesting that general patterns across categories are more valuable than category-specific ones. However, category prediction remains feasible with high accuracy, especially when using TLogic on granular category structures like GICS sectors in FinDKG.

## Method Summary
The study extends TLogic, a rule-based TKG link prediction framework, to incorporate entity categories (C-TLogic) by extending the rule format to include subject/object types. For datasets without pre-defined categories, an LLM-based method generates categories using text embeddings and Gaussian Mixture Model clustering. The framework evaluates two aggregation methods: Noisy-OR and Max+. Experiments compare TLogic and C-TLogic on ICEWS18 (with synthetic categories) and FinDKG (with pre-existing categories), measuring performance using MRR and Hits@k metrics while monitoring "No Candidates" (N.C.) ratios.

## Key Results
- TLogic with Max+ aggregation generally outperforms C-TLogic on entity prediction tasks across both datasets
- Category prediction is feasible with high accuracy, particularly using TLogic on granular category structures
- C-TLogic struggles with data sparsity, showing high N.C. ratios when using more granular categories (100 vs 12 clusters)
- Max+ aggregation benefits entity prediction by ignoring low-confidence rules, while Noisy-OR better accumulates weak signals for category prediction

## Why This Works (Mechanism)

### Mechanism 1: Temporal Random Walk Rule Mining
The learner samples random walks starting from a head fact and moving backward in time, identifying chains where the object of one step becomes the subject of the next. This transforms unstructured temporal sequences into symbolic rules with placeholders, assuming event causality follows repeatable, cyclic chains.

### Mechanism 2: Type-Constrained Rule Grounding
C-TLogic extends the standard quadruple to a sextuple $(s, r, o, t, c_s, c_o)$, filtering rules not just by relation but by matching the subject category $c_s$ of the query to the rule head. This theoretically restricts predictions to relevant entity types, reducing semantic drift.

### Mechanism 3: Category-Level Score Aggregation
The system maps predicted entities to their categories and aggregates scores using either Noisy-OR (probabilistic union) or Max+ (lexicographical sorting). This reveals "meta-trends" invisible in top-k entity rankings by aggregating signal across multiple lower-ranked entity predictions.

## Architecture Onboarding

- **Component map:** Learner (samples temporal walks -> instantiates Rules -> estimates Confidence -> Rule Bank) -> Retriever (accepts Query -> filters Rule Bank by relation & category -> grounds Rules in history -> Candidates) -> Evaluator (aggregates Candidate Scores -> ranks Results)

- **Critical path:** The Rule Grounding in the Retriever. Even if a rule is learned, it is useless if it cannot be matched to the historical time window of a specific query.

- **Design tradeoffs:**
  - Granularity vs. Generalizability: Increasing categories allows specific rules but reduces applicable rules, increasing N.C. rate
  - TLogic vs. C-TLogic: TLogic finds general patterns (higher accuracy); C-TLogic attempts specific patterns (higher potential explainability, but currently lower accuracy)

- **Failure signatures:**
  - High "N.C." Ratio: Indicates rule constraints are too strict for data density
  - Score Saturation: In Noisy-OR, if scores round to 1.0, ranking degenerates to a count of predictions

- **First 3 experiments:**
  1. Baseline Validation: Replicate TLogic vs. C-TLogic comparison on ICEWS18 subset
  2. Aggregation Sensitivity: Compare Noisy-OR vs. Max+ on FinDKG for category prediction
  3. Clustering Robustness: Vary GMM clusters (12 vs. 50) for entity categorization and measure N.C. rate

## Open Questions the Paper Calls Out

### Open Question 1
- Question: Can incorporating time distance between rule grounding and predicted event into confidence estimation improve ranking accuracy?
- Basis: The Conclusion states current confidence estimation ignores this temporal gap
- Why unresolved: Current scoring uses temporal decay for windowing but not for specific gap duration
- Evidence needed: Ablation study comparing current vs. time-aware confidence metrics

### Open Question 2
- Question: Can dynamic rule learning methods reduce high N.C. ratios in C-TLogic?
- Basis: The Discussion notes static random walks miss valuable patterns in granular data
- Why unresolved: C-TLogic struggles with data sparsity when categorized
- Evidence needed: Adaptive rule mining algorithm prioritizing under-sampled relation-category pairs

### Open Question 3
- Question: What dataset characteristics determine whether Noisy-OR or Max+ aggregation provides superior performance for category prediction?
- Basis: The Conclusion identifies aggregation method choice as future research area
- Why unresolved: TLogic favors Max+ while C-TLogic favors Noisy-OR, but underlying cause unclear
- Evidence needed: Theoretical analysis or empirical correlation linking graph density to aggregation performance

## Limitations
- Rule length $b_{max}$ is unspecified, affecting rule mining completeness and computational complexity
- GMM clustering parameters for synthetic category generation are unspecified, potentially affecting category stability
- Study focuses on two datasets with limited comparison to embedding-based TKG methods

## Confidence
- **High confidence:** TLogic with Max+ outperforms C-TLogic on entity prediction; TLogic achieves strong category prediction on FinDKG
- **Medium confidence:** Category prediction is feasible and benefits from aggregation methods; entity-to-category mapping reveals distributed signals
- **Low confidence:** Strict type-constrained rules consistently underperform general rules across all scenarios
