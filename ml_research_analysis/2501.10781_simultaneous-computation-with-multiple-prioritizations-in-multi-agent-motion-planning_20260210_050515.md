# Simultaneous Computation with Multiple Prioritizations in Multi-Agent Motion Planning

## Quick Facts
- arXiv ID: 2501.10781
- Source URL: https://arxiv.org/abs/2501.10781
- Authors: Patrick Scheffe; Julius Kahle; Bassam Alrifaee
- Reference count: 15
- Primary result: Approach achieves near-optimal prioritization with minor computation increase, demonstrating real-time capability for ten vehicles in controlled experiments

## Executive Summary
This paper introduces a novel approach for multi-agent motion planning (MAMP) that enables agents to compute with multiple prioritizations simultaneously, improving solution quality while maintaining low computation time. The method leverages the sequential nature of prioritized planning (PP) to explore multiple priority orderings without exponential computational overhead. Demonstrated in the context of connected and automated vehicles (CAVs) on road networks using receding horizon planning, the approach addresses the fundamental challenge of balancing computational tractability with solution optimality in large-scale multi-agent systems.

## Method Summary
The core innovation involves allowing multiple agents to simultaneously explore different priority orderings during the planning process, rather than committing to a single prioritization sequence. This is achieved by maintaining parallel computation threads where agents can dynamically adjust their relative priorities based on real-time information and local conflicts. The method integrates seamlessly with receding horizon planning frameworks, enabling continuous replanning within strict computation time constraints. By exploring multiple priority configurations in parallel, the approach can identify high-quality solutions that would be missed by traditional sequential prioritized planning, while avoiding the computational explosion associated with exhaustive search methods.

## Key Results
- Simultaneous computation with multiple prioritizations achieves solution quality close to optimal prioritization
- Computational overhead increases only marginally compared to single-prioritization approaches
- Real-time capability demonstrated with ten vehicles in Cyber-Physical Mobility Lab experiments
- Outperforms state-of-the-art methods in solution quality metrics while maintaining computational efficiency

## Why This Works (Mechanism)
The approach works by exploiting the sequential dependency structure inherent in prioritized planning. When agents compute with multiple priority orderings simultaneously, they can share information about conflicts and resolutions across different priority configurations. This parallel exploration allows the system to identify priority sequences that minimize conflicts early in the planning process, leading to better overall solutions. The receding horizon framework provides natural boundaries for computation, ensuring that the simultaneous exploration remains tractable by limiting the depth and breadth of parallel searches.

## Foundational Learning

**Prioritized Planning (PP)** - Planning where agents are assigned priority levels and compute paths sequentially
- Why needed: Provides computational tractability for multi-agent systems
- Quick check: Can solve MAPF problems with polynomial complexity instead of exponential

**Receding Horizon Planning** - Iterative planning approach that recomputes paths at regular intervals
- Why needed: Enables real-time adaptation to changing conditions
- Quick check: Balances solution quality against computation time constraints

**Multi-Agent Path Finding (MAPF)** - Finding collision-free paths for multiple agents simultaneously
- Why needed: Core problem in coordinated multi-agent systems
- Quick check: NP-hard problem requiring approximation or heuristic approaches

## Architecture Onboarding

**Component Map:** Environment State -> Priority Assignment Module -> Parallel Planning Threads -> Conflict Resolution -> Final Path Selection

**Critical Path:** Sensor inputs → State estimation → Priority computation → Parallel planning → Conflict detection → Path output

**Design Tradeoffs:** 
- Computation time vs. solution quality (parallel exploration increases quality but adds overhead)
- Number of parallel priorities vs. real-time capability (more priorities improve quality but risk missing deadlines)
- Communication frequency vs. coordination effectiveness (more frequent updates improve coordination but increase overhead)

**Failure Signatures:** 
- Priority deadlock when agents cyclically conflict across different priority orderings
- Computational overload when too many parallel threads exceed real-time constraints
- Suboptimal convergence when priority exploration terminates prematurely

**First 3 Experiments:**
1. Validate parallel priority computation on simple grid-world scenarios with known optimal solutions
2. Test scalability by incrementally increasing agent count and measuring computation time/quality tradeoff
3. Evaluate robustness to communication delays by introducing artificial latency in agent coordination

## Limitations
- Evaluation limited to small-scale scenarios with ten vehicles in controlled lab environments
- Scalability to larger urban networks with hundreds of CAVs not demonstrated
- Performance under varying traffic densities and network topologies remains untested
- Robustness to communication delays and packet losses not evaluated
