# Duality-based Mode Operations and Pyramid Multilayer Mapping for Rhetorical Modes

## Quick Facts
- arXiv ID: 2511.06601
- Source URL: https://arxiv.org/abs/2511.06601
- Reference count: 3
- Introduces duality-based operations and pyramid multilayer mapping to systematically extend rhetorical modes

## Executive Summary
This paper proposes duality-based mode operations and a pyramid multilayer mapping framework to systematically expand and manage rhetorical modes. The framework introduces four duality operations—split-unite, forward-backward, expansion-reduction, and orthogonal—that generate new rhetorical modes from a base set of 14 modes. A pyramid mapping across rhetorical→cognitive→epistemic layers reduces selection uncertainty compared to flat mode selection, enabling tractable discourse design. The approach provides a measurable, scalable framework linking linguistic form to cognitive function and epistemic purpose.

## Method Summary
The method employs four duality operations (split-unite, forward-backward, expansion-reduction, orthogonal) to generate new rhetorical modes from 14 base modes. Combinatorial analysis quantifies expressive capacity using binomial coefficients and Shannon entropy. The pyramid multilayer mapping reduces cognitive entropy through hierarchical selection across rhetorical→cognitive→epistemic layers. The framework defines a Marginal Rhetorical Bit (MRB) parameter where each mode contributes approximately 1 bit of expressive capacity.

## Key Results
- Duality operations can generate new rhetorical modes like generalization and combination from base modes
- Hierarchical mapping reduces cognitive entropy compared to flat selection across all modes
- Each additional rhetorical mode contributes ~1 bit of expressive capacity (MRB=1)
- Pyramid multilayer mapping enables tractable discourse design through layered constraints

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Four duality operations can systematically generate new rhetorical modes from canonical modes
- Mechanism: Operations like splitting compound modes or reversing direction create new expressive possibilities by exploiting formal properties of modes
- Core assumption: Rhetorical modes have formal properties amenable to algebraic-style operations
- Evidence anchors: Abstract and section 2.1.3; weak direct validation from corpus
- Break condition: Generated modes may be "non-linguistic or cognitively unstable"

### Mechanism 2
- Claim: Hierarchical mapping across layers reduces selection uncertainty compared to flat selection
- Mechanism: Breaking K=20 modes into smaller subsets at each layer reduces entropy from H_R(20)≈3.21 bits to combined ~3.62 bits across two layers
- Core assumption: Layered mapping constrains rhetorical variability in a way that aligns with natural cognitive processing
- Evidence anchors: Abstract, section 3.1 with numerical entropy values; weak corpus validation
- Break condition: Real entropy reduction depends on how layers are organized and may vary across disciplines and languages

### Mechanism 3
- Claim: Each additional rhetorical mode contributes ~1 bit of expressive capacity (MRB=1)
- Mechanism: Total non-empty combinations K_NRC = 2^K − 1; log₂(2^K − 1) ≈ K, so d(K_RC)/dK = 1 bit per mode
- Core assumption: Treating modes as combinatorial objects validly captures expressive capacity
- Evidence anchors: Abstract, section 2.2.1 with formal derivation; no corpus validation
- Break condition: Model assumes equiprobable mode combinations without accounting for semantic coherence constraints

- Component map: R-layer (rhetorical modes) -> C-layer (cognitive functions) -> E-layer (epistemic purposes) -> Operations module (four duality operators) -> Quantification module (MRB calculator, Rscale parameter)

- Critical path: 1) Start with base modes (Table 1), 2) Apply duality operations to generate extended modes, 3) Map R→C→E via Tables 5–6, 4) Calculate combinatorial capacity and entropy, 5) Apply to discourse design (e.g., lesson sequencing)

## Open Questions the Paper Calls Out

- Question: How can a formal operator calculus be developed to abstract logical relations among rhetorical modes for computational implementation?
  - Basis in paper: Section 2.1.6 states it "would be useful to develop... operator calculus" to programmatically search for modes and aid Large Language Models
  - Why unresolved: The paper defines operations schematically but lacks the symbolic algebra necessary for automation
  - What evidence would resolve it: A symbolic system defining operator behavior and a software prototype demonstrating automatic mode generation

- Question: What formal criteria can distinguish a linguistically valid generated rhetorical mode from a cognitively unstable one?
  - Basis in paper: The conclusion acknowledges "there lacks a formal way to evaluate whether a generated rhetorical mode can be used as a rhetorical mode"
  - Why unresolved: Current operations generate hybrids without a mechanism to validate their functional stability in discourse
  - What evidence would resolve it: A set of validity constraints tested against generated modes to filter non-linguistic or unstable operations

- Question: To what extent do the proposed three-layer mappings vary across different academic disciplines and natural languages?
  - Basis in paper: The conclusion notes the mappings "could vary across disciplines and languages, requiring further validation"
  - Why unresolved: The framework is presented as a canonical path, but its universality across distinct rhetorical traditions is unproven
  - What evidence would resolve it: Comparative studies applying the mapping framework to diverse corpora to test structural invariance

- Question: What is the empirical value of the processing capacity C_0 (bits/stage), and does R*_scale > 1 accurately predict cognitive overload?
  - Basis in paper: Section 2.2.4 defines normalized load R*_scale and hypothesizes overload conditions based on constant C_0, but provides no empirical measurement
  - Why unresolved: The model quantifies load theoretically, but the actual human cognitive limit remains unquantified
  - What evidence would resolve it: Experimental data measuring learner performance against varying rhetorical introduction rates to identify failure thresholds

## Limitations
- Generated modes may be "non-linguistic or cognitively unstable" with no formal validation criteria
- Hierarchical entropy reduction depends heavily on layer organization and varies across disciplines and languages
- Combinatorial capacity model assumes equiprobable combinations without semantic coherence constraints

## Confidence
- **Duality Operations Generate Valid Rhetorical Modes** (Low): Formal operations defined, but no validation of generated modes as legitimate rhetorical constructs
- **Hierarchical Mapping Reduces Cognitive Entropy** (Medium): Mathematical proof provided, but dependent on layer organization and selection probabilities
- **MRB=1 Bit per Mode** (Medium): Combinatorial derivation sound, but assumes equiprobable combinations without coherence constraints
- **Pyramid Multilayer Framework** (High): Clear architectural framework with defined layers and mappings, though implementation details vary
