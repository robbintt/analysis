# One-Class Domain Adaptation via Meta-Learning

## Quick Facts
- arXiv ID: 2501.13052
- Source URL: https://arxiv.org/abs/2501.13052
- Reference count: 30
- Primary result: Introduces OC-DA problem and adapts MAML algorithm, showing improved performance on Rainbow-MNIST and centrifugal pump datasets

## Executive Summary
This paper introduces One-Class Domain Adaptation (OC-DA), a novel problem setting addressing the challenge of adapting machine learning models in industrial IoT systems when only normal operational data is available for adaptation in new environments. The authors propose a task sampling strategy to adapt any bi-level meta-learning algorithm to OC-DA, focusing on modifying the MAML algorithm to create OC-DA MAML. Theoretical analysis shows that OC-DA MAML optimizes for meta-parameters enabling rapid one-class adaptation across domains.

## Method Summary
The authors propose a task sampling strategy to adapt bi-level meta-learning algorithms to the OC-DA problem setting. The core innovation involves modifying the MAML algorithm to create OC-DA MAML, which uses a specialized task sampling approach that only uses normal operational data during the meta-training phase. This adaptation allows the meta-learner to focus on adaptation capabilities that are relevant for one-class scenarios. The theoretical analysis demonstrates that the proposed method optimizes meta-parameters that enable rapid one-class adaptation across domains.

## Key Results
- OC-DA MAML significantly improves performance on target domains compared to standard approaches
- The method outperforms standard MAML using classical task sampling strategy
- Demonstrated effectiveness on both synthetic Rainbow-MNIST and real-world centrifugal pump vibration datasets

## Why This Works (Mechanism)
The method works by adapting meta-learning algorithms to focus on one-class adaptation scenarios through specialized task sampling. By only using normal operational data during meta-training, the algorithm learns meta-parameters that are specifically tuned for detecting deviations from normal behavior rather than traditional multi-class classification. This approach leverages the few-shot learning capabilities of meta-learning while addressing the practical constraint of limited abnormal data in industrial settings.

## Architecture Onboarding

Component map: Data sampling -> Meta-training loop -> Inner adaptation -> Meta-parameter update -> Evaluation

Critical path: The critical path involves the meta-training loop where task sampling generates one-class tasks, followed by inner adaptation using gradient descent, and outer loop meta-parameter updates based on performance across tasks.

Design tradeoffs: The primary tradeoff involves balancing the specificity of one-class adaptation against the generalization capabilities of the meta-learner. Using only normal data for adaptation improves relevance but may limit the ability to distinguish subtle variations in abnormal behavior.

Failure signatures: Potential failures include overfitting to specific normal patterns, inability to generalize to unseen types of domain shifts, and sensitivity to the choice of task sampling parameters. The method may also struggle with highly dynamic environments where normal behavior itself evolves significantly over time.
