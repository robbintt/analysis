# Chat2SPaT: A Large Language Model Based Tool for Automating Traffic Signal Control Plan Management

## Quick Facts
- arXiv ID: 2507.05283
- Source URL: https://arxiv.org/abs/2507.05283
- Authors: Yue Wang; Miao Zhou; Guijing Huang; Rui Zhuo; Chao Yi; Zhenliang Ma
- Reference count: 0
- Primary result: Chat2SPaT achieves over 94% accuracy in generating traffic signal plans from user descriptions.

## Executive Summary
Chat2SPaT is an innovative tool that leverages Large Language Models (LLMs) to automate the generation of traffic signal control plans from semi-structured, natural language descriptions. By parsing user input and extracting key phase sequences and attributes, the system converts ambiguous descriptions into structured Signal Phase and Timing (SPaT) results. The tool employs Python scripts and domain knowledge to handle common phasing schemes, pedestrian crossings, and protected turns, significantly reducing the manual workload for traffic engineers. With an accuracy rate exceeding 94% on a diverse dataset of over 300 English and Chinese plan descriptions, Chat2SPaT represents a robust application of LLMs in intelligent transportation systems.

## Method Summary
Chat2SPaT uses a two-step approach to convert user-provided traffic signal descriptions into structured SPaT plans. First, it employs carefully curated prompts to extract phase sequences and attributes from semi-structured, potentially ambiguous input. Second, it utilizes Python scripts to generate complete traffic signal plans based on the extracted information. The system integrates domain knowledge to handle various phasing schemes and special treatments, such as pedestrian crossings and protected turns. By combining natural language processing with traffic engineering expertise, Chat2SPaT streamlines the process of creating traffic signal control plans, reducing the potential for human error and increasing efficiency in traffic management.

## Key Results
- Chat2SPaT achieves over 94% accuracy in generating correct signal plans from user descriptions.
- The tool successfully handles both English and Chinese traffic plan descriptions.
- Chat2SPaT reduces manual workload for traffic engineers by automating the conversion of natural language inputs into structured SPaT results.

## Why This Works (Mechanism)
Chat2SPaT's effectiveness stems from its ability to bridge the gap between human-readable, semi-structured descriptions and machine-readable structured SPaT data. By leveraging the natural language processing capabilities of LLMs, the tool can interpret ambiguous or incomplete user inputs and extract relevant traffic control information. The integration of domain knowledge ensures that the generated plans adhere to traffic engineering principles and can handle complex scenarios involving pedestrian crossings and protected turns. The use of Python scripts for final plan generation provides a flexible and programmable approach to creating standardized signal control outputs.

## Architecture Onboarding

Component Map:
User Description -> LLM with Curated Prompts -> Extracted Phase Information -> Python Scripts -> Structured SPaT Plan

Critical Path:
1. User input reception and preprocessing
2. LLM prompt generation and execution
3. Phase information extraction and validation
4. Python script execution for plan generation
5. Output formatting and delivery

Design Tradeoffs:
- Accuracy vs. processing speed: More complex prompts may improve accuracy but increase processing time
- Domain knowledge integration vs. system flexibility: Extensive domain rules enhance plan quality but may limit adaptability to novel scenarios
- Multilingual support vs. resource allocation: Supporting multiple languages increases complexity and development effort

Failure Signatures:
- Misinterpretation of ambiguous descriptions leading to incorrect phase sequences
- Generation of non-standard or infeasible signal plans due to incomplete domain knowledge
- Language-specific errors in Chinese descriptions affecting plan accuracy
