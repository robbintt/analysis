# Neural Video Compression using 2D Gaussian Splatting

## Quick Facts
- **arXiv ID**: 2505.09324
- **Source URL**: https://arxiv.org/abs/2505.09324
- **Reference count**: 20
- **Primary result**: 88% faster encoding than previous Gaussian splatting-based image codecs

## Executive Summary
This work introduces a novel neural video compression approach using 2D Gaussian Splatting, addressing the computational inefficiency that limits neural codecs in real-time applications like video conferencing. The method leverages region-of-interest (ROI) based processing with content-aware initialization and a novel Gaussian inter-frame redundancy-reduction mechanism. This enables the first video-codec solution in the neural video codec space, achieving real-time decoding capabilities while maintaining competitive quality and significant bitrate savings.

## Method Summary
The proposed method introduces a region-of-interest (ROI) based neural video compression model using 2D Gaussian Splatting. The approach employs content-aware initialization strategy and a novel Gaussian inter-frame redundancy-reduction mechanism to achieve efficient video compression. The codec demonstrates significant improvements in encoding time (reduced from 13 seconds to 1.5 seconds per frame for I-frames and to 1 second for P-frames) while maintaining competitive quality, making it suitable for real-time applications like video conferencing.

## Key Results
- Achieves 88% faster encoding than previous Gaussian splatting-based image codecs
- Enables real-time decoding capabilities for video conferencing applications
- Reduces encoding time from 13 seconds to 1.5 seconds per frame for I-frames and to 1 second for P-frames

## Why This Works (Mechanism)
The method works by leveraging region-of-interest detection to focus computational resources on areas of the frame that contain meaningful content. The content-aware initialization strategy optimizes the starting parameters for the Gaussian splatting process, while the inter-frame redundancy-reduction mechanism efficiently encodes differences between consecutive frames. This targeted approach reduces unnecessary computation on static or less important regions, enabling faster encoding while maintaining quality.

## Foundational Learning
- **2D Gaussian Splatting**: A technique for rendering 3D scenes using 2D projections of Gaussian distributions, adapted here for video compression
  - *Why needed*: Provides a differentiable rendering approach that can be optimized for compression tasks
  - *Quick check*: Verify that the Gaussian parameters (position, scale, rotation) can be efficiently encoded and decoded

- **Region-of-Interest (ROI) Detection**: Identifies areas of frames containing important visual information
  - *Why needed*: Allows focusing computational resources on relevant content rather than processing entire frames uniformly
  - *Quick check*: Ensure ROI detection is accurate and doesn't miss important content

- **Inter-frame Redundancy Reduction**: Encodes differences between consecutive frames to exploit temporal coherence
  - *Why needed*: Reduces bitrate by avoiding redundant encoding of static or slowly changing regions
  - *Quick check*: Verify that motion compensation and difference encoding work correctly

## Architecture Onboarding

**Component Map**: ROI Detector -> Content-aware Initializer -> Gaussian Splatting Encoder -> Inter-frame Redundancy Reducer -> Decoder

**Critical Path**: ROI detection → Gaussian parameter initialization → Frame encoding/decoding → Redundancy reduction between frames

**Design Tradeoffs**: The method prioritizes encoding speed and real-time decoding over maximum compression ratio. This makes it suitable for video conferencing but potentially less optimal for archival storage where compression ratio is paramount.

**Failure Signatures**: Performance degradation occurs with rapid scene changes, multiple moving objects, or complex backgrounds. The ROI-based approach may miss important content in high-motion scenarios.

**First Experiments**: 1) Test encoding speed on static vs. dynamic video content, 2) Measure quality degradation with increasing motion complexity, 3) Compare bitrate savings against traditional codecs (H.265/HEVC, H.266/VVC)
