# SHapley Estimated Explanation (SHEP): A Fast Post-Hoc Attribution Method for Interpreting Intelligent Fault Diagnosis

## Quick Facts
- arXiv ID: 2504.03773
- Source URL: https://arxiv.org/abs/2504.03773
- Reference count: 40
- Primary result: SHEP achieves linear complexity for post-hoc interpretability in intelligent fault diagnosis by approximating SHAP through patch-wise attribution and subset simplification

## Executive Summary
SHapley Estimated Explanation (SHEP) addresses the computational inefficiency of SHAP in post-hoc interpretability for intelligent fault diagnosis (IFD). The method combines patch-wise attribution with SHEP to reduce feature dimensions and simplify subset enumeration, achieving linear complexity from exponential. Patch-wise attribution groups adjacent features into patches, trading granularity for efficiency, while SHEP approximates SHAP by evaluating only two representative cases (remove and add). Experiments validate SHEP's reliability in approximating SHAP across various domains and datasets, including open-source CWRU bearing data and a private helical gearbox dataset.

## Method Summary
SHEP introduces a two-stage approach to make SHAP computationally feasible for intelligent fault diagnosis. First, patch-wise attribution reduces feature dimensions by grouping adjacent features into patches, significantly decreasing the number of features that need to be evaluated. Second, SHEP simplifies the subset enumeration process by approximating the Shapley values using only two representative cases: one where features are removed and another where they are added. This combination achieves linear complexity compared to SHAP's exponential complexity while maintaining interpretability. The method demonstrates strong consistency with SHAP across various domains and datasets.

## Key Results
- Achieves linear computational complexity compared to SHAP's exponential complexity
- Validated on CWRU bearing data and private helical gearbox dataset
- Demonstrates strong consistency with SHAP while significantly reducing computational time
- Enables feasible real-time interpretation for monitoring tasks

## Why This Works (Mechanism)
SHEP works by fundamentally restructuring the computational approach to SHAP. The patch-wise attribution reduces the dimensionality problem by treating groups of adjacent features as single units, which dramatically reduces the number of subsets that need to be evaluated. The SHEP approximation further simplifies the calculation by recognizing that only two representative cases (removal and addition) are needed to approximate the Shapley values effectively. This combination addresses the core computational bottleneck of SHAP while preserving the essential interpretability properties needed for fault diagnosis applications.

## Foundational Learning
- SHAP (SHapley Additive exPlanations): Game-theoretic approach for explaining model predictions by calculating feature contributions
  - Why needed: Provides theoretically sound feature attribution but suffers from exponential computational complexity
  - Quick check: Verify that all feature contributions sum to the difference between model output and baseline
- Patch-wise attribution: Grouping adjacent features into larger units to reduce dimensionality
  - Why needed: Reduces the exponential growth of feature combinations that need to be evaluated
  - Quick check: Ensure patch boundaries don't split semantically related features
- Subset enumeration: Process of evaluating all possible feature combinations to calculate Shapley values
  - Why needed: Traditional SHAP requires evaluating all 2^n subsets for n features
  - Quick check: Confirm that the approximation maintains the symmetry and linearity properties of Shapley values

## Architecture Onboarding
- Component map: Input features -> Patch-wise grouping -> Subset simplification (SHEP) -> Attribution calculation -> Output interpretation
- Critical path: Feature grouping occurs first, followed by subset evaluation using the two-case approximation method
- Design tradeoffs: Granularity vs. efficiency (larger patches = faster but less detailed), accuracy vs. speed (two-case approximation vs. full enumeration)
- Failure signatures: Poor patch boundaries leading to mixed feature contributions, inadequate patch size causing loss of important features
- First experiments: 1) Test on simple synthetic data with known feature importance, 2) Compare computational time vs. SHAP on small datasets, 3) Validate patch size impact on attribution accuracy

## Limitations
- Trade-off between computational efficiency and granularity due to patch-wise attribution
- Validation limited to single open-source dataset (CWRU) and one private dataset
- Approximation quality validated only against SHAP, without comparison to other attribution methods
- Real-time monitoring validation not empirically demonstrated

## Confidence
- High confidence: Computational complexity reduction claim is mathematically sound
- Medium confidence: Consistency with SHAP claims supported by limited experimental data
- Medium confidence: Real-time interpretation capability theoretically supported but not empirically validated
