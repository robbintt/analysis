# VayuChat: An LLM-Powered Conversational Interface for Air Quality Data Analytics

## Quick Facts
- arXiv ID: 2511.01046
- Source URL: https://arxiv.org/abs/2511.01046
- Reference count: 14
- Enables natural language air quality queries with executable Python code and interactive visualizations

## Executive Summary
VayuChat is a conversational AI system designed to make air quality data analytics accessible through natural language queries. The system integrates CPCB air quality data with state demographics and NCAP funding records to enable comprehensive environmental analysis. By generating transparent, executable Python code alongside responses, VayuChat ensures reproducibility and reduces the risk of AI hallucinations in data analysis tasks.

## Method Summary
VayuChat employs a conversational interface that translates user questions into executable Python code for air quality data analysis. The system accesses multiple datasets including real-time CPCB monitoring data, demographic information, and environmental funding records. Code generation is central to the approach, providing users with transparent, reproducible analysis workflows that can be verified and modified. The interface produces interactive visualizations alongside code outputs, enabling users to explore air quality patterns and correlations across different geographic and temporal dimensions.

## Key Results
- Successfully analyzed Delhi's December 2024 pollution showing PM2.5 exceeding 300 μg/m³ when wind speeds dropped below 1.0 m/s
- Identified strong correlation (r = 0.47) between PM2.5 and CO concentrations in Delhi
- Demonstrated system capability to generate executable Python code for air quality data analysis

## Why This Works (Mechanism)
VayuChat leverages large language models to bridge the gap between natural language queries and complex data analysis workflows. By generating actual Python code rather than just textual responses, the system ensures that analyses are transparent, verifiable, and reproducible. The code-first approach provides a concrete mechanism for users to understand exactly how conclusions are derived, reducing ambiguity and enabling expert validation of results.

## Architecture Onboarding
- **Component Map**: User Query -> LLM Code Generator -> Data Processing Pipeline -> Visualization Engine -> Results Display
- **Critical Path**: Query interpretation and code generation must complete before data processing can begin; visualization depends on successful data processing.
- **Design Tradeoffs**: Code generation prioritizes transparency over speed; uses multiple data sources for comprehensiveness but increases complexity.
- **Failure Signatures**: Query ambiguity leads to incorrect code generation; data format changes break processing pipeline; visualization errors indicate data integration issues.
- **First Experiments**:
  1. Test system with simple query like "Show Delhi PM2.5 data for last week" to verify basic functionality
  2. Execute generated code independently to confirm reproducibility
  3. Validate visualization outputs against known data patterns
