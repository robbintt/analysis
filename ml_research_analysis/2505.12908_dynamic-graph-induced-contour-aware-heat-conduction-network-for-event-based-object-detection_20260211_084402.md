# Dynamic Graph Induced Contour-aware Heat Conduction Network for Event-based Object Detection

## Quick Facts
- **arXiv ID:** 2505.12908
- **Source URL:** https://arxiv.org/abs/2505.12908
- **Reference count:** 40
- **Primary result:** CvHeat-DET achieves 53.6 mAP on EvDET200K, outperforming state-of-the-art detectors

## Executive Summary
This paper addresses event-based object detection, which suffers from limited feature modeling and high computational costs when using traditional CNNs or Transformers. To overcome these issues, the authors propose CvHeat-DET, a dynamic graph induced contour-aware heat conduction network that leverages the clear contour information in event streams to guide heat diffusion within a heat conduction model. The method integrates hierarchical graph structures to capture multi-scale features, predicting thermal diffusivity coefficients dynamically based on contour cues. Extensive experiments on EvDET200K, GEN1, and DSEC datasets demonstrate CvHeat-DET's superior performance, achieving 53.6 mAP on EvDET200K and outperforming state-of-the-art detectors. The approach effectively balances accuracy and efficiency, showing strong generalization across diverse scenarios.

## Method Summary
CvHeat-DET introduces a novel approach to event-based object detection by integrating heat conduction principles with dynamic graph structures. The method constructs hierarchical graph structures at multiple scales to capture spatial relationships in event data. Contour information from event streams is used to guide heat diffusion processes, with thermal diffusivity coefficients predicted dynamically based on these contours. The model combines local contour awareness with global heat conduction to enhance detection accuracy while maintaining computational efficiency. The architecture leverages the sparse nature of event data through graph-based representations rather than dense convolutional operations.

## Key Results
- Achieves 53.6 mAP on EvDET200K dataset, setting new state-of-the-art performance
- Outperforms existing event-based detection methods across multiple benchmarks
- Demonstrates strong generalization capabilities on GEN1 and DSEC datasets

## Why This Works (Mechanism)
The method exploits the clear contour information inherent in event-based vision data. Event cameras capture motion-induced brightness changes, producing sparse but temporally precise data with well-defined object boundaries. By using these contours to guide heat diffusion processes, the model can effectively propagate detection confidence across object regions while maintaining spatial coherence. The hierarchical graph structures enable multi-scale feature extraction without the computational overhead of dense convolutions, while the dynamic prediction of thermal diffusivity coefficients allows the model to adaptively focus on relevant regions based on contour information.

## Architecture Onboarding

**Component Map:** Event Stream -> Contour Extraction -> Graph Construction -> Heat Diffusion -> Feature Aggregation -> Detection Head

**Critical Path:** The heat diffusion process guided by contour information represents the critical path. This involves predicting thermal diffusivity coefficients based on contours, then using these coefficients to control how detection confidence propagates through the hierarchical graph structure.

**Design Tradeoffs:** The method trades off between leveraging the sparse nature of event data (computational efficiency) and ensuring sufficient context for detection (accuracy). The graph-based approach reduces computational load compared to dense convolutions but requires careful graph construction. The heat diffusion mechanism provides smooth confidence propagation but introduces additional hyperparameters.

**Failure Signatures:** The method may struggle with low-contrast scenarios where contour information is degraded, leading to poor thermal diffusivity coefficient prediction and ineffective heat diffusion. Noisy event streams could produce spurious contours, causing incorrect diffusion patterns. Objects with weak motion signatures may not generate sufficient events, resulting in incomplete contour information.
