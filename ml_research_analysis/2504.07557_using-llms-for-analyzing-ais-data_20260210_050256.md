# Using LLMs for Analyzing AIS Data

## Quick Facts

## Executive Summary
This study investigates the effectiveness of Large Language Models (LLMs) for analyzing Automatic Identification System (AIS) maritime data through four distinct approaches: natural language interfaces to spatial databases (NLIDB), reasoning on raw data, compressed trajectories, and semantic trajectories. The research designs 27 queries across four categories (attribute, individual trajectory, interaction, and data-fusion) and establishes ground truth for systematic evaluation. The study reveals that while NLIDB methods demonstrate stable performance across varying dataset sizes, Zero-Shot Approaches (ZSA) experience significant performance degradation as data volume increases.

The findings demonstrate that raw data ZSA approaches outperform compressed and semantic variants on smaller datasets, showcasing LLMs' potential to directly process spatiotemporal data without preprocessing. However, all methods encounter substantial challenges with complex queries involving ship interactions, particularly cluster detection. The research highlights both the promise of LLMs functioning as Geographic Information Systems (GIS) and their current limitations in handling large-scale spatiotemporal analysis tasks.

## Method Summary
The study evaluates LLMs for AIS data analysis using four approaches: NLIDB with PostGIS, ZSA on raw AIS data, ZSA on compressed trajectories, and ZSA on semantic trajectories. Researchers designed 27 queries across four categories and established ground truth using PostGIS for comparison. Performance was tested across different dataset sizes to assess scalability.

## Key Results
- NLIDB methods showed stable performance across dataset sizes, while ZSA methods degraded as data volume increased
- Raw data ZSA approach outperformed compressed and semantic variants on smaller datasets
- All methods struggled significantly with complex interaction queries, particularly cluster detection

## Limitations
- Constrained query set of 27 questions may not represent real-world complexity
- Evaluation framework relies on ground truth from PostGIS, introducing potential bias
- Performance degradation of ZSA methods with increasing dataset sizes indicates scalability concerns
- Limited exploration of temporal reasoning capabilities in LLMs
- Insufficient analysis of error patterns and failure modes

## Confidence
**High Confidence:** Comparative performance analysis between NLIDB and ZSA approaches across different dataset sizes is well-supported by experimental results.

**Medium Confidence:** LLMs' ability to process spatiotemporal data directly from raw formats is supported but requires validation across more diverse query types.

**Low Confidence:** Claim that LLMs can function as GIS systems is premature given demonstrated limitations in complex interaction queries.
