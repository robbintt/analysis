# Dual Engines of Thoughts: A Depth-Breadth Integration Framework for Open-Ended Analysis

## Quick Facts
- arXiv ID: 2504.07872
- Source URL: https://arxiv.org/abs/2504.07872
- Authors: Fei-Hsuan Yu; Yun-Cheng Chou; Teng-Ruei Chen
- Reference count: 40
- Key outcome: Dual-engine architecture achieves 77-86% win rate over GPT-4o and Perplexity AI on news analysis tasks

## Executive Summary
The paper introduces the Dual Engines of Thoughts (DEoT), a framework for comprehensive open-ended reasoning that addresses the challenge of balancing broad exploration and deep investigation in complex analytical tasks. DEoT employs a dual-engine architecture with a Breadth Engine for systematic multi-dimensional exploration and a Depth Engine for focused in-depth analysis, coordinated by an Engine Controller. The framework demonstrates superior performance on news-to-question analysis tasks across five domains, outperforming both GPT-4o and Perplexity AI in analytical depth, comprehensiveness, and innovation.

## Method Summary
DEoT implements a dual-engine architecture where a Breadth Engine conducts systematic multi-dimensional exploration of topics while a Depth Engine performs focused in-depth analysis on specific aspects. An Engine Controller dynamically selects between these engines based on context and progress signals. The framework includes a Base Prompter for query optimization, a Solver Agent for task decomposition and execution, and an Analysis Toolbox with specialized tools. The system processes news content through iterative cycles of exploration and investigation, with each engine contributing complementary analytical capabilities to produce comprehensive insights.

## Key Results
- Total win rate of 77-86% over GPT-4o and Perplexity AI across five domains
- Superior analytical depth performance with 85-93% win rate
- Strong innovation metrics with 84-92% win rate
- Best performance in Economics (85% win rate) and Geopolitics (84% win rate)

## Why This Works (Mechanism)
The dual-engine architecture enables complementary analytical approaches by separating broad exploration from deep investigation. The Breadth Engine systematically covers multiple dimensions of a topic, ensuring comprehensive coverage and identifying relevant subtopics. The Depth Engine then focuses on specific aspects identified during exploration, conducting thorough analysis with specialized tools. The Engine Controller's dynamic switching between these modes allows the system to adapt its analytical approach based on task requirements and progress, preventing both shallow coverage and excessive focus on irrelevant details.

## Architecture Onboarding

**Component Map:**
Base Prompter -> Engine Controller -> (Breadth Engine <-> Depth Engine) -> Solver Agent -> Analysis Toolbox

**Critical Path:**
News input → Base Prompter optimization → Engine Controller decision → Breadth/Depth Engine execution → Solver Agent orchestration → Tool utilization → Final analysis output

**Design Tradeoffs:**
- Breadth vs Depth balance: Prioritizes comprehensive coverage but adds complexity
- Dynamic switching overhead: Enables adaptability but increases computational cost
- Multi-agent coordination: Provides specialized capabilities but requires complex orchestration

**Failure Signatures:**
- Stale exploration when Breadth Engine continues without Depth Engine convergence
- Tunnel vision when Depth Engine fails to return control to Breadth Engine
- Context loss during engine transitions leading to fragmented analysis

**First Experiments:**
1. Unit test individual engine components with controlled inputs
2. Integration test engine switching with synthetic news articles
3. End-to-end test on single-domain news analysis before multi-domain evaluation

- Engine Controller decision-making lacks transparency about underlying mechanisms
