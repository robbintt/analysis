# Retrieval-Augmented Search for Large-Scale Map Collections with ColPali

## Quick Facts
- **arXiv ID**: 2510.25718
- **Source URL**: https://arxiv.org/abs/2510.25718
- **Reference count**: 19
- **Primary result**: map-RAS enables multimodal search across 101,233 historic maps using ColPali, processing queries in under one second on consumer hardware

## Executive Summary
This paper introduces map-RAS, a retrieval-augmented search system designed for large-scale historic map collections. The system addresses the limitations of traditional metadata and OCR-based search methods by leveraging ColPali, a document-retrieval framework that uses multiple patch embeddings to capture both visual and textual elements of maps. Built on 101,233 map images from the Library of Congress, map-RAS supports multimodal queries, dynamic corpus expansion, and LLM-based summary generation. The system demonstrates practical feasibility by processing embeddings on consumer-grade hardware while maintaining query response times under one second.

## Method Summary
map-RAS employs ColPali to extract patch embeddings from map images, enabling retrieval based on both visual and textual features. The system processes 101,233 map images from the Library of Congress, generating embeddings that capture different sections of each document. Users can search via text or image, receiving ranked results with associated metadata. The architecture supports dynamic corpus expansion, allowing new maps to be added without retraining. Query processing leverages GPU acceleration where available, with fallback to CPU processing. The system integrates with LLMs to generate summaries of search results, enhancing discoverability and contextual understanding.

## Key Results
- Successfully processes 101,233 map images from the Library of Congress
- Achieves query response times averaging under one second on consumer hardware
- Enables multimodal search combining visual and textual query capabilities
- Supports dynamic corpus expansion without retraining requirements

## Why This Works (Mechanism)
The system works by leveraging ColPali's patch-based embedding approach, which captures both visual elements (geographic features, symbols) and textual components (labels, annotations) within maps. By processing multiple patches per document, the system creates rich multimodal representations that traditional OCR or metadata-based approaches cannot achieve. The embedding generation runs efficiently on consumer hardware, while the retrieval pipeline maintains sub-second response times even at large scale. The integration with LLMs for result summarization adds contextual understanding beyond simple ranked lists.

## Foundational Learning

**Patch-based document embeddings**: Needed because maps contain both visual and textual information that single embeddings cannot capture effectively. Quick check: Verify that multiple patches per document improve retrieval relevance compared to single embeddings.

**Multimodal retrieval**: Essential for maps that combine cartographic elements with textual annotations. Quick check: Test retrieval performance using only visual features versus only textual features to quantify multimodal benefit.

**Dynamic corpus expansion**: Critical for archival collections that grow continuously. Quick check: Measure embedding generation time and storage requirements for incremental additions to validate scalability claims.

## Architecture Onboarding

**Component map**: User query -> ColPali embedding generation -> Vector database search -> Result ranking -> LLM summary generation -> Display interface

**Critical path**: Query input → Patch embedding extraction → Vector similarity search → Result ranking → LLM processing → User display

**Design tradeoffs**: ColPali patch approach balances comprehensiveness against computational cost; vector database indexing enables fast retrieval but requires significant storage; LLM integration adds contextual value but introduces latency and cost.

**Failure signatures**: Poor retrieval quality when maps have unusual layouts; increased latency with larger corpora; LLM hallucinations in generated summaries; hardware limitations affecting embedding generation speed.
