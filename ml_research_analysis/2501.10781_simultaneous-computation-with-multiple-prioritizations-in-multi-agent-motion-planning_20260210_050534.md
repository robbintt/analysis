# Simultaneous Computation with Multiple Prioritizations in Multi-Agent Motion Planning

## Quick Facts
- arXiv ID: 2501.10781
- Source URL: https://arxiv.org/abs/2501.10781
- Reference count: 15
- Key outcome: Introduces simultaneous computation with multiple prioritizations for MAPF, achieving near-optimal solutions with low computation time for CAVs on road networks

## Executive Summary
This paper addresses the computational challenge of multi-agent path finding (MAPF) in large networks by introducing an approach for simultaneous computation with multiple prioritizations. The core method involves allowing agents to compute with multiple prioritizations simultaneously, leveraging the sequential nature of prioritized planning (PP) to improve solution quality while maintaining low computation time. The approach is demonstrated in the context of multi-agent motion planning (MAMP) for connected and automated vehicles (CAVs) on road networks, using receding horizon planning subject to computation time constraints. Numerical experiments show that the proposed approach comes close to optimal prioritization and outperforms state-of-the-art methods with only a minor increase in computation time, achieving real-time capability in experiments with ten vehicles in a Cyber-Physical Mobility Lab.

## Method Summary
The proposed method enables agents to compute with multiple prioritizations simultaneously by leveraging the sequential nature of prioritized planning. Instead of committing to a single agent ordering, the approach allows agents to explore different priority levels in parallel. This is particularly useful in multi-agent motion planning for CAVs on road networks, where the system operates under receding horizon planning with strict computation time constraints. The method balances solution quality and computational efficiency by intelligently distributing computational resources across multiple prioritization scenarios, allowing for better overall path coordination without excessive computation time.

## Key Results
- The approach achieves near-optimal prioritization compared to state-of-the-art methods
- Maintains low computation time while significantly improving solution quality
- Demonstrates real-time capability with ten vehicles in a Cyber-Physical Mobility Lab

## Why This Works (Mechanism)
The mechanism works by exploiting the inherent parallelism in multi-agent decision-making while maintaining the benefits of sequential prioritization. By allowing agents to compute with multiple priority levels simultaneously, the system can identify better coordination opportunities that single-prioritization approaches might miss. The sequential nature of prioritized planning provides a natural structure for distributing computational effort across different scenarios, while the receding horizon framework ensures the system remains responsive to dynamic changes in the environment.

## Architecture Onboarding

**Component Map**: PP Planner -> Multi-Prioritization Module -> Coordination Layer -> CAV Controller

**Critical Path**: Multi-prioritization computation -> solution selection -> plan execution -> feedback to planning

**Design Tradeoffs**: 
- Computation time vs. solution quality
- Number of simultaneous prioritizations vs. resource allocation
- Planning horizon length vs. real-time responsiveness

**Failure Signatures**:
- Deadlock situations when too many agents prioritize simultaneously
- Suboptimal solutions when computational resources are insufficient
- Timing violations when coordination overhead exceeds budget

**3 First Experiments**:
1. Single intersection with varying vehicle densities to test basic coordination
2. Multi-intersection network with ten vehicles to validate real-time capability
3. Stress test with maximum simultaneous prioritizations to find computational limits
