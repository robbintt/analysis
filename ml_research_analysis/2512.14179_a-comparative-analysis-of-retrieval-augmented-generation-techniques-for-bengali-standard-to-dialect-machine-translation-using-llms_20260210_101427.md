# A Comparative Analysis of Retrieval-Augmented Generation Techniques for Bengali Standard-to-Dialect Machine Translation Using LLMs

## Quick Facts
- arXiv ID: 2512.14179
- Source URL: https://arxiv.org/abs/2512.14179
- Authors: K. M. Jubair Sami; Dipto Sumit; Ariyan Hossain; Farig Sadeque
- Reference count: 30
- Primary result: Sentence-pair retrieval pipeline reduced WER from 76% to 55% for Chittagong dialect

## Executive Summary
This paper addresses the challenge of translating from standard Bengali to its regional dialects, which is difficult due to limited parallel data and linguistic variation. The authors propose two novel retrieval-augmented generation (RAG) pipelines: one based on audio transcripts and another using structured standard-to-dialect sentence pairs. Evaluated across six Bengali dialects and multiple large language models (LLMs), the sentence-pair pipeline significantly outperformed the transcript-based approach, reducing Word Error Rate (WER) from 76% to 55% for the Chittagong dialect. Critically, this approach enabled smaller models (e.g., Llama-3.1-8B) to outperform much larger ones (e.g., GPT-OSS-120B), demonstrating that a well-designed retrieval strategy can be more important than model size for low-resource dialect translation.

## Method Summary
The study introduces two retrieval-augmented generation pipelines for Bengali standard-to-dialect translation. The first uses audio transcripts from six regional dialects as the retrieval database, while the second employs structured standard-to-dialect sentence pairs. Both approaches leverage context from the retrieval database to improve translation quality. The methods were evaluated using Word Error Rate (WER) across six Bengali dialects, testing multiple LLM configurations including GPT-OSS-120B, Llama-3.1-8B, and Llama-3.1-70B. The sentence-pair approach demonstrated superior performance by providing more structured and relevant context compared to the transcript-based method.

## Key Results
- Sentence-pair retrieval pipeline reduced WER from 76% to 55% for Chittagong dialect
- Smaller models (Llama-3.1-8B) outperformed larger models (GPT-OSS-120B) when using effective retrieval strategies
- Structured sentence pairs provided better context than audio transcripts for dialect translation

## Why This Works (Mechanism)
The sentence-pair retrieval pipeline succeeds because it provides structured, parallel context that directly maps standard Bengali to dialect expressions. This structured approach allows the model to understand dialect-specific variations more effectively than audio transcripts, which may contain noise and less direct mappings. The retrieval database acts as a bridge between standard and dialect forms, enabling the model to generate more accurate translations by leveraging learned patterns from the paired examples.

## Foundational Learning

**Retrieval-Augmented Generation (RAG)**
- *Why needed*: Combines retrieval systems with generative models to enhance context-aware generation
- *Quick check*: Verify that retrieved documents are relevant to the input query

**Word Error Rate (WER)**
- *Why needed*: Standard metric for evaluating speech and translation quality
- *Quick check*: Calculate WER using reference and hypothesis word sequences

**Low-resource Language Translation**
- *Why needed*: Addresses challenges when limited parallel data exists for dialect pairs
- *Quick check*: Assess vocabulary overlap between standard and dialect forms

## Architecture Onboarding

**Component Map**: Input Text -> Retrieval Module -> LLM Generator -> Output Translation

**Critical Path**: Standard Bengali text → Sentence-pair retrieval database lookup → Contextual prompt augmentation → LLM generation → Dialect translation

**Design Tradeoffs**: Structured sentence pairs provide better context but require manual creation, while audio transcripts are easier to collect but contain more noise and less direct mappings

**Failure Signatures**: High WER indicates poor retrieval relevance or insufficient coverage of dialect variations in the database

**Exactly 3 first experiments**:
1. Test retrieval relevance by measuring overlap between retrieved sentences and input queries
2. Evaluate WER reduction when varying the size of the sentence-pair database
3. Compare performance across different dialect pairs to identify which benefit most from the approach

## Limitations
- Results rely on a single crowdsourced dialect translation dataset without external validation
- Performance gains may be specific to the data collection methodology and dialect selection
- Does not address potential biases in crowdsourced translations or generalization to unrepresented dialects

## Confidence

**High confidence in**: The relative performance comparison between the two retrieval pipelines (sentence-pair vs transcript-based) and their impact on WER reduction.

**Medium confidence in**: The generalizability of findings across different Bengali dialects and the claim about retrieval strategy importance over model size.

**Low confidence in**: The robustness of results to different data collection methods and the absence of validation on external benchmark datasets.

## Next Checks

1. Replicate the study using an independent Bengali dialect translation dataset from a different source to verify the consistency of performance gains across data collection methodologies.

3. Conduct ablation studies varying the retrieval database size, quality thresholds, and document selection criteria to quantify the contribution of each component to the observed performance improvements.
