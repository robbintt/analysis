# WebCoach: Self-Evolving Web Agents with Cross-Session Memory Guidance

## Quick Facts
- arXiv ID: 2511.12997
- Source URL: https://arxiv.org/abs/2511.12997
- Reference count: 21
- Key outcome: WebCoach improved WebVoyager task success from 47% to 61% without retraining

## Executive Summary
WebCoach is a model-agnostic framework that equips web browsing agents with persistent cross-session memory, enabling them to retrieve and condense past navigation experiences into actionable coaching advice. This self-evolving approach improves long-term robustness and efficiency without requiring retraining of the base agent. On the WebVoyager benchmark, WebCoach significantly boosted task success rates and reduced average steps, with smaller models achieving performance comparable to much larger ones.

## Method Summary
WebCoach introduces a memory module that stores past trajectories and a retrieval system that identifies relevant experiences from previous sessions. These are condensed into coaching advice, which is integrated into the agent's decision-making process during new tasks. The framework is designed to be model-agnostic, meaning it can be applied to various web agents without architectural changes. By leveraging its own past experiences, the agent improves over time through continuous learning from its own navigation patterns.

## Key Results
- WebCoach increased WebVoyager task success rates by 14.4 points (47% to 61%) for a 38B agent
- Smaller models with WebCoach achieved performance comparable to GPT-4o
- Agents learned most effectively from their own trajectories, not external demonstrations
- The approach maintained or reduced average steps per task, indicating improved efficiency

## Why This Works (Mechanism)
WebCoach enables persistent learning by storing and retrieving past navigation experiences across sessions. The system condenses these memories into actionable coaching advice, which guides the agent during new tasks. This mechanism allows agents to avoid repeating past mistakes and to leverage successful strategies from previous sessions. The model-agnostic design ensures broad applicability, and the focus on self-trajectories fosters continuous improvement without external data.

## Foundational Learning
- **Cross-session memory**: Stores and retrieves past experiences across multiple sessions, enabling cumulative learning over time. Needed to overcome the limitations of session-bound memory. Quick check: Verify memory storage and retrieval work as intended across sessions.
- **Memory condensation**: Transforms raw past trajectories into concise, actionable advice. Needed to prevent information overload and ensure relevance. Quick check: Confirm condensed advice is both accurate and useful.
- **Model-agnostic integration**: Allows the framework to be applied to different web agents without retraining. Needed for broad applicability and ease of adoption. Quick check: Test with multiple agent architectures.
- **Self-trajectory learning**: Focuses on learning from the agent's own experiences rather than external demonstrations. Needed to foster robust, task-specific improvement. Quick check: Compare learning curves with and without self-trajectory focus.

## Architecture Onboarding

**Component Map**: WebAgent -> Memory Module <- Trajectory Storage -> Retrieval System -> Condenser -> Coaching Advice -> WebAgent

**Critical Path**: During a new task, the agent consults the memory module, which retrieves and condenses relevant past trajectories into coaching advice. This advice is then integrated into the agent's decision-making process.

**Design Tradeoffs**: The system prioritizes cross-session persistence and self-evolution over retraining, trading off some initial performance for long-term improvement. The focus on self-trajectories avoids reliance on external data but may limit diversity of experience.

**Failure Signatures**: Poor retrieval or condensation can lead to irrelevant or misleading advice, potentially degrading performance. Over-reliance on memory may reduce adaptability to novel scenarios not present in past trajectories.

**First Experiments**:
1. Verify basic memory storage and retrieval across sessions.
2. Test coaching advice integration and its impact on task success.
3. Conduct ablation studies to isolate the contribution of each component.

## Limitations
- Ablation study is limited, making it hard to isolate individual component contributions
- Only compared to baseline models, not newer approaches
- Focus on single-session memory condensation may not generalize to all task types

## Confidence
- Task success improvements: High
- Cross-session robustness: Medium
- Scalability and generalizability: Low

## Next Checks
1. Conduct a more comprehensive ablation study to quantify the individual contributions of the memory module, retrieval strategy, and coaching advice.
2. Benchmark WebCoach against recent state-of-the-art web agents and test on a wider variety of tasks, including those with longer or noisier trajectories.
3. Evaluate the scalability of the cross-session memory system with respect to trajectory length, diversity, and the introduction of noisy or irrelevant past experiences.
