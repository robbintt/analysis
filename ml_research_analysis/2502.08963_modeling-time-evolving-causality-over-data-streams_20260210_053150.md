# Modeling Time-evolving Causality over Data Streams

## Quick Facts
- arXiv ID: 2502.08963
- Source URL: https://arxiv.org/abs/2502.08963
- Reference count: 40
- Primary result: ModePlait achieves lower SHD/SID for causal discovery and lower RMSE/MAE for forecasting than state-of-the-art baselines on synthetic and COVID-19 data

## Executive Summary
This paper introduces ModePlait, a streaming method for modeling time-evolving causality in multivariate co-evolving data streams. The approach expresses exogenous variables as superpositions of modes and captures their temporal dynamics to identify causal relationships that evolve with distinct dynamical patterns (regimes). ModePlait discovers time-evolving causality by detecting regime transitions and forecasting future values in a streaming fashion. The method demonstrates superior performance over baselines like TimesNet, PatchTST, DeepAR, OrbitMap, and ARIMA in both causal discovery and forecasting accuracy.

## Method Summary
ModePlait operates by decomposing multivariate time series into mode superpositions, where each mode represents a distinct dynamical pattern. The method continuously monitors for regime transitions and updates causal structure estimates accordingly. At each time step, ModePlait processes incoming data points, updates mode representations, detects potential regime changes, and adjusts causal models. The streaming nature allows real-time adaptation to changing causal relationships while maintaining forecasting capabilities.

## Key Results
- ModePlait achieves lower Structural Hamming Distance (SHD) and Structural Intervention Distance (SID) for causal discovery compared to state-of-the-art methods
- The method demonstrates lower Root Mean Square Error (RMSE) and Mean Absolute Error (MAE) in forecasting tasks
- ModePlait maintains constant computational time relative to stream length, outperforming traditional methods in scalability

## Why This Works (Mechanism)
ModePlait leverages the principle that complex time series can be decomposed into simpler mode components, each with distinct dynamical properties. By identifying regime transitions where causal relationships change, the method can adapt its causal structure estimation in real-time. The superposition approach allows efficient representation of exogenous variables while maintaining the ability to detect subtle changes in causality patterns that would be missed by static methods.

## Foundational Learning

**Mode Decomposition**: Breaking time series into constituent modes representing different dynamical patterns. *Why needed*: Simplifies complex multivariate relationships. *Quick check*: Verify mode orthogonality and reconstruction accuracy.

**Regime Detection**: Identifying points where system dynamics change fundamentally. *Why needed*: Causality structures evolve over time. *Quick check*: Test detection accuracy on synthetic data with known regime boundaries.

**Streaming Causality**: Real-time estimation of causal relationships from sequential data. *Why needed*: Static methods cannot capture evolving dependencies. *Quick check*: Measure adaptation speed to regime changes.

## Architecture Onboarding

**Component Map**: Data Stream -> Mode Decomposition -> Regime Detection -> Causal Structure Update -> Forecasting

**Critical Path**: Incoming data points flow through mode decomposition, which feeds regime detection; detected changes trigger causal structure updates that directly impact forecasting outputs.

**Design Tradeoffs**: ModePlait trades computational overhead of continuous mode decomposition for improved adaptability to regime changes. The superposition approach balances expressiveness with computational efficiency.

**Failure Signatures**: Poor mode separation leads to missed regime transitions; excessive false positives in regime detection cause unnecessary model updates; incorrect causal structure estimation degrades forecasting accuracy.

**First Experiments**: 1) Test mode decomposition accuracy on synthetic data with known modes, 2) Validate regime detection on data with artificially inserted transitions, 3) Compare forecasting performance with and without regime adaptation.
