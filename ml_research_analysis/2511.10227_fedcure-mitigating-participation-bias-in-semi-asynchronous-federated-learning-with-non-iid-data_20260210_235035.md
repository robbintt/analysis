# FedCure: Mitigating Participation Bias in Semi-Asynchronous Federated Learning with Non-IID Data

## Quick Facts
- arXiv ID: 2511.10227
- Source URL: https://arxiv.org/abs/2511.10227
- Reference count: 18
- Primary result: 5.1x accuracy improvement over baselines in semi-asynchronous federated learning with non-IID data

## Executive Summary
FedCure introduces a three-rule framework to address participation bias in semi-asynchronous federated learning (SAFL) when dealing with non-IID data distributions. The system tackles the challenge of client availability disparities by strategically forming coalitions, scheduling participation through Bayesian estimation, and optimizing resource allocation based on coalition dynamics. By implementing preference rules for coalition formation, scheduling rules using virtual queues, and resource allocation rules for CPU frequency optimization, FedCure significantly improves model accuracy while reducing latency variation compared to existing approaches.

## Method Summary
FedCure employs a three-rule framework to mitigate participation bias in SAFL with non-IID data. The preference rule forms coalitions through client-Edge Server associations to minimize non-IID effects. The scheduling rule uses Bayesian-estimated coalition dynamics and virtual queues to balance participation across clients. The resource allocation rule optimizes client CPU frequencies based on coalition dynamics. The framework was evaluated on four real-world datasets, demonstrating substantial improvements in accuracy and latency consistency compared to four state-of-the-art baselines.

## Key Results
- Achieved up to 5.1x accuracy improvement compared to four state-of-the-art baselines
- Maintained lowest coefficient of variation for per-round latency at 0.0223
- Demonstrated long-term balance across diverse scenarios

## Why This Works (Mechanism)
FedCure addresses participation bias through three interconnected mechanisms: coalition formation that reduces non-IID effects, dynamic scheduling that balances client participation, and resource optimization that adapts to coalition dynamics. The system leverages Bayesian estimation to predict client availability patterns and uses virtual queues to manage participation fairness. By optimizing CPU frequencies based on coalition dynamics, the framework ensures efficient resource utilization while maintaining model convergence.

## Architecture Onboarding

Component Map: Client -> Edge Server -> Coalition Formation -> Scheduling -> Resource Allocation -> Model Training

Critical Path: Client availability detection → Coalition formation → Bayesian scheduling → Resource allocation → Local training → Aggregation

Design Tradeoffs: The framework balances between coalition formation complexity and scheduling efficiency, choosing Bayesian estimation over simpler heuristics for better long-term performance. Resource allocation optimization adds computational overhead but improves convergence speed and fairness.

Failure Signatures: Poor coalition formation may lead to increased non-IID effects, Bayesian estimation failures can cause scheduling imbalances, and suboptimal resource allocation may result in training bottlenecks or energy inefficiencies.

## Limitations
- Evaluation limited to specific real-world datasets without systematic testing across diverse data distributions
- 5.1x accuracy improvement lacks statistical significance testing across multiple runs
- Three-rule framework's computational overhead and deployment challenges in real-world edge environments remain unclear
- Assumes known client dynamics for Bayesian estimation, which may not hold in practice

## Confidence
- Accuracy improvements: Medium (limited statistical validation, potential hyperparameter sensitivity)
- Latency reduction: High (quantitative metrics with coefficient of variation provided)
- Long-term balance maintenance: Medium (simulation-based validation only, real-world deployment untested)
