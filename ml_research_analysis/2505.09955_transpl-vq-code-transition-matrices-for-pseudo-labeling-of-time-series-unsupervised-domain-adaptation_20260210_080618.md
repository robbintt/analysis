# TransPL: VQ-Code Transition Matrices for Pseudo-Labeling of Time Series Unsupervised Domain Adaptation

## Quick Facts
- arXiv ID: 2505.09955
- Source URL: https://arxiv.org/abs/2505.09955
- Reference count: 27
- Primary result: Achieves 6.1% accuracy and 4.9% F1 improvements over state-of-the-art UDA methods

## Executive Summary
This paper introduces TransPL, a novel pseudo-labeling approach for unsupervised domain adaptation (UDA) in time series classification. The method models the joint distribution of the source domain through code transition matrices derived from vector quantization (VQ) of time series patches. By constructing class- and channel-wise transition matrices and applying Bayes' rule, TransPL generates pseudo-labels for the target domain while explicitly modeling temporal transitions and channel-wise shifts between domains. Extensive experiments on four time series UDA benchmarks demonstrate consistent improvements over existing methods.

## Method Summary
TransPL addresses UDA in time series by modeling the source domain's joint distribution P(X,y) through vector quantization. The method partitions time series into patches, applies VQ to obtain discrete codes, and constructs class- and channel-wise transition matrices from the source domain. For target domain adaptation, it uses Bayes' rule to compute class-conditional likelihoods based on these transition matrices, generating pseudo-labels through channel-wise weighted likelihoods. This approach explicitly captures temporal patterns and domain-specific channel shifts, offering versatility across different UDA scenarios while providing explainable pseudo-label generation.

## Key Results
- Achieves 6.1% average accuracy improvement over state-of-the-art UDA methods
- Delivers 4.9% average F1 score improvement across benchmark datasets
- Demonstrates consistent performance gains across four diverse time series UDA benchmarks
- Provides interpretable insights through learned code transition matrices

## Why This Works (Mechanism)
TransPL works by explicitly modeling the temporal and channel-wise structure of time series data through vector quantization. The method captures how time series evolve over time (temporal transitions) and how different channels relate to each other (channel-wise relationships). By constructing these relationships from the source domain and applying Bayes' rule to the target domain, it can effectively transfer knowledge while accounting for domain shifts. The channel-wise weighting allows the method to handle cases where different channels have varying importance across domains.

## Foundational Learning
1. Vector Quantization (VQ) for time series
   - Why needed: To discretize continuous time series into manageable codes for transition matrix construction
   - Quick check: Verify that VQ preserves meaningful temporal patterns in the data

2. Transition matrices in temporal modeling
   - Why needed: To capture how time series states evolve over time and between classes
   - Quick check: Ensure transition probabilities reflect actual temporal dependencies

3. Bayes' rule for domain adaptation
   - Why needed: To compute class-conditional probabilities in the target domain using source domain knowledge
   - Quick check: Validate that prior and likelihood terms are properly estimated

4. Channel-wise adaptation
   - Why needed: To handle cases where different channels have different domain shifts
   - Quick check: Confirm that channel weights appropriately reflect domain differences

## Architecture Onboarding

Component Map: VQ -> Patch Extraction -> Transition Matrix Construction -> Bayes' Rule Application -> Pseudo-Label Generation

Critical Path: The method flows from vector quantization through patch extraction, transition matrix construction for each class and channel, followed by Bayes' rule application to compute class-conditional likelihoods, ultimately generating pseudo-labels for the target domain.

Design Tradeoffs: The approach trades computational complexity (constructing multiple transition matrices) for improved modeling of temporal and channel-wise relationships. The method requires source domain data for transition matrix construction, limiting its applicability in source-free scenarios.

Failure Signatures: Potential failures may occur when: (1) temporal patterns in source and target domains are fundamentally different, (2) transition matrices become sparse in high-dimensional settings, or (3) vector quantization fails to capture essential patterns in highly irregular time series.

First Experiments:
1. Validate VQ code quality on source domain time series patches
2. Test transition matrix construction with synthetic time series having known temporal patterns
3. Evaluate Bayes' rule application on a simple two-class adaptation scenario

## Limitations
- Generalizability beyond tested benchmark datasets remains uncertain
- Performance may degrade with highly irregular or non-stationary time series
- Source-free UDA scenarios are not supported due to reliance on source domain data
- Computational complexity increases with higher-dimensional time series

- High confidence: Mathematical formulation of code transition matrices and Bayes' rule application
- Medium confidence: Empirical performance improvements require validation on additional diverse datasets
