# Modality-Collaborative Low-Rank Decomposers for Few-Shot Video Domain Adaptation

## Quick Facts
- arXiv ID: 2511.18711
- Source URL: https://arxiv.org/abs/2511.18711
- Authors: Yuyang Wanyan; Xiaoshan Yang; Weiming Dong; Changsheng Xu
- Reference count: 40
- Key outcome: Proposed MC-LRD framework achieves significant improvements in few-shot video domain adaptation, outperforming existing multimodal methods by 4.2-5.0% on EPIC-Kitchens benchmark

## Executive Summary
This paper addresses the challenging task of Few-Shot Video Domain Adaptation (FSVDA) where limited labeled target videos are available. The authors propose Modality-Collaborative Low-Rank Decomposers (MC-LRD) to tackle the multimodal nature of videos, where each modality contains components with varying domain shifts. The framework decomposes modality-unique and modality-shared features using progressively shared low-rank decomposers and Multimodal Decomposition Routers (MDR) that selectively activate decomposers for better domain alignment.

The proposed method demonstrates significant performance improvements across three benchmarks (EPIC-Kitchens, UCF-HMDB, and Jester), achieving 49.9% and 52.2% mean accuracies in 1-shot and 5-shot settings on EPIC-Kitchens, respectively. The approach also reduces training complexity by updating fewer parameters compared to parallel multimodal approaches, while orthogonal decorrelation constraints and cross-domain activation consistency loss ensure efficient decomposition.

## Method Summary
MC-LRD decomposes features from each modality into modality-unique and modality-shared components using low-rank decomposers with progressively shared structures. Multimodal Decomposition Routers (MDR) selectively activate these decomposers based on input characteristics, producing features more amenable to domain alignment. The framework incorporates orthogonal decorrelation constraints to prevent feature redundancy and employs a cross-domain activation consistency loss to maintain consistent decomposition patterns across domains. This collaborative approach leverages both modality-specific and shared information while addressing varying domain shifts within each modality.

## Key Results
- Achieves 49.9% and 52.2% mean accuracies on EPIC-Kitchens in 1-shot and 5-shot settings, outperforming second-best multimodal method by 4.2% and 5.0%
- Demonstrates consistent improvements across three benchmarks (EPIC-Kitchens, UCF-HMDB, and Jester)
- Reduces training complexity by updating fewer parameters compared to parallel multimodal approaches
- Shows effectiveness of MDR routing mechanism in selective feature decomposition for domain alignment

## Why This Works (Mechanism)
The method works by decomposing multimodal video features into modality-unique and modality-shared components using low-rank decomposers. The Multimodal Decomposition Routers (MDR) act as adaptive selectors that route features through appropriate decomposers based on their domain shift characteristics. This selective routing ensures that features with similar domain properties are processed together, making subsequent domain alignment more effective. The orthogonal decorrelation constraints prevent feature redundancy by maintaining independence between decomposed components, while the cross-domain activation consistency loss ensures that the decomposition patterns remain stable across source and target domains, facilitating more reliable adaptation.

## Foundational Learning

**Few-Shot Video Domain Adaptation**: Transfer learning from source to target domain with minimal labeled target data - needed to handle real-world scenarios where target domain annotations are scarce; quick check: compare performance with increasing shot numbers.

**Low-Rank Decomposition**: Matrix factorization technique that approximates data using lower-dimensional representations - needed to separate modality-specific and shared components; quick check: verify rank selection impact on performance.

**Multimodal Decomposition Routers**: Neural network modules that selectively route features to appropriate processing paths - needed to handle varying domain shifts within modalities; quick check: analyze router activation patterns across domains.

**Orthogonal Decorrelation**: Constraint ensuring decomposed features are mutually independent - needed to prevent redundant feature representations; quick check: measure feature correlation before and after decorrelation.

**Cross-Domain Activation Consistency**: Loss function maintaining consistent decomposition patterns across domains - needed for stable domain adaptation; quick check: compare activation consistency between source and target domains.

## Architecture Onboarding

**Component Map**: Input Videos -> Feature Extractors -> Low-Rank Decomposers -> MDR Routers -> Domain Alignment Module -> Classifier

**Critical Path**: RGB/Flow Input → Feature Extractor → Low-Rank Decomposers → MDR Routing → Domain Alignment → Classification

**Design Tradeoffs**: The progressively shared low-rank structure balances between modality-specific processing and shared representation learning, trading some modality uniqueness for better domain alignment. The MDR adds routing complexity but enables more targeted decomposition.

**Failure Signatures**: Poor performance may manifest when domain shifts are too extreme for the decomposition strategy, when MDR routing becomes inconsistent across domains, or when low-rank decomposition fails to capture essential modality-specific features.

**First Experiments**: 1) Ablation study removing MDR routing to measure its contribution, 2) Test with different low-rank decomposition ranks to find optimal configuration, 3) Evaluate performance with and without orthogonal decorrelation constraints.

## Open Questions the Paper Calls Out

The paper does not explicitly call out open questions, but implicit limitations include the lack of detailed analysis of MDR routing decisions across different video domains, absence of comprehensive computational overhead evaluation, and limited exploration of failure cases where the decomposition strategy might be suboptimal.

## Limitations

- MDR routing mechanism lacks detailed analysis of routing decisions and adaptation to different video domains
- Computational overhead introduced by low-rank decomposers and MDR routing is not comprehensively evaluated
- Evaluation focuses primarily on accuracy metrics without considering computational efficiency, memory usage, or real-time performance implications
