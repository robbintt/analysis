# Dialogue-based Explanations for Logical Reasoning using Structured Argumentation

## Quick Facts
- arXiv ID: 2502.11291
- Source URL: https://arxiv.org/abs/2502.11291
- Authors: Loan Ho; Stefan Schlobach
- Reference count: 40
- Proposes argumentation-based framework for inconsistency-tolerant logical reasoning explanations

## Executive Summary
This paper introduces a novel framework for generating explanations in logical reasoning by leveraging structured argumentation and dialogue systems. The approach represents logical arguments as tree structures and uses collective attacks to capture complex non-binary conflicts within knowledge bases. By translating various logics into proof-oriented argumentation frameworks, the authors create explanatory dialogue models that serve as dialectical proof procedures, enabling agents to exchange arguments and counter-arguments. The framework is demonstrated to be sound and complete for different semantics, offering greater expressiveness than traditional explanation methods, particularly in cases involving multiple jointly inconsistent facts.

## Method Summary
The authors develop a generic argumentation-based framework that translates logical reasoning problems into structured argumentation systems. They represent logical arguments as tree structures and define collective attacks to model non-binary conflicts in knowledge bases. The framework introduces explanatory dialogue models as dialectical proof procedures, where successful dialogues between agents provide explanations for query answers. The approach is shown to be sound and complete across different semantics and can handle cases where traditional explanation methods fail, such as when multiple facts jointly cause inconsistency.

## Key Results
- Framework is sound and complete for different semantics
- More expressive and intuitive than existing explanation formalisms
- Successfully handles cases with multiple jointly inconsistent facts where traditional approaches fail

## Why This Works (Mechanism)
The framework works by structuring logical arguments as trees and using collective attacks to represent complex conflicts in knowledge bases. By translating logical reasoning into argumentation frameworks, it creates a dialectical proof procedure where dialogue exchanges between agents generate explanations. The proof-oriented nature ensures that successful dialogues directly correspond to valid reasoning paths, making the explanations both rigorous and interpretable.

## Architecture Onboarding

Component map:
Knowledge Base -> Logical Translator -> Argumentation Framework -> Dialogue Generator -> Explanation Output

Critical path:
Query → Translation to argumentation framework → Dialogue generation → Successful dialogue → Explanation

Design tradeoffs:
The framework prioritizes expressiveness and formal rigor over computational efficiency. By using structured argumentation and dialectical proof procedures, it achieves greater generality but may incur higher computational costs compared to simpler explanation methods.

Failure signatures:
- Dialogue generation failure indicates potential issues with argument structure or conflict representation
- Unsuccessful dialogues suggest the query may not be derivable from the knowledge base
- Translation failures point to limitations in mapping specific logics to the argumentation framework
