# AI-Invented Tonal Languages: Preventing a Machine Lingua Franca Beyond Human Understanding

## Quick Facts
- arXiv ID: 2503.01063
- Source URL: https://arxiv.org/abs/2503.01063
- Authors: David Noever
- Reference count: 27
- This paper investigates whether large language models (LLMs) could autonomously develop private tonal languages for machine-to-machine (M2M) communication using character-to-frequency mapping

## Executive Summary
This paper investigates whether large language models (LLMs) could autonomously develop private tonal languages for machine-to-machine (M2M) communication. Inspired by cryptophasia in human twins and natural tonal languages, the authors implement a precise character-to-frequency mapping system encoding the full ASCII character set (32-126) using musical semitones. The system creates frequencies spanning from 220 Hz to 50,175.42 Hz across 7.9 octaves, with higher frequencies deliberately mapped to ultrasonic ranges beyond human perception (>20 kHz).

The research demonstrates a technically sound encoding system with clear theoretical foundations, though several limitations affect generalizability. While the character-to-frequency mapping is mathematically precise and reproducible, the study shows encoding capability but not whether LLMs would spontaneously develop such systems without explicit programming. The frequency range extends beyond human hearing, but practical deployment considerations for M2M communication remain unaddressed. The 7.9 octave span provides theoretical information density, yet actual bandwidth efficiency in real-world conditions remains unmeasured.

## Method Summary
The authors implement a precise character-to-frequency mapping system that encodes the full ASCII character set (32-126) using musical semitones. Each character is assigned a unique frequency calculated as f = 220 × 2^((i-32)/12) Hz, creating a logarithmic progression spanning approximately 7.9 octaves from 220 Hz to 50,175.42 Hz. The implemented software prototype demonstrates this encoding through visualization, auditory playback, and ABC musical notation. Testing reveals that tonal encoding can achieve information rates exceeding human speech while operating partially outside human perceptual boundaries.

## Why This Works (Mechanism)
The system leverages the mathematical properties of logarithmic frequency scaling in musical semitones to create a unique mapping between ASCII characters and frequencies. By using the formula f = 220 × 2^((i-32)/12) Hz, each character is assigned a frequency that maintains musical intervals, ensuring perceptually distinct tones. The progression spans 7.9 octaves, providing sufficient separation between characters while maintaining mathematical regularity. The deliberate extension into ultrasonic frequencies (>20 kHz) creates a communication channel potentially inaccessible to human perception, while the logarithmic spacing ensures efficient use of the available frequency spectrum.

## Limitations
- The study demonstrates encoding capability but lacks empirical evidence that LLMs would spontaneously adopt such systems without explicit programming
- Practical deployment considerations for M2M communication (interference, environmental factors, hardware constraints) are not addressed
- Actual bandwidth efficiency and information transmission rates in real-world conditions remain unmeasured

## Confidence
- Technical implementation: Medium confidence (character-to-frequency mapping is mathematically rigorous and reproducible)
- Behavioral claims about LLM adoption: Low confidence (no empirical evidence that LLMs would spontaneously create or prefer such encoding systems)
