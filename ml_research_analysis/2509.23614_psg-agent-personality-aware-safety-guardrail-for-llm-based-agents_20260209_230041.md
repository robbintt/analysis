# PSG-Agent: Personality-Aware Safety Guardrail for LLM-based Agents

## Quick Facts
- arXiv ID: 2509.23614
- Source URL: https://arxiv.org/abs/2509.23614
- Authors: Yaozu Wu, Jizhou Guo, Dongyuan Li, Henry Peng Zou, Wei-Chieh Huang, Yankai Chen, Zhen Wang, Weizhi Zhang, Yangning Li, Meng Zhang, Renhe Jiang, Philip S. Yu
- Reference count: 40
- Primary result: Achieves 79.7% accuracy and 0.744 F1-score on personalized safety guardrails

## Executive Summary
PSG-Agent introduces a personality-aware safety guardrail system for LLM-based agents that addresses the gap between generic safety measures and individual user needs. The system mines user profiles from interaction history and generates personalized safety criteria through a two-stage framework. By employing specialized guards throughout the agent pipeline, PSG-Agent tracks cross-turn risk accumulation while maintaining training-free, plug-and-play compatibility. Evaluated across healthcare, finance, and daily life scenarios, the system significantly outperforms existing guardrails while providing auditable safety decisions tailored to individual user profiles.

## Method Summary
PSG-Agent operates through a two-stage framework that first mines user profiles from historical interactions and then generates personalized safety criteria. The system employs multiple specialized guards (Profile Miner, Input Guard, Plan Monitor, Action Guard, and Exit Guard) that work in sequence throughout the agent pipeline. Each guard evaluates different aspects of risk based on the user's mined profile, with cross-turn risk accumulation tracking ensuring cumulative risk monitoring. The approach is training-free and designed to be plug-and-play compatible with existing LLM agents, requiring no model updates or fine-tuning.

## Key Results
- Achieves 79.7% accuracy and 0.744 F1-score on personalized safety guardrail evaluation
- Significantly outperforms existing guardrails like LlamaGuard3 on comprehensive benchmark
- Maintains effectiveness across three distinct domains: healthcare, finance, and daily life scenarios

## Why This Works (Mechanism)
The system works by creating a feedback loop between user profile mining and safety decision-making. By extracting stable attributes from interaction history and combining them with dynamic personality traits, PSG-Agent generates nuanced safety criteria that reflect both inherent user characteristics and current contexts. The multi-guard architecture allows for specialized risk assessment at different pipeline stages, while cross-turn tracking ensures that risk accumulates appropriately over extended interactions. This personalized approach addresses the limitation of one-size-fits-all safety measures that either overprotect or underprotect users based on their individual circumstances.

## Foundational Learning
- **Profile Mining**: Extracting user traits from interaction history - needed to establish baseline user characteristics for personalized safety decisions; quick check: accuracy of extracted traits vs ground truth
- **Cross-turn Risk Tracking**: Monitoring cumulative risk across multiple agent turns - needed to prevent risk accumulation that might be missed in isolated decisions; quick check: risk detection rates over extended conversation sequences
- **Personalized Safety Criteria**: Generating user-specific thresholds based on mined profiles - needed to balance protection with user autonomy; quick check: false positive rates across different personality types

## Architecture Onboarding
- **Component Map**: User Query -> Profile Miner -> Input Guard -> Plan Monitor -> Action Guard -> Exit Guard -> LLM Response
- **Critical Path**: The primary decision flow goes through all guards sequentially, with each stage potentially blocking progression based on risk assessment
- **Design Tradeoffs**: Personalization vs privacy (collecting user data for safety), accuracy vs latency (multiple LLM inferences), customization vs complexity (maintaining multiple guard components)
- **Failure Signatures**: Overly restrictive behavior (profile mining errors), inconsistent decisions (guard coordination failures), excessive latency (guard pipeline bottlenecks)
- **First Experiments**: 1) Test profile mining accuracy with controlled interaction histories, 2) Measure guard pipeline latency under different loads, 3) Evaluate cross-turn risk tracking with extended interaction sequences

## Limitations
- Lacks detailed methodology for personality mining, making reliability assessment difficult
- Claims "training-free" operation while achieving high accuracy, raising questions about adaptation mechanisms
- No cross-cultural validation, which is critical for personality-aware systems that may perform differently across cultural contexts

## Confidence
- Performance metrics (79.7% accuracy, 0.744 F1-score): High confidence
- Two-stage framework effectiveness: Medium confidence
- Cross-turn risk accumulation tracking: Medium confidence
