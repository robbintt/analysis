## Executive Summary

The integration of Large Language Models (LLMs) with 3D spatial reasoning faces a critical scalability bottleneck: existing methods require serializing entire 3D Scene Graphs (3DSGs) into text within the LLM's context window. This approach fails to scale effectively as environments grow larger and richer, leading to context overflow and degraded reasoning capabilities. This paper addresses the fundamental need for an interface that allows LLMs to reason over complex 3D environments without being constrained by the rigid limits of token-based context management.

The authors introduce a novel Retrieval Augmented Generation (RAG) framework that replaces full graph serialization with a structured graph database interface. Technically, the system encodes 3DSGs into a graph database and equips the LLM with the Cypher query language as a functional tool. When processing natural language inputs, the LLM generates Cypher queries to dynamically retrieve only the specific, relevant subsets of the scene graph required for the task. This selective retrieval mechanism allows the model to interact with rich world modelsâ€”comprising voxel maps, objects, and regionsâ€”without overwhelming the context window.

Empirical validation against baseline methods, including context window stuffing and code generation, demonstrates that the Cypher-based interface yields substantial quantitative improvements. On instruction following and scene question-answering tasks, the proposed method improved accuracy by approximately **20â€“30%** over code generation baselines and successfully processed large graphs where context-stuffing methods failed entirely due to overflow. Furthermore, the framework drastically reduced computational overhead, decreasing the token count of processed scene data by **up to 98%** (e.g., reducing inputs from over 40,000 tokens to fewer than 1,000 tokens per query) while maintaining performance across both local and cloud-based LLMs.

This work represents a significant advancement in the field of robotic spatial reasoning by successfully applying RAG principles to 3D scene understanding. By decoupling scene representation from the LLM's context window, the research solves a persistent scalability bottleneck and provides a more computationally efficient pathway for grounded language tasks. The findings establish that structured query interfaces are a viable and superior alternative to text serialization, paving the way for deploying LLMs in complex, real-world environments where data richness and scale are paramount.

## Technical Details

The system utilizes a Retrieval Augmented Generation (RAG) framework to bridge natural language inputs with 3D world representations.

### Core Architecture
*   **3DSG Storage:** 3D Scene Graphs (3DSGs) are encoded within a graph database rather than serialized as text.
*   **Query Interface:** The LLM is provided with a structured query language interface (Cypher) as a tool.

### Operational Workflow
1.  **World Modeling:** The robot maintains a structured world model (voxel maps, object detections, regions).
2.  **Query Generation:** The LLM generates Cypher queries from natural language commands.
3.  **Data Retrieval:** The system retrieves only the relevant subset of the 3DSG based on the query.
4.  **Deployment:** The approach is designed to function with both local and cloud-based LLMs.
