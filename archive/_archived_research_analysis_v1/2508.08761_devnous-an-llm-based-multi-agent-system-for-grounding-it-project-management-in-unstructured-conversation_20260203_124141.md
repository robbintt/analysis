## Executive Summary

IT project management faces a persistent disconnect between the informal, unstructured communication that occurs in team chat environments and the structured, formal data required by project governance tools. This gap forces teams to manually extract actionable intents, update tasks, and synthesize progress reports, creating significant administrative overhead and increasing the risk of data loss or misinterpretation.

This paper addresses the challenge of automating the translation of natural language dialogue into the structured artifacts necessary for effective project oversight, aiming to reduce the friction between daily collaboration and formal management requirements. The authors introduce **DevNous**, a hierarchical multi-agent system designed to function as an "ambient administrative agent" directly within communication platforms.

Technically, the architecture utilizes a combination of handcrafted agent personas and a hybrid memory system—comprising both short-term and long-term components—to maintain context across ongoing conversations. It employs a constrained ReAct-style feedback loop to facilitate stateful reasoning, enabling the system to autonomously manage complex, multi-turn administrative workflows rather than relying on single-turn interactions.

To evaluate the system's efficacy, the researchers developed a new benchmark dataset consisting of 160 realistic, interactive conversational turns. DevNous demonstrated high precision in translating dialogue into structured data, achieving an **Exact Match Turn Accuracy of 81.3%** and a **Multiset F1-Score of 0.845**. These results highlight the system's robustness in converting unstructured dialogue into formal project operations, providing compelling evidence that LLM-based multi-agent systems can effectively bridge the gap between informal team communication and formal IT governance requirements.

## Technical Details

DevNous is a hierarchical multi-agent system designed to operate autonomously in communication-centric environments, bridging informal dialogue with structured project artifacts.

*   **Architecture:** Hierarchical multi-agent system.
*   **Agent Design:** Utilizes handcrafted agent personas.
*   **Memory System:** Implements a hybrid memory system consisting of:
