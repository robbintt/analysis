## Executive Summary

The paper addresses the critical barrier to adopting autonomous AI agents: **user hesitation** stemming from a lack of trust in delegating complex, real-world workflows. As agentic systems become more capable, users struggle to align autonomous behaviors with their specific values and manage the inherent uncertainty of execution. This reluctance creates a significant gap between the potential efficiency of agentic workflows and the human need for control, transparency, and reliability. Without effective mechanisms to bridge this gap, users are prevented from leveraging these systems for high-stakes or sensitive tasks.

The authors introduce **DoubleAgents**, a planning tool built on the ReAct framework that integrates **interactive simulation** to enable iterative alignment. The system utilizes a multi-agent architecture that separates "Core ReAct Agents" for execution from "System Orchestration Modules" for management. Key technical components include a central Coordination Agent, Simulated Respondents (background agents that emulate human behavior to generate realistic scenarios), and a Context Management Module that maintains dynamic state via JSON structures. The user interface features a Policy Panel for editing coordination rules, approval mechanisms for plan/action pop-ups, and an Assignment Tracker, allowing users to rehearse scenarios and define executable stop hooks within a safe environment before live deployment.

The system was evaluated through a mixed-method approach comprising a two-day controlled lab study (n=10), three real-world deployment studies, and a specific technical evaluation. While the study primarily yielded qualitative findingsâ€”showing a distinct behavioral shift where users moved from initial hesitation to increased delegation after using the simulation for calibrationâ€”the paper supports the fidelity of this approach with quantitative context from related literature, noting that LLM-based simulations replicate behaviors observed in over 80% of psychology experiments. The deployment studies confirmed the system's utility for high-complexity tasks, defined in this context as multi-step workflows requiring multi-agent coordination and the management of uncertainty. Results demonstrated that rehearsing and refining policies within the simulation proved effective for calibrating agentic behavior before live execution.

This research validates interactive simulation as a practical pathway for aligning agentic AI with human expectations, effectively bridging the gap between system efficiency and user values. By embedding specific mechanisms for transparency, control, and uncertainty management, DoubleAgents provides a blueprint for designing agentic systems that users can confidently rely on.

## Key Findings

*   **Initial Hesitation:** Participants initially exhibited hesitation in delegating tasks to the agent, highlighting a significant barrier to the adoption of agentic workflows.
*   **Simulation as a Probe:** Users utilized the built-in simulation to probe system behaviors and adjust policies, which facilitated a gradual increase in delegation.
*   **Real-World Utility:** Deployment studies confirmed that the system is relevant and useful for managing real-world tasks, particularly those characterized by high complexity and uncertainty.
*   **Calibration via Rehearsal:** The process of rehearsing and refining policies within a simulation environment proved effective for calibrating agentic behavior before live deployment.

## Technical Details

### Architecture & Framework
The system is built upon the **ReAct (Reasoning + Acting) framework**, utilizing a multi-agent architecture with human-in-the-loop interaction.

*   **Core ReAct Agents:** Responsible for execution tasks.
*   **System Orchestration Modules:** Responsible for high-level management.

### Key Components
*   **Coordination Agent:** Acts as the central planner.
*   **Simulated Respondents:** Background agents that emulate human behavior to generate realistic scenarios.
*   **Context Management Module:** Maintains dynamic state via JSON structures.

### User Interface Features
*   **Policy Panel:** For editing coordination rules.
*   **Interactive Chat Interface:** Direct communication channel.
*   **Plan/Action Pop-ups:** Mechanisms for user approval.
*   **Assignment Tracker:** Features color-coding for status tracking.
*   **Communication History/Calendar Views:** For oversight and logging.

### System Input/Output
*   **Inputs:** User goals, personas, templates, and policies.
*   **Outputs:** Finalized assignments, refined policies, updated templates, and executable stop hooks.
