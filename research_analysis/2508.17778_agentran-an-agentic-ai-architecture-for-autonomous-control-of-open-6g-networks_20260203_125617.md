# AgentRAN: An Agentic AI Architecture for Autonomous Control of Open 6G Networks

*Maxime Elkael; Salvatore D'Oro; Leonardo Bonati; Michele Polese; Yunseong Lee; Koichiro Furueda; Tommaso Melodia*

---

## Executive Summary

As networks evolve toward Open RAN and 6G architectures, the complexity of managing radio resources across disparate time scales, spatial domains, and protocol layers has outpaced the capabilities of traditional static automation. Current approaches often rely on "black box" machine learning models that lack explainability and require extensive pre-existing datasets to function, resulting in a significant "cold start" problem. Consequently, operators face a critical challenge in translating high-level business intents into low-level, real-time control signals without manual intervention, creating a need for a more adaptive, transparent, and intelligent control paradigm.

AgentRAN introduces an AI-native, agentic framework that replaces static control loops with a hierarchy of Large Language Model (LLM)-powered agents acting as rApps, xApps, and dApps within the Open RAN architecture. The system utilizes an "Intent-Based Orchestration" mechanism where agents interpret natural language intents and negotiate control strategies, decomposing complex tasks across time, space, and protocol layers. A key technical component is the "AI-RAN Factory," a continuous learning loop that autonomously generates and refines control algorithms via code generation and model distillation. This architecture leverages LLM reasoning to provide bootstrapped intelligence with zero-start capability and ensures full transparency by making all agent reasoning auditable through structured conversations.

The framework was empirically validated in a live 5G Over-the-Air (OTA) private network environment, successfully managing a three-stage Intent Evolution scenario involving power control and scheduling. This research represents a paradigm shift from static, reactive network management to dynamic, intent-based autonomous control, effectively bridging the gap between high-level operator goals and low-level network execution. By solving the cold start problem through reasoning-based intelligence and introducing the concept of a self-improving "AI-RAN Factory," AgentRAN establishes a foundational architecture for future 6G networks. Furthermore, the emphasis on auditable reasoning addresses the growing demand for Explainable AI (XAI) in telecommunications, ensuring that network automation remains transparent and trustworthy as systems become increasingly autonomous.

## Methodology

The research implements **AgentRAN**, an AI-native, agentic framework designed specifically for Open RAN architectures. The methodology is built upon four core pillars:

1.  **LLM-based Agent Fabrication:** Utilizing Large Language Models to orchestrate distributed AI agents that function as the primary control mechanism.
2.  **Intent-Based Orchestration:** Agents interpret natural language inputs from operators and autonomously negotiate control strategies to meet those intents.
3.  **Hierarchical Organization:** Breaking down complex tasks across various dimensions:
    *   Time scales
    *   Spatial domains
    *   Protocol layers
4.  **Continuous Learning Loop:** Implementing an "AI-RAN Factory" to refine algorithms and generate new control logic.
5.  **Empirical Validation:** The approach was validated through rigorous live 5G network experiments.

## Technical Details

### Architecture & Hierarchy
AgentRAN utilizes a hierarchy of LLM-powered AI agents functioning directly as control applications within the Open RAN architecture:
*   **L3 Manager**
*   **L2 Manager**
*   **L1 Manager**
*   **Leaf Agents** (acting as rApps, xApps, and dApps)

### Timing & Alignment
*   **O-RAN Alignment:** The system aligns strictly with O-RAN defined timescales.
*   **Range:** Spans from Non-Real-Time (>1s) to Near-Real-Time and Real-Time (<10ms) operations.

### Communication & Protocols
*   **Protocols:** Agents communicate using **Agent-to-Agent (A2A)** and the **Model Context Protocol (MCP)**.
*   **Format:** Data exchange occurs over **JSON-RPC**.
*   **Logic:** Operational logic uses structured natural language for intent negotiation to ensure auditable reasoning.

### The AI-RAN Factory
A dedicated architectural component for continuous learning, utilizing:
*   Code generation
*   Model distillation
*   Fine-tuning
*   Hybrid creation of models
