## Executive Summary

Current security research in Large Language Models (LLMs) predominantly focuses on model-level threats, such as prompt injection and adversarial attacks, while largely neglecting the security of the underlying software supply chain. This gap is critical because LLM repositories rely heavily on complex, third-party dependency ecosystems (primarily Python) to function. If these foundational packages contain vulnerabilities, the integrity of the entire LLM system is compromised.

This paper addresses this oversight by investigating the prevalence and management of vulnerable package dependencies within open-source LLM repositories. The study introduces a novel three-step empirical framework designed specifically to analyze the LLM supply chain. By parsing source code and configuration files, aggregating vulnerability data from Snyk.io and Libraries.io, and analyzing maintenance practices via Pull Requests, the authors reveal a precarious security posture.

The empirical analysis indicates severe latency in vulnerability remediation: **50%** of the identified vulnerabilities remain undisclosed for more than **56.2 months**, a duration significantly longer than observed in the broader Python ecosystem. This research bridges a major research gap by providing the first large-scale empirical evidence of supply chain risks specific to LLMs, offering actionable insights for practitioners and establishing a vital baseline for future hardening of the LLM supply chain.

## Methodology

The researchers conducted an empirical analysis on **52 open-source LLM repositories**, specifically examining their third-party dependencies and associated security vulnerabilities.

The methodology consisted of three primary activities:
1.  **Repository Analysis:** Examining activities within the repositories to understand how maintainers manage third-party vulnerabilities.
2.  **Benchmarking:** Comparing findings against the general Python ecosystem to highlight security timeline discrepancies.
3.  **Data Collection:** A hybrid search strategy combining an academic corpus (Hou et al.) and a Multivocal Literature Review (MLR) was used to construct the dataset.
