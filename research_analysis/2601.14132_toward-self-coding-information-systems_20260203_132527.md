### Definition
A self-coding information system is defined as an information system capable of **generating and utilizing the source code of its own subsystems during runtime**. This capability is enabled primarily by agents based on fine-tuned language models.

### Architectural Differentiation
The paper distinguishes SCIS from existing paradigms:
*   **vs. Autonomic Computing:** SCIS focuses on full code generation rather than simple patching.
*   **vs. Self-Adaptive Systems:** SCIS generates entirely new code rather than using dynamic weaving techniques.
*   **vs. Configuration-Based Systems:** SCIS performs direct modification of source code rather than adjusting configuration parameters.

### Technical Trade-offs
The authors identify several critical challenges and trade-offs:
*   **Analyzability:** Difficulty in analyzing system behavior due to constant evolution.
*   **Functional Correctness:** Risks associated with non-determinism in code generation.
*   **Resource Utilization:** High demand for computational resources.
*   **Hardware Requirements:** Need for specialized hardware to support fine-tuned LLM agents.
