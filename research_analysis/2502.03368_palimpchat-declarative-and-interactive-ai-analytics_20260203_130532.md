## Executive Summary

Declarative AI frameworks offer powerful methods for constructing machine-learning pipelines that can be optimized for cost, latency, and quality. However, these systems typically require significant technical expertise to define schemas, manage relational operators, and construct logical plans. This creates a substantial accessibility barrier, preventing non-expert users who possess domain knowledge but lack programming skills from leveraging high-performance AI analytics for complex tasks such as scientific discovery or data processing.

## Technical Details

### System Architecture
PalimpChat operates as a hybrid integration of two distinct components:

*   **Palimpzest (Declarative Backend):** Uses a relational algebra data model. Schemas are defined by attributes, types, and descriptions.
*   **Archytas (Frontend Framework):** Implements the ReAct paradigm to decompose natural language requests and interact with Palimpzest tools.

### Core Operators
The system utilizes a variety of operators to manipulate data:
*   **Convert:** Transforms Schema A to Schema B using LLMs.
*   **Filter:** Refines data using natural language or User-Defined Functions (UDFs).
*   **Standard Relational Operators:** Traditional database operations (e.g., Join).

### Optimization Strategy
The system employs a multi-objective optimization approach:
*   **Separation of Concerns:** Logical plans are separated from physical implementations.
*   **Ranking:** Plans are ranked based on user-defined preferences for **Quality**, **Runtime**, and **Cost**.

### Workflow Execution
The operational pipeline follows this sequence:
1.  **User Input:** Natural language request.
2.  **Agent Decomposition:** Archytas breaks down the request.
3.  **Plan Generation:** Optimizer generates execution plans.
4.  **Execution:** The pipeline is run.
5.  **Result Return:** Data is returned to the user.
